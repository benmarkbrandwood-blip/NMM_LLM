# Mill Training and Acceptance Plan v5

> Core objective: make the machine play like a mature human player—establish a sound position first, then tighten the position, wait for the opponent to err, and finally convert the resulting advantage reliably—while shipping a compact product. The complete Malom tablebase exists only in the training and acceptance environments; it is never included in the release bundle.

## Evidence Boundary and Shortest Closed Loop for the v4 Baseline

The v4 training pipeline—three specialists plus generalist self-play; see `docs/v4-specialist-plan.md`—has already completed an end-to-end run at relatively low cost and produced tangible improvements. After approximately 16,000 training games, the generalist was demonstrably playable against this repository's 11–14-ply Alpha-Beta engine, including one game that exhibited a multi-capture tactical sequence. A high draw rate could cause noise to breach the old recovery condition, `win_rate < 0.12`: a draw counted as 0.5 but was included in neither the win count nor the loss count. Replacing the trigger with `loss_rate > win_rate` corrected an obvious semantic error in the metric. These achievements must be preserved: **v4 is this plan's low-cost playing-strength prototype and a candidate source; it is not being repudiated.**

Before deciding whether v4 is sufficient, verified facts must be separated from claims that remain to be tested. The most consequential discrepancy is the kind of semantic error that readily arises in the absence of a differential gate against a reference implementation. Stage 0 therefore makes differential comparison the first gate; until that gate passes, training curves cannot substitute for correctness evidence.

### A Prerequisite Fact: the Current Malom Decoder Disagrees with the Reference Semantics

The current Python decoder, `ai/malom_db.py`, diverges in two respects from the semantics of the Sanmill reference implementation at `crates/perfect-db/src/database.rs`, relative to the checkout selected by the `sanmill_checkout` key in the [local training layout](local-training-layout.md). `decode_entry` classifies non-virtual intermediate values as W or L directly from the sign of `raw key1`, while `MalomDB.query` does not use the sector value already loaded by `_load_secval`. The correct reference semantics are `absolute_key1 = raw_key1 + sector_value`, with W or L assigned only when the absolute value equals the global `virt_win` or `virt_loss`; every other value is D. Direct inspection shows that **491 of the 498 sector values** in `std.secval` are non-zero. This is not a rare boundary case but a systematic discrepancy affecting the overwhelming majority of sectors; it misreads many ultra-strong draw grades as wins or losses.

The consequences are not confined to charts. `learned_ai/models/lookahead_advisor.py` uses this query for **early termination** of 12-ply trajectories; an incorrect W/D/L value can turn a trajectory that should continue into a spurious terminal and write it into the training features. SpecialistDB merges `malom_label` values using a “retain the first write” rule, so an incorrect label can persist indefinitely once inserted. The dashboard's `malom_win_move_rate` also uses this value in its numerator. Consequently, until the decoder is corrected and passes field-by-field comparison against Sanmill, **existing checkpoints and Malom labels already written to storage are exploratory artifacts only**. This is a matter of evidence level, not a judgment about playing strength.

### What the Dashboard Metrics Actually Measure

The existing training curves are useful **training-health monitors**, but under their implemented definitions they cannot answer the acceptance questions:

| Curve | What it actually measures | What it does not establish |
| --- | --- | --- |
| `malom_win_move_rate` | Among successfully queried selected moves, the fraction with a quality score ≥ 0; D→D and L→L both count as successes, unknown values are excluded from the denominator, and labels come from the decoder that still requires correction | Win rate, agreement with the full comparator, or theoretical-safety rate |
| `policy top-1` | The fraction of sampled actions that happen to equal the model's own argmax; it rises naturally as the temperature anneals from 0.9 to 0.2 | Predictive accuracy or playing strength |
| `best win rate` | The historical peak persisted by the checkpoint after repeated inspection of rolling windows | An unbiased estimate of the current win rate |
| win/draw rate | A mixture of self-play, heuristic games, branch/retry samples, and difficulty levels that change during training | Controlled win rate against a frozen opponent |
| policy/value loss | Training loss; the Generalist panel has shown value-loss spikes on the order of $10^{20}$ | That the spikes have been eliminated; this requires a new, corrected training run |

The correct recovery semantics still require a dedicated regression test and a corrected, frozen training run. `loss_rate > win_rate` has neither a minimum effect-size requirement nor a confidence gate. For example, a one-game difference in a window containing 15 wins, 69 draws, and 16 losses would still trigger the disruptive sequence “reload checkpoint + rebuild optimizer + reset temperature.” It is therefore a candidate fix, not evidence of improved playing strength.

### What the Three Recorded Games Can Establish

The three recorded games in which the generalist faced this repository's 11–14-ply Alpha-Beta engine demonstrate that the single network has local playing strength and is playable. That is genuine progress. They do not, however, contain terminal markers; they have n = 3; and they lack colour-swapped pairing and a baseline control. In the first game, moves 12–15 enter a `d2↔f2` / `g1↔d1` repetition, which also exposes the engine's existing repetition-adjudication defect described in §5. Moreover, the tested object is a single generalist network, whereas the core architecture defined by the v4 document consists of three independent specialists: what was demonstrated and what was claimed to be “sufficient” are not the same object. These game records therefore cannot estimate the generalist's win rate against a frozen 11–14-ply Alpha-Beta baseline, the theoretical W/D→L downgrade rate, conversion of already-won positions, improvement against the target human distribution, or stability after swapping colours.

In addition, all four current trainers zero out the Malom reward—the comment says that it “encourages safe draws rather than winning”—while retaining a draw penalty and bonuses for Mills and exploration. In a game whose perfect-play outcome is a draw, that objective has a structural tension with “avoid losing first.” Item 5 in the v4 document's own Next Steps already proposes an ablation of these flat rewards. This document merely moves that ablation earlier and requires it to be evaluated over a theoretically safe denominator (§5 and §13.4).

### Combined Route: Let Each Track Do What It Does Best

Implementing all of v5 in full is currently too heavy. The remedy in this document is gated, incremental execution with expensive components activated only conditionally—not a retreat to a route without acceptance criteria. The shortest combined path is:

1. Stop treating the current curves as Malom evidence. Retain the checkpoints as exploratory baselines; training may continue, but its conclusions are limited to health monitoring.
2. Complete the minimum correctness core first: the Malom comparator and differential gold set in P0-A, rules/history/atomic capture in P0-B0, and oracle-gate/proof semantics in P0-B1. Deliver each as a separately time-boxed, independently acceptable slice; do not commit to a schedule before resources have been measured.
3. If v4/SpecialistDB is to be reused as a candidate source, regenerate the Malom labels after the fix, then retrain the existing checkpoints or perform a rigorous contamination ablation. This migration does not block construction of a reference-only student directly from `q_ref`.
4. In parallel, treat any usable v4/generalist as a candidate source and compare it with the gated ultra-strong reference and the current Alpha-Beta system in **frozen paired tests**—colour-swapped, fixed starts, and interval-reported—measuring the W/D downgrade rate and certified-W conversion rate.
5. If the `q_ref` reference-only bundle already satisfies the product thresholds, stop honestly at `compact_reference_candidate`; do not build `T_full`, HumanPolicy specialisation, DAgger, RiskSurvival, or PPO. This is an explicitly anticipated normal outcome, not a failure. The name and conclusions assigned to a v4 source depend separately on its actual lineage and the gates it has passed.
6. Activate later modules one gate at a time under §§12 and 18 only if the minimum closed loop demonstrates that the candidate is insufficient.

This route branches at P2-B1 in §18. If reference-only closure is sufficient, proceed directly to P3 and the reference-release path. Enter P2-B2 only when a concrete gap remains and the human-player branch is feasible. In one sentence: **v4 remains worth pursuing as a low-cost playing-strength prototype; however, the evidence is presently insufficient to conclude that “v4 is enough” for this document's two co-primary effects and safety objective, while “full v5 is necessary” is likewise not the default conclusion. The optional research stack is deferred behind gates, but the three foundational layers—oracle correctness, rule semantics, and frozen acceptance—cannot be removed.**

## Design Boundaries and Execution Path

The objective is closed separately for each delivery environment. The strict objective of the **oracle workbench** is to preserve the full-rules value whenever a non-losing line exists, while improving practical win rate and winning-position conversion rate against a specified distribution of human players. If the current state is already L, it prioritises a finite-horizon safe line that guarantees survival for at least $X$ ply and seeks to induce an error that rescues the game to a draw. In ordinary states not covered by its tablebase pack, the **compact release product** cannot prove move by move that the Malom W/D value has not regressed. It therefore adopts the explicitly weaker standard signed by the product owner before the first gate: when the current state is W or D, the selected move must either preserve the mode-relative positional value under an exact-pack proof, or have a completed adversarial proof showing that the strongest opponent cannot force a terminal win within the currently effective $X_{rt}$ ply. When the current state is L, a pack hit may prove retention of the best positional L tier, but only a completed survival proof warrants the term `runtime-loss-bound(X_rt)`. Without either form of evidence, the move must not be described as safe. If every candidate has been proved to fall short of $X_{rt}$, selecting the line with the longest survival is merely a best-effort shortfall; that state or game instance does not satisfy the finite-horizon safety objective. `theory-safe`, `horizon-safe(X)`, and the compact product's `runtime-loss-bound(X_rt)` must be reported separately. Rules, positional game-theoretic value, human observations, and model predictions occupy different evidence layers and must be physically separated in data structures, training objectives, and acceptance reports. These boundaries are grounded in the project code, the Malom implementation selected by `sanmill_checkout`, the standard tablebase assets selected by `malom_db_path`, and the corresponding primary research.

### Two Delivery Environments

Product size is a hard constraint. The approximately 83.6 GB `std` positional tablebase is a development asset and never enters the release bundle; the exact byte count is fixed by the asset manifest frozen in P0-A.

| Environment | Composition | Safety semantics |
| --- | --- | --- |
| **Oracle workbench**—training, labelling, acceptance, and research; the only environment that mounts the complete tablebase | Rules engine, the 498-sector `std` assets, the Sanmill reference, RuleAwareSafetyGate, and the full teacher chain | Online hard guarantees for `theory-safe`, W-liveness, and `horizon-safe(X)`; all label generation and acceptance in this document occur here. It may serve as a local research/analysis play mode with the database mounted, but it is not a release form. |
| **Compact release product**—the only release form | Rules engine, unified deployable policy, safety-prior head, compact calibration/OOD manifest, optional `endgame_exact_pack`, and finite-horizon survival-proof search | Legality is a global hard guarantee. An exact positional value exists only on a complete pack hit, and `runtime-loss-bound(X_rt)` exists only after search has completed a proof for the selected action. Theoretical non-loss is a Stage 3 offline measurement only—`offline-verified`—and must not be described as certified. |

See §6.7 for the runtime protocol and pack rules, §12 Stage 3 and §15.4 for deployment closure, distillation, and offline measurement, and §17 for size and latency budgets.

### Four Product Axes That Cannot Offset One Another

Every compact candidate must answer four questions separately; they must not be collapsed into a composite score:

| Product axis | Core question | Decision rule |
| --- | --- | --- |
| **Safety correctness** | Can the rules, pack, proof system, or authorisation state machine admit an action that should not be admitted? | False authorisation, a move made without authorisation, or an incorrect safety claim is a `fatal_stop`; a higher win rate cannot compensate for it. |
| **Safety availability** | Under frozen hardware, deadline, support domain, and $X_{rt}$, can the system produce an authorised action often enough? | `runtime_bound_unavailable`, X-viability failure, and proof-pool censoring are each adjudicated under frozen caps and intervals. Exceeding a gate means that safety contract is not deliverable; it does not license a fabricated safety conclusion. |
| **Playing strength** | Relative to a reference using the same runtime stack and to the current product control, does the system preserve theoretical or finite-horizon outcomes and improve the corresponding outcome measure? | Use frozen paired controls and endpoints appropriate to the candidate's name. Proxy evidence cannot substitute for the human co-primary effects. |
| **Latency and resources** | Do bundle size, RSS, startup time, per-move latency, and energy consumption meet the target-device budgets? | Gate independently. Passing is not permitted by weakening proofs, concealing refusal to move, or deleting difficult states. |

After P2-B1 produces the first availability curve on target hardware—and before any specialised-candidate results are inspected—the product owner must sign `product_availability_decision`: either retain the default strict fail-closed behaviour, or define a separate availability-relaxed bundle with a different product name and `runtime_safety_contract`. The support domain, $X_{rt}$, deadline, refusal/adjudication rules, and every availability cap are frozen at the same time. The contract cannot later be switched in response to poor candidate performance. If any mandatory gate on any one of the four axes fails, improvements on the other three cannot promote the candidate.

### Deployment Host, Sole Move Authority, and External Claims

The repository's default user entry points, `run_nmm.bat` and `run_nmm.sh`, launch `uvicorn web.app:app`. `main.py` is a separate console entry point; merely residing in the same repository does not make it the same release product. The first compact release defaults to this Web/FastAPI process as its canonical deployment host, but P1-B0 must freeze that decision in `deployment_host_contract`: entry command and process, supported game modes and API routes, the adapter from the actual board state to `DecisionState`, bundle loader, sole move-authorisation boundary, the mapping from `shortfall/unavailable` to UI/API/match results, logs and user-visible surfaces, and whether `main.py` is explicitly out of scope or reuses the same adapter. A later host change requires a new contract and plan and a rerun of integration, closure, hardware, and release gates; replacing only the launch script is insufficient.

At every move in v5 compact mode there is exactly one final action authority: rules-state adaptation → candidate ordering by the compact policy → authorisation and adjudication by the pack/proof layer. Existing `GameAI` score adjustments, Sentinel, specialist/generalist routers, Gap/Value networks, direct Malom branches, and `SpecialistDB` post-game write-back must not co-decide the action, override the selected action, or silently take over on error. They may exist only in separately named control or shadow modes. The LLM may explain; it cannot obtain move authority. If `main.py` is included in the support domain, an identical state/bundle/configuration must yield the same action, authorisation source, and refusal reason as the canonical host. Otherwise, the release manifest must explicitly exclude it. P2-B1/P3 measurements of availability and latency, and their fault injection, must traverse this real host path; a standalone benchmark harness may supplement but not replace it.

Release wording is itself a testable release gate. `product_claims_contract` enumerates the exact language permitted in the UI, API, release portions of the README, settings/help, and startup/status output for each candidate name and evidence state. `claims_surface_gate` performs a context-aware scan and behavioural snapshot of the surfaces actually packaged. The compact product must not call itself “theoretically non-losing,” “certified,” or “perfect-play,” and it must not conceal unavailable or shortfall states. This does not wrongly prohibit factual descriptions of the external Malom tablebase or of the oracle workbench as a perfect positional tablebase. If a control such as “Use Malom perfect DB” is retained, it must be either an unpackaged oracle/developer mode or a clearly separate non-compact product; it cannot be enabled covertly in compact mode. Copy, route, and API-schema hashes are frozen with the bundle hash. Any user-visible claim that exceeds its evidence level blocks release and cannot be cured by a disclaimer in the footer.

### Semantic Closure from Training to Release

The compact product is an independent inference graph, not “a Stage 2 checkpoint with the oracle unplugged.” It must satisfy six closure invariants:

1. At runtime, every input, threshold-based route, and head selection may read only state fields genuinely available in the bundle. Oracle W/D/L, proof rank, `A_allow`, HumanPolicy support, and teacher tier may generate supervision or evaluation strata; they cannot become inference inputs for states that miss the pack.
2. Stage 2 W/D/L heads that depend on oracle mode cannot be packaged directly. Stage 3 must distil the frozen multi-head teacher distribution into a unified compact policy that reads only deployable fields, or prove that the selected router is genuinely computable at runtime. The default is one unified policy; predicted W/D/L is not treated as a reliable router.
3. The Stage 2 policy loss is trained only inside `A_allow` and therefore cannot constrain logits for oracle-denied actions. Compact deployment distillation must explicitly assign zero target mass to every proved-denied action across the full legal action set and audit this independently against the safety-prior head. Neither component, however, receives authorisation authority.
4. A compact online guarantee can arise only from mode-relative positional closure under an exact-pack hit, or from a completed $X_{rt}$-ply adversarial proof for the selected action. A pack value in an L state cannot replace a survival proof. A safety prior, policy confidence, a partially searched bound, or a low offline error rate cannot authorise an action. If the deadline arrives with unresolved candidates and no authorised action, return `runtime_bound_unavailable`. The default product stops the game instance and adjudicates it under the signed rule, because continuing could overlook a still-unresolved action that would in fact have survived. Permitting an unproved best effort requires a separately named availability-relaxed product mode that makes no safety claim. It remains a bundle variant within the compact release family, bound to a different `runtime_safety_contract`; it is not a third delivery environment.
5. During development, `compact_exposure_gate` must operate on states reached in closed loop by the compact runtime stack itself. If it finds an oracle-only feature, an unseen route, an OOD/unsupported state, or an excessive theoretical downgrade, return to a new Stage 2 data version for additional labelling and retraining; do not patch the issue in place on confirmation or test data.
6. The release support domain cannot be made vacuous by excluding difficult states after the fact. Before the first compact gate, `release_support_contract` freezes the target variant, start distribution and minimum mass, a membership predicate formed only from runtime-available fields, and the closure obligation that “after every legal opponent reply, the bundle's next decision remains within the support domain or the game instance fails explicitly.” Closed-loop sampling can estimate exposure, not prove global closure. Apply model checking where enumeration is possible; elsewhere use adversarial state generation, bounds, and runtime fail-closed behaviour. Any escape first discovered during confirmation or final evaluation fails the candidate; the domain may not then be narrowed.

The resulting names are precise. `reference_safe_baseline` denotes only the exact, oracle-gated baseline. A tablebase-free bundle built from a reference source and taken through full P3 release closure is `compact_reference_candidate`. A release-form specialised source that has passed P3 closure and the target proxy is `compact_proxy_specialised_candidate`. Only a bundle that passes both prospective human co-primary effects and the compact safety gates is `human_validated_release`. P2-B1 produces only a reference-only closure prototype and confers neither candidate nor release status. Any fallback artifact may have engineering value, but it does not automatically satisfy this document's strict objective of improving practical win rate and conversion rate.

The final product has exactly two co-primary effects. `human_match_win_rate` is the actual terminal win rate under the preregistered target human population and natural start distribution. By default, `W_conversion_rate` is the conditional proportion of wins from a preregistered suite of certified-W starting states shared by both arms. Both are measured on the **release form—the compact bundle**. Conversion after a natural game first enters certW is influenced by the D policy's choice of entry state, so it is a transport/mechanism metric by default. On the training side, legality, full-rules W/D non-regression, and W-liveness must first pass on the oracle workbench. Each compact move's guarantee is accounted for separately as exact-pack positional evidence or a complete $X_{rt}$ proof; a failure to authorise enters a separate safety-availability ledger and cannot be disguised as a safe move. The candidate then passes the §15.4 offline safety gate, the §15.7 conversion-endpoint alignment gate, and the human A/B test. The treatment-minus-control lower bound for each co-primary effect must exceed its preregistered `delta_human.{win,conversion}`. `time_to_convert` is secondary and is considered only when conversion itself is non-inferior. First-error rate, `P(certW)`, safe width/mass, squeeze, strategic labels, and search disagreement are all training intermediates or mechanism metrics. Improvement in any intermediate without confirmed improvement in both human co-primary effects must not be called “better practical play.”

### Evidence Levels

Every quantity used below must belong to exactly one of the following categories; they must not be conflated:

1. **Deterministic truth**: rule legality, Malom positional W/D/L validated by three-way comparison, and the complete Malom ordering tuple.
2. **Observed fact**: an actual human move, actual Elo, or actual first error; valid only for the action and trajectory that occurred.
3. **Model estimate**: HumanPolicy; rollout estimates of $T_{pos}/T_{certW}$; and optional GapNet/SelfRiskNet outputs. Each must carry its data domain, policy/proof version, calibration error, and uncertainty interval.
4. **Diagnostic proxy**: safe width, entropy, search disagreement, and strategic labels. These may explain or screen experiments; they must not be called win rates, causal root causes, or theoretical truth.

### Decision Authority of Metrics

Evidence level answers “what kind of number is this?” Decision role answers “what may this number decide?” The two are orthogonal. Every `metric_id × stage × candidate` must be registered in the contract under exactly one of the following roles. An implementer may not elevate a metric's authority merely because its value looks favourable:

| Decision role | What it may decide | Typical metrics |
| --- | --- | --- |
| `release_gate` | Whether a **specifically named** bundle may be released | Rule/oracle/authorisation invariants, safety-availability caps, compact-reference playing-strength non-inferiority, size/latency budgets, sole move authority in the host, and claims-surface consistency. A specialised human release additionally requires both human co-primary effects. |
| `stage_gate` | Whether it is worth building or entering the next research stage | HumanPolicy support, recruitment power, MDE, valid-label yield and cost for `T_full`, abstention and distinguishable pairs, teacher/student regret, and independent proxy stability |
| `diagnostic` | Localisation, explanation, stratification, or formation of the next-version hypothesis | `policy top-1`, safe width/mass, squeeze, $T_{pos}$ first-error rate, entropy, search disagreement, and route/strategic statistics not preregistered as chain-coverage gates |

A `diagnostic` cannot by itself block release and cannot serve as evidence of improved playing strength. A `stage_gate` controls research investment only; it does not automatically become a release gate. If a quantity genuinely requires greater authority in another stage, create a new metric ID/version, state its hypothesis and threshold, and update the access ledger before viewing that stage's candidate results. One number cannot serve multiple roles opportunistically.

### Executable Decision Path

The default engineering path first demonstrates a reference-only compact closed loop; it does not build every research component concurrently. Readability and manifest preflight for all required roots remain the common starting point. Once that passes, a read-only HumanDB feasibility scan may run alongside the reference path, but the full HumanDB builder, HumanPolicy, and target teacher do not block reference-only closure:

```text
P0-0 + required-root preflight
├─ reference path: P0-A → P0-B0 (rules/history/atomic actions) → P0-B1 (oracle gate/proof semantics)
│                  → P1-B0 (host and minimum deployable schema) → P2-B1
│  ├─ closure fails: stop T_full/DAgger; repair the compact architecture, or end the compact path
│  └─ closure passes: take the reference source through P3 → compact_reference_candidate → P4/reference_release_gate
└─ human-feasibility branch: P0-D0 (read-only behaviour-support + trial-recruitment screen)
   ├─ behaviour support insufficient/unknown: do not build HumanPolicy/T_full; gather evidence or reopen after P0-E data collection
   ├─ trial recruitment infeasible/unknown: do not build product specialisation by default; a separately approved research-only plan may proceed no further than a proxy candidate
   └─ both feasible: an application for target specialisation is permitted

P2-B1 passes + P0-D0 permits progression + the reference prototype does not meet signed product requirements (or a separate, rigorous research objective exists)
→ P0-D (full builder/split) → P2-A → P2-B2 → P2-C → P3 (specialised source) → P4
```

Each decision node retains one core hypothesis and is adjudicated by the smallest experiment capable of testing it:

| Decision node | Required inputs | Sole core hypothesis | Minimum experiment | Branch result |
| --- | --- | --- | --- | --- |
| Reference semantics | P0-0; required roots readable | Oracle, rules, history, and atomic actions can form a consistent source of truth | P0-A differential gold set → P0-B0 state-machine enumeration → P0-B1 gate/proof fault injection | Repair the first failing sub-gate; enter the compact slice only after all three pass. |
| Reference compact closure | Semantic gates passed; P1-B0 | A `q_ref` student and pack/prover can satisfy all four product axes simultaneously on the target device | Minimum student on frozen starts, closure/exposure testing, and a hardware benchmark | Failure closes the expensive specialisation work; success permits the reference release path. |
| Human-target feasibility | Read-only P0-D0 inputs; signed delta and recruitment ceiling | Existing behavioural support and future recruitment can identify a meaningful improvement | Support/MDE/projected-abstention analysis plus an upper bound from cluster-aware trial-power simulation | If insufficient, stop or collect data; open specialisation only if both dimensions are feasible. |
| Selective specialisation | Reference closure; complete split; frozen HumanPolicy | A small number of high-support `T_full` labels can improve reliably over the ultra-strong reference | Planning/confirmation pilot plus minimum listwise distillation | Return to the reference on no stable gain; permit at most two DAgger rounds only after a gain. |
| Release | Frozen source and complete compact bundle | The candidate passes all four axes and the effect gates appropriate to its claims | Full P3 closure plus one-shot P4 acceptance | If it fails, retain the current control; if it passes, name it according to its evidence level. |

Migration of the old path in P0-C and the P1-A scenario library are parallel or on-demand deliverables, not default dependencies of P2-B1. They enter an implementation's dependency graph only when an approved implementation explicitly consumes their artifacts. A shared backbone, derived features, `aux_q_head`, RiskSurvivalModel, and PPO all lie outside the default critical path; their absence does not make the plan incomplete. DAgger may run zero rounds and at most two; it starts only when offline distillation has already shown a gain and correcting reached states has sufficient return on investment.

The first minimum closed loop fixes `target_specialisation.modes: [D]`. For W, first distil the `q_ref` permitted by an augmented-history winning-strategy proof and use continuous W chains to train reliable conversion. For D, apply target specialisation only in states with high HumanPolicy support, $|A_{allow}|>1$, and a distinguishable `T_full` result. L must have a complete `horizon-safe(X)` rules module and non-zero training samples under `T_ref`, while HumanPolicy-based swindle ranking is deferred. Activating target-specific W conversion or target-specific L rescue requires a newly approved objective and budget. This does not remove L-state runtime safety behaviour; it moves the most expensive and least reliable specialisation out of the first release.

## 0. Preface: What Exactly Are We Teaching the Machine?

### 0.1 Not “Capture Whatever You See,” but How to Conduct an Entire Game

A beginner looking at Mill most readily notices three things: form a Mill, capture a piece, then form another Mill. It is tempting to conclude that a machine becomes stronger merely by forming Mills sooner and capturing more pieces. Intermediate and advanced players know the opposite can be true. A Mill formed too early may pin down one's own pieces. Capturing too quickly may release an opponent with four pieces into flying. A threat can lose all restraining force once executed. An apparently quiet waiting move may in fact compel the opponent to dismantle their own structure first. A game lost on move 40 may have been poisoned by move 6.

*Discretion is the better part of valour*; *An ounce of prevention is worth a pound of cure*. Advancing before the position is secure is not courage. Preserving an escape route in advance is often worth more than finding ten brilliant moves after the position has deteriorated. This plan therefore trains not a capture-hungry machine, but a player capable of conducting a position: first ensure that a moment of impatience cannot surrender the theoretical result; then preserve enough retreats and conversion routes; next, progressively compress the opponent's comfortable choices; and finally wait for the opponent's first genuine error and convert the advantage. *Live to fight another day* does not mean avoiding combat forever. It means retaining the ability to continue the fight and to choose its moment: consolidate when the position is uncertain; hold the draw calmly when the opponent defends correctly; tighten gradually, without premature execution, when the opponent's structure loosens; remain safe when the opponent sees through a trap; and switch immediately from containment to conversion once the opponent errs.

### 0.2 Why Knowing W/D/L Is Still Not the Same as Winning in Practice

Once validated against the reference, the Malom DB can be viewed as an exact **topographic map of positional value**. It supplies not only perfect-play W/D/L, but also a first key that distinguishes numerous ultra-strong grades among theoretical draws and a second key that expresses distance within a grade. This multivalued ordering was designed precisely to play more effectively against fallible opponents without sacrificing the theoretical result. It must be the primary baseline for any learned error-inducing policy and must not be collapsed into coarse W/D/L during decoding.

A settled-position Malom query contains the board, each side's unplaced-piece count, and the side to move. The database family—`std`, `lask`, or `mora`—determines the variant outside the key. Neither the query nor the key contains repetition multiplicity or a no-progress counter. A formal decision therefore still requires `DecisionState = settled positional state + rule history + matched variant`. This project combines the move and any capture into one atomic candidate before querying, then uses the Malom positional value jointly with the historical rule state.

But a map does not tell the traveller which road an ordinary person is most likely to take incorrectly; which theoretically drawn line is hardest for a human to defend; which route demands a long sequence of counterintuitive, difficult-to-remember moves; or how to steer an opponent into positions familiar to oneself but unfamiliar to them. That is the distinction between theoretical and practical playing strength. If five candidates all preserve D, they are equal under coarse W/D/L, yet they may differ in the complete Malom ordering, the target Elo's behavioural distribution, and the risk of extrapolation beyond the data domain. One move may leave the opponent ten comfortable safe replies; another may leave only one awkward safe reply. **The central hypothesis to be tested in this document** is that some safe moves yield a higher practical chance of winning against a specified human distribution. Until the FrozenTeacher, proxies, and human trial provide evidence in succession, “the second move induces errors more effectively” must not be written as an established fact. In every case, a candidate must not endanger its own position first.

Malom therefore has three responsibilities. Coarse W/D/L is the inviolable positional safety boundary. The complete value ordering is the strong/ultra-strong baseline. The reference `move_value`/comparator preserves positional W and orders search. Under the full rules, eligibility to convert a W position is established by the augmented-history winning-strategy proof in §6.3. Human data, search, and policy models may propose target-Elo specialisation only within the action set permitted by the same positional W/D/L value and the corresponding historical proof, and must demonstrate an actual gain over the complete Malom baseline.

### 0.3 Why Alpha-Beta Must Not Pursue Depth Without Limit

Alpha-Beta is a lamp cast forward: a three-ply search illuminates nearby ground, while a ten-ply search reaches farther. If the student spars only against the deepest search, it learns that the opponent always sees the best defence; it becomes hard to defeat, but poor at inducing human errors. If it spars only against shallow search, it grows accustomed to unrealistic elementary mistakes and fails as soon as it meets a somewhat stronger player. The correct design uses sparring partners of several strengths: shallow profiles emulate players who see the immediate move but not its distant consequences; medium profiles emulate ordinary experienced players; deeper profiles emulate strong players; and the deepest search appears only for verification. Every profile must be calibrated against actual HumanDB moves from the corresponding Elo band. Depth is not a “truth knob”; it is a control for the sparring partner's strength and state of deliberation. The same Elo band should mix adjacent search profiles rather than face an identical machine in every game.

### 0.4 Threats, Patience, and Directional Constriction

*The threat is stronger than the execution.* An unclosed Mill may simultaneously pin one of the opponent's blocking pieces, force the opponent to attend to that side on every turn, and buy time for containment on the other side. Closing it immediately and capturing may instead free the opponent, or even grant flying by reducing them to three pieces. Feints, patient waiting, voluntarily opening a Mill, and consuming the opponent's placement opportunities all express the same principle: a move's value lies not only in what it does immediately, but also in what it prevents the opponent from doing later. Training therefore cannot assign a fixed bonus to “forming a Mill.” Whether to form it, when to form it, which piece to capture, and whether deliberately to avoid granting flying must all be judged by their game-wide consequences.

The “slow” in *Slow and steady wins the race* does not mean circling until a repetition draw. It must have measurable direction: preserve ample safe moves for oneself; steadily reduce the opponent's safe moves; crowd the opponent's pieces or force them into blocking duties; and recognise a winning position immediately when the opponent errs. The plan therefore separates the objective into a decision mode and a positional-management layer:

1. **Certified mode** on the oracle workbench: when a non-regressing W/D line exists, choose only from it. If the current state is already L, use `horizon-safe(X)` and report the finite guarantee honestly.
2. **`bounded-risk(X)` mode**, an isolated oracle-workbench research mode: when risk is explicitly accepted, an L action may be chosen, but the strongest opponent must be proved unable to win within at least $X$ ply. Models, metrics, and UI for this mode are isolated from certified mode.
3. **Practical positional management**: within the action set permitted by the active mode, select the move that best constricts the opponent, induces an error, and remains easy for the machine itself to convert.

### 0.5 Why Different Teachers Must Have Distinct Responsibilities

The complete training system resembles a chess school. The rules engine is the disciplinarian that first guarantees legality. The Malom reference is the marking standard, supplying complete positional values and the ultra-strong baseline. HumanDB is a recording of 79,253 human games and tells us only what people actually chose under different conditions. Alpha-Beta is an adjustable-strength sparring partner. The curated strategic-scenario library teaches the student to recognise containment, feints, sacrifices, and phase boundaries. FrozenTeacher performs versioned, multi-branch comparisons within the same permitted action set. If GapNet is enabled, it approximates only the teacher's two frozen trajectory risks: $T_{pos}$ is diagnostic only, while D-mode ranking reads only $T_{certW}$. SelfRisk measures the risk that the raw policy will later require intervention by the safety gate. The final student uses this knowledge to choose a move in play.

If every teacher simply assigns a score and the scores are averaged, absurd outcomes follow. Malom may state unequivocally that a move turns a draw into a loss, yet the move could still be selected after averaging because it forms a Mill immediately, scores well in shallow search, and is common in human play. Adjudication must therefore be ordered:

> First exclude illegal actions and theoretical downgrades; next compare robustness for the machine itself; only then compare which remaining move poses the harder problem for a human.

This is the most important design principle in the entire plan. Every database, model, ratio, and metric that follows exists to turn this sentence into an executable, auditable engineering system that cannot deceive itself.

### 0.6 What Machine Learning Is Actually Learning

Writing the words “patience,” “containment,” and “inducement” into code does not make a machine understand them. A minimal learning step is as follows: present the student with a state; enumerate every legal action; let RuleAwareSafetyGate produce the permitted set; let FrozenTeacher produce ordered groups, group mass, and uncertainty over the permitted candidates; let the student allocate probability over that same permitted set and compare it with the frozen target; adjust parameters so that its distribution approaches the teacher's; then use DAgger to collect states actually reached by the student and repeat labelling and distillation. The essential design choices extend beyond model selection: which states the model sees, who marks them, which forms of evidence may generate labels and which may only explain them, which principles are hard rules, which learning algorithm applies at which stage, and which unseen problems constitute the final examination.

From the raw 24-node graph, candidate action, and minimum rule state, the network learns reusable representations. It may compress Mills, blocks, 4→3 transitions, and phase boundaries into internal patterns analogous to the “chunks” of board shape used by human players. D4 consistency shares experience across rotations and reflections. At the **behavioural level**, HumanPolicy approximates the human choice distribution under specified conditions, but it does not establish that the model reproduces human reasons or plans. The claim is therefore deliberately limited: **the representation layer generalises patterns of board shape, and HumanPolicy partially fits human behaviour; there is no evidence that the system replicates human cognition.** OOD states must abstain or return to the direct teacher/reference. “More human-like,” “possessing different personalities,” and “more capable of defeating the specified humans” are three separate objectives. The first release promises only the last. If configurable styles are later required, sampling may vary only within statistical ties under the same `A_allow` and the same primary utility; style must not alter safety, W conversion, or the human co-primary effects.

### 0.7 Two Fundamentally Different Questions in One Position

**Question A: What is the value of this complete atomic action under positional perfect play?** This is a deterministic oracle question. The current `std` assets contain the 498 sectors declared by `std.secval` and the corresponding 498 `std_*.sec2` files. A positional value may be called ground truth only after the adapter has passed checks for variant, coverage, perspective, atomic capture, and differential agreement with the reference. The value is not a “90% chance to win”; it is the complete `(absolute_key1, key2)` ordering and its W/D/L projection. Let $M(S,a)$ be the reference `move_value`, from the current actor's perspective, after completing the placement or movement and any required capture, and let $V(S)=\max_{a\in A(S)}M(S,a)$. Under exact minimax, the actor cannot “upgrade” its current best value from D to W merely by its own choice; an advantage appears only after the **opponent** selects a reply below that opponent's current optimum. If the same-perspective trace appears to show the actor itself moving D→W, first suspect an error in perspective, side-to-move transformation, sector selection, or completion of the capture.

**Question B: What is the probability that a 1500-Elo human chooses a given move here?** This is a behavioural question, and its answer is inherently probabilistic. Percentages in the plan therefore do not conflict with the categorical W/D/L result, but every percentage must identify whose probability it is, under what conditions, and over which denominator. Malom W/D/L describes the trajectory under perfect play. HumanPolicy percentages describe which route a human is more likely to select. A GapNet percentage must name its endpoint: $T_{pos}$ is the diagnostic probability of a positional first error within the human's next several decisions; $T_{certW}$ is the D-mode conditional probability that the human first moves the AI into certified W under frozen policies for both sides, target Elo, proof semantics, and horizon. A SelfRisk percentage describes the probability that the shadow raw policy will require RuleAwareSafetyGate intervention in the future. A reported 90% may be nothing more than the sample fraction of decisions that preserved W/D/L.

### 0.8 Roles of the Different Learning Paradigms

| Method | Analogy | Responsibility in this plan | Primary risk and mitigation |
| --- | --- | --- | --- |
| Imitation learning | Observe how experts and ordinary players move | HumanPolicy fits “what this kind of player usually chooses”; each sample supervises only the action actually taken | Humans also err, and popular openings can swamp rare ones. It teaches what humans do, not what theory requires. |
| Supervised learning with answers | Solve tactical exercises with answer keys | Malom supplies the complete value tuple for every candidate; strategic scenarios provide intent labels; the safety auxiliary learns, over all legal candidates, which actions would require interception in the absence of the gate | Malom gives the result but not necessarily the best move against a human. Label responsibilities must not be reversed. |
| Self-play | Practise against other versions of oneself | Expose states reached by the student but absent from offline data, then have the frozen teacher relabel them | Both sides may drift together into conventions valid only inside their own world. The terminal result of joint drift cannot become the teacher. |
| Reinforcement learning | Discover behaviour from multi-step outcomes | Only an incremental experiment after ablation—E-PPO; a clipped surrogate guarantees neither improvement nor safety | Credit assignment is difficult. The default substitute is “frozen teacher → listwise distillation → DAgger.” |
| Adversarial training | Drill the problems the student fears most | Hard-state replay collects traps, shallow/deep disagreements, and 4→3 errors | It cannot replace the ordinary practical distribution; common human openings retain a minimum proportion. |
| Preference/ranking learning | Learn which move is better among several safe, reasonable moves | FrozenTeacher produces ordered candidate groups with confidence intervals inside `A_allow`; StrategyPolicy learns the listwise ranking | Declare A ≻ B only when intervals distinguish them. The AI need not resemble a human, but it must learn a more constricting order within the safe set. |

Trajectories continue to record the theoretical first error, earlier candidate precursors, and same-state counterfactuals. They generate attribution candidates to be tested; temporal precedence alone is not causal evidence. See §13.5.

### 0.9 Why There Are Multiple Models Rather Than One Composite Score

| Module | Plain-language question | Output |
| --- | --- | --- |
| RuleAwareSafetyGate | May this action be selected under the positional value and current rules history? | Permit/deny, oracle value, and rule-based reason |
| HumanPolicy | How does a person at this Elo usually reply? | Calibrated reply probabilities and uncertainty |
| FrozenTeacher | Within the safe set, which move has the higher robust target value? | Ordered candidate groups, group mass, confidence, and version |
| GapNet (optional) | Can it cheaply approximate the teacher's positional-first-error diagnostic and D-mode certified-W arrival risk? | Endpoint-specific hazard/survival, partial-identification interval, and OOD flag |
| SelfRisk—computed directly; Net optional | How heavily does the raw policy depend on the safety gate? | One-step mass and gate-intervention survival measured on the AI-decision clock |
| StrategyPolicy | Which safe action should ultimately be selected? | Distilled policy distribution, top group, and partial order |
| Phase representation | Which rules phase is each side in? | 3×3 encoding, reachability mask, and transition flags |

A shared encoder is only an engineering option to be ablated. The tasks have different data domains and targets. Establish independent-encoder baselines first; any shared design must demonstrate that it does not damage HumanPolicy calibration, safety-auxiliary recall, or StrategyPolicy teacher-following.

### 0.10 Why Safety Principles Must Be Hard Gates, Not Merely Rewards

Suppose the training score awards +1.5 for a win, −1 for a loss, and a small penalty for a draw. A gambling line that wins 35% and loses 65% can then have higher expected reward than a line that draws with certainty; the machine would rationally learn to gamble. The principles of oracle certified mode therefore cannot be implemented by tuning weights: do not turn W into D or L; do not turn D into L; never make an illegal move; in a W state, choose only an action that preserves positional W and belongs to an augmented-history winning-strategy proof; and require model versions and database schemas to match. These are gates or assertions. If `bounded-risk(X)` is enabled, it uses a different, explicit allow policy rather than lowering the weight of the same gate. Compact mode is not entitled to reuse the word “certified.” Its online authorisation can arise only from W/D exact-pack positional closure or a completed runtime proof; an L-state pack ordering cannot substitute for the latter. No matter how much the teacher likes a trap, how many HumanDB players chose it, or how highly a shallow search scores it, none can buy off the safety referee.

### 0.11 Sample Proportions Define Playing Style; What Training, Validation, and Test Mean

What the machine sees most often becomes what it regards as most important. Training exclusively on common fixed openings produces a student that memorises standard answers. Flooding it with unconstrained random positions wastes time on bizarre states that humans will never reach. The proportions of common, long-tail, trap, and unconventional positions therefore shape playing style directly: common samples ensure practical hit rate; long-tail samples preserve orientation after an opponent deviates; trap samples train the system both to recognise and to set inducements; unconventional samples teach genuine positional understanding. The proportions may change in response to recurring errors, but the fixed examination set never changes with them.

Data has three roles. The training set is daily practice. The validation set is a stage examination used to choose proportions, search profiles, and model versions. The test set is the final examination and is touched only after the design is frozen. Scores become inflated if the same player, a symmetry of the same game, or near-duplicate branches from a common source appear in both training and test. Split by player and time, and keep every symmetry and same-origin branch on the same side. A meaningful result remains stable across unseen players, unmemorised opening deviations, and unseen trap variants, and across Phase and Elo strata.

### 0.12 Complete Path from a Candidate Action to the Final Choice

On the **oracle workbench**, for label generation, acceptance, and research games, every move proceeds in this order:

1. The rules engine enumerates every legal candidate. A mandatory capture after forming a Mill is incorporated into the same atomic action, and only the settled state after the complete action is queried.
2. The Malom reference returns the raw value, sector value, absolute key, complete ordering tuple, and W/D/L.
3. RuleAwareSafetyGate combines repetition/no-progress history to construct the safe set and, in a W state, verifies an augmented-history winning-strategy proof.
4. The HumanPolicy ensemble predicts the target Elo's reply distribution, calibration error, and OOD status.
5. For every safe candidate, FrozenTeacher evaluates both human-probability branches and the strongest safe reply branch, computing one-step safe mass, multi-step survival, conversion, and a robust lower bound. A validated amortiser is invoked only when the direct budget is insufficient.
6. StrategyPolicy supplies a distilled distribution within the safe set. On high uncertainty or excessive teacher/student disagreement, execution falls back to FrozenTeacher; it never relaxes the safety gate.
7. After selection, legality, atomic-action semantics, positional value, rule history, and version assertions are replayed. The trajectory enters an append-only log; only states that pass offline counterfactual review enter the DAgger or hard-state correction sets.

In the **compact release product**, each move follows the §6.7 runtime protocol. The rules engine enumerates legal atomic candidates → on a complete `endgame_exact_pack` hit, if the current state is W or D, reconstruct the mode-relative positional set from exact values; if the state is L or the pack hit is incomplete, order candidates with the safety prior and policy and complete an $X_{rt}$-ply adversarial proof → the unified compact policy may choose only within the applicable positional/proved set → when no candidate is authorised, route separately to `shortfall` or `unavailable` and enter independent audit/availability accounting → write the event to a read-only log; do not learn online.

This process assigns machine learning to what it does best—reasoning about complex human tendencies and positional pressure—while reserving deterministic rules and theoretical boundaries for verifiable programs.

## 1. One-Sentence Conclusion

Training must separate the “strategic curriculum” from the “theoretical referee.” Curated strategic scenarios teach intent and pattern; Malom adjudicates win, draw, and loss. Responsibilities are as follows. The rules engine owns legal actions, terminal states, and history-dependent draw rules. After reference validation, the Malom DB owns the complete positional value ordering and its W/D/L projection, forming both an inviolable positional safety boundary and the ultra-strong baseline. HumanDB records only how actual humans at a given Elo moved; HumanPolicy estimates their behavioural distribution in supported states. Alpha-Beta profiles of different depths and styles act as sparring partners of different strengths. The curated strategic-scenario library supplies lessons in patience, containment, inducement, sacrifice, and phase transitions. A Stage 2 source learns to constrain humans only among moves permitted by Malom and the historical proof. Stage 3 then distils it into a unified compact policy that does not read the oracle; at runtime, that policy may move only within a W/D exact-pack positional set or among candidates authorised by a complete survival proof.

The system does not add together the scores of five teachers. Its fixed priority is: full-rules legality and terminal semantics > positional W/D/L > an augmented-history winning-strategy proof in W states > robustness constraints > calibrated human-pressure objective > style. In a D state, the Malom ultra-strong ordering is the mandatory baseline. A learned policy may deviate from that baseline only after frozen evaluation demonstrates that target-Elo specialisation is worthwhile.

## 2. Training Objective in Terms of Playing Strength and Strategic Understanding

Every move first asks three questions. Could this move turn a position that was not lost into a loss? If the position is already W, does this move belong to a winning strategy that covers the complete current history, rather than merely preserving positional W? After the move, how many natural, comfortable, and safe routes remain for the opponent, and is that estimate within the data domain?

The ideal move is not one that “forms a Mill immediately” or “captures immediately.” It preserves many safe retreats for the machine; leaves the opponent progressively fewer safe retreats; makes the opponent's most natural replies increasingly likely to plant the seed of a theoretical downgrade; never turns a win into a draw or a draw into a loss even when the opponent answers correctly every time; and switches from waiting to conversion as soon as the opponent errs. This is the machine-operational definition of the “constrictor strategy”: not a reckless strike, but tightening one coil at a time while always retaining room to breathe.

## 3. Core Strategic Principles Included in Training, and Scope Control

| Core principle | Plain-language meaning | Training signal that can represent it |
| --- | --- | --- |
| Patience and delayed execution | Even with a threat, do not rush to form a Mill; preserve its restraining force | W/D/L after the delay, own safe mass, and opponent safe mass |
| Containment and constriction | Do not rush to capture; block exits first and compress comfortable moves | Mobility, number of safe replies, and escape region |
| Feints and forced replies | Use a threat that need not be executed to make the opponent assume a blocking duty | Target-Elo reply distribution and its consequences |
| Structural sacrifice and capture choice | Give up a local piece or permit a Mill to protect more important connectivity and mobility | W/D/L after capture, replacement distance, connectivity, and blocking value |
| Early turning points and phase boundaries | An ineffective early placement may reveal its cost only after placement→movement | Theoretical first error, gate-intervention risk, and boundary counterfactuals |
| 4→3 flying decisions | Capturing the fourth piece may release the opponent rather than simply gain material | Capture-to-fly, capture/no-capture counterfactuals, and safe mass |
| Advantage conversion and zugzwang | When every opponent reply deteriorates, switch from waiting to conversion at the right time | Opponent safe mass approaching zero, switch timing, and W/D/L giveback |

### 3.1 Controlling Complexity

- Do not build a vast encyclopedia of named tactics. The first stage uses only the seven core principles above.
- Strategic labels describe intent only; they cannot directly designate the best action.
- Every scenario must be a legal position, include the complete rules state, and carry Malom labels for every candidate.
- Add a category only when fixed acceptance repeatedly exposes a stable new weakness.
- Core strategic scenarios are capped at 10% of supervised pretraining. Human data, Malom-labelled states, and adversarial positions remain the majority.

## 4. Which Teacher Does the Student Obey?

### 4.1 Fixed Teacher Hierarchy

| Level | Role | What it may decide | What it may not decide |
| ---- | ---- | ---- | ---- |
| 0 | Rules engine | Legal atomic actions, terminal states, repetition/no-progress, and variant | Positional perfect-play value |
| 1 | Malom reference | Raw/absolute values, complete comparator, positional W/D/L, and strong/ultra-strong baselines | Historical draw state or human probabilities |
| 2 | FrozenTeacher | Versioned, multi-branch counterfactual evaluation inside the safe set, with uncertainty output | Create theoretical truth or cross the gate |
| 3 | HumanDB/HumanPolicy | Observed human choices / calibrated behavioural probabilities for different Elo bands | Fabricate observed outcomes for unchosen candidates or declare theoretical correctness |
| 4 | Alpha-Beta profile | Short-term tactics, bounded-compute proxy, strongest safe reply, and difficult examples | Replace Malom or automatically equal a human |
| 5 | Curated strategic-scenario library | Curriculum, concept labels, and targeted stress tests | Directly designate the best move |
| 6 | Stage 2 StrategyPolicy | Distil the teacher and rank quickly inside the oracle safe set | Serve directly as a tablebase-free release policy; cross rule, positional-value, or liveness constraints |
| 7 | Compact runtime stack | Let the unified policy rank exact/proved candidates; record shortfall/unavailable | Authorise candidates from a safety prior or offline accuracy; describe a best-effort shortfall as safe |

### 4.2 Resolving Conflicts

- If Malom says a move lowers W to D, it cannot be chosen even if many strong HumanDB players selected it.
- When several moves all preserve D, report the complete Malom ultra-strong ordering first. FrozenTeacher may then specialise for the target Elo within the same coarse W/D/L class, but must retain the reference baseline as an active control.
- For every legal, settled position that matches the locked `std` variant and lies within the manifest-declared support domain, Malom must return a result. A null result must be treated as a missing database mount, missing sector, canonicalisation, hash, decoding, or rule-adapter error and must stop the training batch or current move immediately. Unsupported variants, intermediate states awaiting capture, and unexpected query failures use distinct status codes.
- A program without Malom is permitted only as an isolated baseline/ablation or as the compact release product defined in §6.7. It cannot be an automatic fallback in the oracle-workbench chain. The SelfRisk shadow computation reads the pre-mask distribution and terminates at the first action on which intervention would have occurred; it need not execute the unsafe action.
- The rules engine adjudicates history-dependent terminal rules, including repetition and no-progress counters, before positional coverage is considered. Positional-domain coverage is not a reason to ignore the historical rules state.
- Code and data-structure anomalies must surface through assertions or explicit errors; a silent fallback must not conceal them.

## 5. Assessment of and Trade-offs for the Current Repository Implementation

The current implementation serves only as a comparison baseline; it does not constrain the future design. The following issues must be addressed first.

### P0: Malom Decoding, Action Semantics, and the Reference Oracle

`ai/malom_db.py` reads each sector's `sval` in `_load_secval` and stores it in `_secvals`, but `decode_entry` never uses that value to recover absolute `key1`. It instead interprets non-virtual intermediate values as W or L directly from their sign, misclassifying ultra-strong draw grades as wins or losses. In the actual assets selected by `malom_db_path`, `std.secval` declares `virt_loss=-299`, `virt_win=299`, and 498 sectors. The directory contains the corresponding 498 `std_*.sec2` files, totalling 83,582,099,742 bytes. Asset completeness and adapter correctness are separate acceptance questions.

`crates/perfect-db/src/database.rs` and `wdl_plane.rs`, relative to the checkout selected by `sanmill_checkout`, already give the correct recovery for coarse W/D/L: `absolute_key1 = raw.key1 + sector_value`, and only the two virtual extrema map to W or L. Preserving the ultra-strong ordering requires more than calling an interface that collapses the value into `PerfectOutcome` or `(wdl, steps)`, and it precludes reuse of a simplified C API that still maps by sign. The reference is the complete behaviour of the C++ `gui_eval_elem2`, `PerfectPlayer::move_value`, `undo_negate`, and comparator, or an equivalent full-value API added to the Rust bridge.

The correct direction is:

- Return `raw_key1`, `sector_value`, `absolute_key1`, `key2`, entry kind, perspective, sector ID, and query status; do not first collapse them into a scalar score.
- Map to W or L only when `absolute_key1 == virt_win/virt_loss`; map every other legal value entry to D. Count entries and symmetry redirects must be parsed according to the file format before they are treated as values.
- Candidate ordering must apply side-to-move conversion, sector correction, `undo_negate`, and the two-key comparator equivalently to reference `move_value`. Do not assume that raw `key2` is always “smaller is better” across sectors or strictly decreases on every move.
- A move that forms a Mill must include a legal capture before it is queried. Malom does not represent the intermediate state “capture pending”; the adapter must preserve atomic-action semantics.
- The gold set covers placement, movement, and flying; both colours and side changes; sector boundaries; symmetry redirects; multiple legal captures after a Mill; and virtual terminal values. Python, the Sanmill bridge, and the reference must agree field by field.
- Until this passes, the output must not be used for the safety gate, training labels, full-value ordering, or any playing-strength percentage.

### P0: Historical Draw Rules Must Be Specified and the Implementation Corrected First

The current `game/game_engine.py` cannot serve as training truth. `_post_placement_moves` increments after every half-move following placement and does not reset after a capture, despite being described as “100 half-moves without capture.” `_is_threefold_repetition()` recognises only one fixed-length back-and-forth pattern rather than the general third occurrence of the same `DecisionState`. `BoardState.to_fen_string()` also omits repetition multiplicity and the no-progress counter.

Stage 0 must first lock the standard rules variant for this project and establish a single `RuleState`. It must define precisely what constitutes progress, the counting unit, reset conditions, claimable versus automatic draws, the repetition-equivalence key, and terminal semantics during placement. Training, search, caches, canonicalisation, logs, and evaluation all read the same implementation.

### P0: Specialists Must Not Override Safe Search Without Constraint

In the current high-difficulty mode—difficulty ≥ 9, the `_spec_mode` branch in `web/app.py`—the specialist scores every legal action and its argmax overrides the Alpha-Beta result. The router encoding also passes `db=None` to `score_moves` in `learned_ai/agents/specialist_router.py`. The future design must instead construct `A_allow` with RuleAwareSafetyGate first, permit the specialist to rank only within that set, fail immediately if any legal candidate query fails rather than allowing Alpha-Beta to stand in for Malom authorisation, and assert again before the final selection that the action remains in the candidate set.

### P0: Reward Direction Conflicts with “Avoid Losing First”

The current `scripts/train_s_{open,mid,end}_v2.py` scripts fix `WIN_REWARD=1.5`, `LOSS_REWARD=-1.0`, `DRAW_SHORT=-0.15`, and `DRAW_LONG=-0.25`, allowing a high-risk gamble to dominate a safe draw. A fixed `MILL_BONUS=0.20` can encourage forming a Mill too early, capturing too early, or releasing the opponent into flying. Future training assigns no unconditional fixed reward to forming a Mill, capturing, prolonging, or shortening the game. The objective must be interpreted relative to the starting theoretical W/D/L and the decision mode; see §§13.1, 13.4, and 13.6:

| Starting theoretical result | Acceptable result | Failure |
| --- | --- | --- |
| W | Preserve positional W, follow an augmented-history winning strategy, and ultimately win | W→D/L, a rules draw, or an unproved conversion line |
| D | Preserve D after the machine's own move; only the state after the opponent's first error becomes W | D→L |
| L | Prefer `horizon-safe(X)`; if unavailable, maximise the proved guaranteed survival distance and then seek a rescue to a draw within the same tier | Describing finite survival or the possibility of an opponent error as theoretical non-loss |

### P0: The HumanDB Schema and Split Cannot Support Generalisation Claims

The raw PlayOK records retain both players' Elo, identity, and date, but the aggregate table produced by `tools/build_human_db.py`, whose `SCHEMA_VERSION = "2"`, retains neither actor Elo, opponent Elo, nor player identity. It can learn only “the average move across everyone.” The new database is HumanDB v3. It retains actor and opponent Elo, hashes of both identities, date, source, whether the actor is human or AI, each move's Phase pair and candidate set, Malom W/D/L before and after the move, and query status; it is split by player and time.

`human_database/split_manifest.json` records 79,253 games—55,336 train and 23,917 test. The current rule is a seeded 70/30 random split by content SHA-256, and 41 cross-split near-duplicates have been found. This test set may serve only as a development control; it cannot support player extrapolation, temporal extrapolation, or a final claim about humans. v3 must be rebuilt from the raw JSONL rather than reverse-engineered from an aggregate table that has already discarded fields. It must also distinguish observational labels from candidate labels. The action actually selected by the human and its continuation are observations; the outcome of an unselected action is missing, not a loss. Without platform-recorded action propensities, ordinary game records may train and calibrate only HumanPolicy. Counterfactual estimation requires separately collected logs with known safe-set randomisation probabilities, or an explicit statement of its assumptions.

### P1: Phase Partitioning Is Too Coarse

The current specialists are limited to opening, midgame, and endgame; endgame begins whenever either side has no more than five pieces, and the features record only whether the actor can fly. The future design encodes all nine `{place, move, fly} × {place, move, fly}` Phase pairs. The rules variant supplies the `reachable` mask—under standard alternating placement, the final round can at least produce `place/move`, and a capture can produce `place/fly`. Add event flags for “placement about to end,” “just entered movement,” and “this action captures the opponent from four pieces to three.”

### P1: Opening Training Is Overly Fixed

The current opening V2 trainer uses a 100% probability of a fixed opening prefix; most other scripts use 50% and sample uniformly from fixed lines. Standard future training forces only one prefix before releasing both players. Forcing an entire line is reserved for targeted curriculum and regression tests.

### P1: The Current GapNet Semantics Are Insufficiently Precise

When HumanDB is sparse, the existing data pipeline uses “best candidate score minus the average score of all candidates” as the target—the `gap = best_q - avg_q` fallback in `scripts/build_gap_dataset.py`. The model predicts an ambiguous opportunity gap that is then added as a leaf bonus at inference. Retain the name GapNet, but redefine its semantics as specified in §10.

### P1: Silent Fallbacks Conceal Training Errors

Current opening loading, phase features, and specialist routing contain several instances of `except: pass`, neutral-value returns, or substitution of the wrong phase specialist. Future implementations use assertions or explicit exceptions for inputs, model dimensions, database versions, and illegal game records. Malom W/D/L must be non-null for a standard legal position. Database corruption, an unmounted file, or a missing sector must not degrade silently. An opening specialist may not substitute for a missing endgame specialist.

Its central assumption—that SpecialistDB will eventually replace Malom—conflicts with the local positional-tablebase assets and mandatory safety gate. ## 6. RuleAwareSafetyGate and the “Constrictor Metrics”

### 6.1 Decision State and Atomic Actions

The formal interface does not accept a board-only FEN. It accepts:

$$
S=(\text{board},\text{side to move},\text{in-hand counts},\text{variant},\text{position multiplicities},\text{no-progress counter},\text{claim state})
$$

A candidate $a$ must be a complete atomic action: a placement or movement plus any mandatory capture after forming a Mill. Malom is queried only on the settled state after the entire action is complete and the side to move has changed. An intermediate “Mill formed, capture pending” state must not enter the tablebase, cache, or training set. The canonical key returns both the move transform and its inverse, and the history key uses the same canonical orientation.

Sector routing is derived only from the settled board, both sides' in-hand/on-board counts, and the matched variant **after capture and after the side changes**. `sector_value` must come from that destination sector. Even when a capture causes a 4→3/flying transition, the end of placement, a colour change, and a symmetry redirect simultaneously, the sole permitted order remains: apply full action → validate counts/phase → derive target sector → raw decode/correction → reference perspective.

### 6.2 Formal Definition of Positional Value and the Safe Set

The Malom adapter returns a complete `OracleValue` containing at least `raw_key1`, `sector_value`, `absolute_key1`, `key2`, entry kind, sector, original perspective, and query status. Coarse W/D/L is only a projection of this value. Candidate values must apply side-to-move conversion and the comparator equivalently to the Sanmill reference `move_value`. Let $M(S,a)$ be the complete candidate value from the current actor's perspective, let $V_{pos}(S)=\max_{a\in A(S)}M(S,a)$, and let `wdl(·)` be the coarse projection. The positional safe set is:

$$
\mathcal A_{pos}(S)=\{a\in A(S):\operatorname{wdl}(M(S,a))=\operatorname{wdl}(V_{pos}(S))\}.
$$

Provided action enumeration and the comparator are correct, if $V_{pos}(S)$ is W or D, $\mathcal A_{pos}(S)$ is non-empty by definition. “No action preserves W/D” indicates a missing action, perspective error, side-to-move error, sector-correction error, or query failure. In W, retain only W actions and then apply the conversion/liveness gate in §6.3. In D, retain only D actions. In L, apply `horizon-safe(X)` first, then use the complete Malom ordering, rule-based opportunities to rescue a draw, and validated human-error probability to rank swindles; this must not be described as “theoretically drawable.” If $A(S)$ is non-empty but all oracle candidate values are missing, the safe set is empty, or the actor appears to obtain a minimax-violating “upgrade” such as D→W or L→D through its own move, abort and record an adapter error. Every cached value carries oracle/schema/variant hashes and perspective.

There are two levels of safe state:

- `theory-safe`: the action preserves the current full-rules W/D value or, if the state is already L, preserves the best L value attainable under the oracle. This is the default tier.
- `horizon-safe(X)`: define $D_{survive}(S)=\max_{\pi_{self}}\min_{\pi_{opp}}T_{loss}$, and retain only actions for which history-aware adversarial search proves $D_{survive}\ge X$ ply. The result may still be L, but even the strongest opponent cannot end the game within $X$ ply. If no action reaches $X$, select an action maximising $D_{survive}$ and record `survival_shortfall=X-D_survive`. $X$ is measured in ply, may be configured by Phase, and is locked before final acceptance. Raw `key2` must not stand in for $D_{survive}$ without proof from the reference semantics.

`horizon-safe(X)` automatically becomes the primary set only when the current full-rules value is already L. `bounded-risk`, which permits an intentional downgrade from W/D to L, is a separate product mode with an independent configuration, UI warning, and report. It may share with certified mode only the rules/oracle code, immutable raw-state schema, and read-only deterministic feature functions. It must not share any trainable encoder, normalisation statistics, optimizer, checkpoint, calibrator, replay buffer, or target-label snapshot. Online selection switches between distinct model-bundle/process namespaces. There is no permitted path that “shares first and separates later.”

### 6.3 Historical-Rules Gate and Proof of W Conversion

The positional tablebase contains neither repetition nor no-progress history, so `A_pos` is not a full-rules safety proof. `RuleAwareSafetyGate` must also:

1. Apply immediate terminal, repetition, claim, and counter transitions from the complete `RuleState` first. A candidate that would immediately trigger, or permit an immediate claim of, a rules draw must not be treated as converted merely because it retains positional W.
2. Beginning **at the first entry into positional W**, construct an on-demand, cacheable augmented AND/OR graph. Node keys contain complete `DecisionState/RuleState`; edges are atomic actions; terminal nodes are rules terminals. The same board with different multiplicity or no-progress history represents different nodes.
3. On this finite historical-state graph, compute a reachability-game attractor or retrograde proof. A terminal win has rank 0. At an AI node, only actions to lower-ranked states may belong to the winning strategy. An opponent node enters the attractor only if **every** legal action—including an action that claims a draw—leads to a lower rank. The rank comes from monotone fixed-point iteration, not raw `key2`. Proof rank strictly decreases along every realised ply, although the Malom comparator of the positional board need not. A proof may traverse several positionally equivalent boards or even revisit the same board, provided their augmented histories differ and the fixed point proves a terminal win before the claim/no-progress threshold.
4. Use the complete `PerfectPlayer::move_value`/`gui_eval_elem2` only to preserve positional W, order candidates, and determine search order.
5. Retain in `A_allow^W` only winning-strategy actions proved by the same proof artifact. A state labelled certified W with an empty set indicates a proof, cache, history, or action-enumeration error and causes `fatal_stop`. An insufficient search budget yields `proof_unknown`: only positional W may be reported, not a full-rules forced win.
6. Within `A_allow^W`, W-mode `q_ref` first orders by ascending successor proof rank. A successor that is an immediate terminal win has rank 0 and must be preferred whenever available. Only equal-rank actions use the complete Malom comparator and the frozen tie policy.
7. Validate with small-domain enumeration, an independent solver, and property tests: the attractor's AND/OR conditions, proof rank, every opponent reply, repetition/no-progress/capture resets, the same board under different histories, necessary positionally equivalent detours, preference for an immediate win, and absence of any rules draw under the selected strategy.

W liveness is therefore a standing augmented-state reachability proof for every W state in RuleAwareSafetyGate. It is neither a heuristic activated only near a draw nor a greedy gate requiring the key to decrease on every move. Strong full-rules guarantees apply only to the state domain covered by a history-aware proof; elsewhere, only a positional guarantee may be reported.

An “overly narrow gate” must be diagnosable. Each decision separately reports `|A_legal|`, `|A_pos|`, and `|A_allow|`, with per-candidate reasons for `deny_exact`, `proof_unknown`, and `unsupported`. Within enumerable domains covered by an independent reference, the count of genuinely safe actions excluded in error must be zero. `proof_unknown` means “not proved within budget”; it neither masquerades as genuine danger nor permits target specialisation. A state with `|A_allow|=1` is a forced safe-execution sample and is excluded from the effective denominator for learnable error-inducing ranking.

### 6.4 Directly Computable Safe Width and Safe Mass

`safe_width(S)=|A_pos(S)|` is a diagnostic quantity computed directly by the oracle. For a state $S_h$ in which the target human is to move, define the one-step human-weighted safe mass as:

$$
\widehat m_h(S_h,e)=\sum_{b\in A(S_h)}\widehat\pi_h(b\mid S_h,e)\,\mathbf 1[b\in A_{pos}(S_h)],
$$

where $e$ denotes the Elo/platform/time conditions. This is a **HumanPolicy estimate** and must be reported together with an ensemble interval, calibration slices, effective sample size, and OOD/abstention status. It is not Malom ground truth. If the opponent has five theoretically safe moves but virtually all probability lies on the one natural reply, `safe_width` is large while `safe_mass` may still be close to 1. `squeeze_delta` describes only the change in constraints across a candidate—the opponent's safe mass falls, the strongest safe branch does not harm the AI, and the AI's own gate-intervention risk does not rise. It is a teacher feature and diagnostic proxy, not automatically a win rate or a causal measure of inducement.

### 6.5 A Valid Practical Trap

A candidate enters evaluation as an “inducement” only if all of the following hold:

1. It passes positional W/D/L; if the state is W, it must also be an action permitted by the augmented-history proof in §6.3.
2. The opponent has at least one theoretically correct reply, so the position is not a forced win disguised as a trap.
3. A frozen, calibrated target HumanPolicy assigns non-zero probability to an incorrect reply, and the state does not trigger OOD abstention.
4. Under multi-branch rollout, the incorrect reply genuinely causes the opponent's first W/D/L downgrade. An observed game validates only the branch that actually occurred.
5. Against the strongest safe reply, the AI still preserves the original rules value and conversion requirements.
6. A paired experiment against the Malom ultra-strong, ordinary W/D/L-safe, and AB baselines supplies the preregistered confidence evidence.

Defence must not depend on an opponent error. Offence may, provided safety is preserved, make it harder for the target opponent to answer correctly forever.

### 6.6 Boundaries Between a Discrete Tablebase and Learnability

Malom labels are deterministic and discrete, and tactical boundaries may be highly non-local. A finite network cannot be presumed to reproduce the entire tablebase smoothly and exactly. Nor does this plan define success as “distilling away the 83.6 GB tablebase”:

- During label generation, training, and acceptance, the Malom reference remains online and no neural network has theoretical adjudication authority. A distilled compact release artifact has no such authority either; its safety level can be disclosed only through the offline measurements in §15.4.
- Once the support domain is matched and decoding is correct, Malom labels are not stochastic, “high-noise rewards.” If field-level disagreement with the reference appears, stop and diagnose the adapter, perspective, atomic capture, sector, label leakage, or misuse as reward. Preserve genuine same-tier candidates as ties, then let a qualified teacher compare them.
- The safety/WDL auxiliary covers all legal candidates and serves representation learning, shadow diagnostics, and anomaly detection only. Address class imbalance through stratified sampling or loss correction, but its prediction never replaces a query on the oracle workbench.
- Stage 2 StrategyPolicy distils only the teacher distribution inside `A_allow`, a far narrower learning problem than reproducing the complete tablebase. Stage 3 then uses targets over every legal action to constrain denied logits explicitly; the Stage 2 model cannot be used directly as the release policy.
- Plot learning curves stratified by unseen complete games, canonical families, sectors, Phases, and value boundaries, with particular attention to unsafe recall and boundary calibration. High accuracy under a random position split is not evidence.
- If an auxiliary head fails to improve with more data on held-out sectors or boundaries, reduce or remove its shared weight. RuleAwareSafetyGate remains fully functional.

### 6.7 The Two Runtime Environments and the Compact Release Form

The oracle workbench is the only environment that mounts the complete tablebase. It performs all label generation, training, acceptance, and research games. It may remain available as a local analysis mode with the database mounted, but it is not a release form. The compact release product is the only release form. Its bundle contains no Malom sector other than an optional `endgame_exact_pack`, no HumanDB, no rollout table, and no proof cache.

`X_rt` may be a global constant or a Phase-pair/transition-stratified mapping $X_{rt}(p)$ frozen before the first compact gate. No cell may be below the product owner's signed global floor $X_{rt,min}$. Each root decision selects its effective value once from deployable Phase fields and retains it throughout the proof. It must not decrease dynamically in response to load, model confidence, game result, or search progress. Runtime logs and the UI record both the effective value and the global floor.

The runtime prover first attempts fast certificates proved admissible in P0-B1, then uses cached or incremental AND/OR search with complete branch expansion. A fast certificate may use only rule-provable lower bounds—for example, terminal-trigger conditions, the maximum number of pieces the opponent can capture per turn, on-board and in-hand piece counts, or lower bounds from mobility blockage. Every concrete formula first requires a lemma for the matched variant, exhaustive boundary testing, and differential validation by an independent verifier. Every authorisation record stores the proof tier, premises, $X_{rt}$, covered branches, and verifier hash. The safety prior determines proof order only.

**Compact runtime protocol**, on every move:

1. The rules engine enumerates every legal atomic candidate, including any mandatory capture. It is the sole online source of deterministic truth at runtime.
2. **`endgame_exact_pack` path**: enable exact coverage only if both the current state and every candidate-successor sector lie in the pack. If any successor lies outside it, the entire decision does not use pack values; no partially exact mixture is permitted. In W or D, reconstruct from the complete comparator the mode-relative set that preserves positional W or D, respectively. The rules engine first removes avoidable W successors that would immediately trigger or permit an immediate claim of a repetition/no-progress draw; within the same value tier, prefer actions with greater remaining historical margin. If historical filtering empties the set, the pack grants no authorisation and processing proceeds to Step 3. In L, the pack identifies only the best positional L tier; it may schedule proofs and break equal-bound best-effort ties, but grants no safety authorisation. Pack W is not full-history W liveness, and pack L is not a finite-horizon survival guarantee. Every L state continues to Step 3.
3. **Finite-horizon survival-proof path**: when the pack does not hit completely, or when the current state is L, run an adversarial proof search depending only on the rules engine and frozen certificates. Only an action for which all required strongest-opponent branches have been fully covered and $D_{survive}(S,a)\ge X_{rt}$ has been proved enters `A_runtime_proved`. “No counterexample found yet,” partial expansion, predicted probability, and timeout are not proofs. When the proved pool is empty, continue until at least one action is proved, every candidate is refuted, or the deadline arrives. Once the proved pool is non-empty, continue by default until every candidate is decided or the deadline arrives, but an early stop under a `proof_pool_early_stop_rule` frozen before confirmation is allowed. That rule reads deployable fields only and, at minimum, must require the number of proved actions to reach `min_proved_actions`, the proved pool to cover at least `rho_proved_mass` of the compact policy's raw pre-pool mass, and no unresolved action to exceed `tau_unresolved_action` in raw mass; its thresholds may not be relaxed dynamically under load. If unresolved candidates remain at early stop or deadline, select only within the proved pool and mark `proof_pool_incomplete`. Record the stop reason; counts of legal, proved, refuted, and unresolved actions; and the compact policy's raw mass in all four classes. A censored pool must not be represented as a complete policy choice.
4. **Failure routing**: whenever the proof path applies and `A_runtime_proved` is non-empty, select only from it. If every legal candidate has been fully proved to fall below $X_{rt}$, record raw `runtime_survival_shortfall`. It grants no safety authorisation; a best-effort continuation maximising the proved lower bound may be selected only if the pre-signed product rule explicitly permits it. If the proved pool is empty and candidates remain unresolved at the deadline, record `runtime_bound_unavailable`. The default product stops the game instance and adjudicates it under the pre-signed rule; an unresolved candidate must not be renamed “best survival.” The first case means “the active threshold is proved to admit no action”; the second means “unknown within budget.” Both enter safety-availability accounting, but the match ITT endpoint receives only one actual terminal or pre-signed adjudicated result; no additional synthetic loss is stacked onto it.
5. **Unified compact policy**: on a W/D pack hit, distribute and select within the mode-relative positional set; otherwise, do so within `A_runtime_proved`. The safety prior is used only for search order, secondary logit suppression, and OOD warnings; it cannot enlarge the authorised set. Replay legality and the authorisation-source assertion after selection. Write events to a read-only log; do not learn or alter thresholds online.

**`endgame_exact_pack`**: measure sector sizes and choose read-only sectors within the signed budget by target visit mass, complete decision closure, and benefit per byte. Retain “fewest pieces first” as a mandatory simple baseline. Store a hash per sector. In-pack query semantics—including absolute-key correction and the complete comparator—must agree field by field with the oracle workbench and pass the same P0-A differential gold set. Coverage depth is determined by measured size and closure constraints; it is not assumed in advance.

**Disclosure semantics**: positional non-regression for the compact product is measured in Stage 3 by large-scale shadow replay of the frozen bundle on the oracle workbench—reporting the theoretical downgrade rate and its confidence upper bound, post-policy unsafe selection, W conversion inside and outside the pack, and L survival. Reports and product copy use `offline-verified`; the word `certified` is reserved for the oracle workbench. The only per-move runtime guarantees are the logged W/D `pack_exact_positional` or `runtime-loss-bound(X_rt)`. `runtime_survival_shortfall`, `runtime_bound_unavailable`, and `proof_pool_incomplete` are not a third kind of guarantee. On every state actually visited through both the pack and proof paths, an independent verifier must determine whether an action satisfying the effective $X_{rt}$ exists, so that a pack hit cannot conceal entry into an unsalvageable tail. An aggregate pack average cannot vouch for an individual move. The compact horizon-bounded fallback is an independent release semantic; it does not enable the oracle `bounded-risk` training mode in §6.2, which actively optimises W/D→L downgrades.

## 7. HumanDB v3 and Data Splitting

### 7.1 Data Sources and Current State

The frozen snapshot in `human_database/split_manifest.json` records 79,253 valid deduplicated games and 3,780,124 actual decisions. The current schema v2 uses content hashes to split them randomly into 55,336/23,917 games and contains 41 cross-split near-duplicates. Statistical independence is far lower than the number of decision rows suggests: there are only 4,420 players; the median player has three games; and the 100 most active players account for 62.38% of player-game records. Of the 2,805 players in the current test set, 2,298—or 81.93%—also appear in train. The dates span only December 2025 through July 2026. The 1,600–1,799 Elo band has only 260 participant-game records, while flying accounts for approximately 108,607 decisions. These values are preflight baselines from the current manifest only; the v3 builder must recompute and reconcile every item.

The aggregate volume is therefore sufficient to attempt a basic pooled HumanPolicy whose target is observed choice, but it is insufficient to presume reliable support in every narrow Elo × Phase × opening/structure × history cell. Although the current train SQLite aggregates 1,392,374 state rows, 1,605,927 `(state,move)` rows, and 2,639,346 observations, the non-null count is zero for both `positions.malom_wdl` and `moves.malom_wdl_after`; it contains no label directly usable for safety or the teacher. v3 is rebuilt from raw JSONL and a fixed source-archive hash. v2 remains read-only as a development baseline.

### 7.2 Strict Separation of Three Record Types

1. `games`: `game_id`, raw hash, source, time, anonymised hashes and Elo for both players, colours, rules variant, terminal result, and terminal reason.
2. `observed_decisions`: the complete `DecisionState` actually reached, actor/opponent conditions, legal choice set, the **action actually chosen**, Phase pair, opening family, and pointer to the actual continuation.
3. `candidate_oracle`: for every complete atomic action in that choice set, the Malom raw/absolute values, complete comparator key, W/D/L, rule transition, query status, and optional AB features.

The critical missingness constraint is that only `observed_decisions.chosen_action` has a human-choice label, and only the actual trajectory containing that action has an observed continuation. The schema uses `outcome_source ∈ {observed, oracle, rollout, search, missing}`; a missing value must not be filled with `0`, draw, or the shared actual terminal result. Each state also stores the canonical key and forward/inverse transforms, position multiplicity, no-progress counter, in-hand/on-board counts, both sides' Phases, from/to/capture, capture-to-fly, and data/oracle/feature-schema hashes. A capture after forming a Mill is part of the action; the intermediate removal state is not persisted.

### 7.3 Unit of Supervision for HumanPolicy

HumanPolicy uses the complete choice set of one human decision in a masked categorical NLL; aggregate frequencies are not decomposed into independent binary examples. Continuous Elo and platform/time conditions are retained as recorded, while quantile bins are used only for sampling and reporting. HumanPolicy conditions only on variables genuinely present in the raw record and obtainable under the same semantics at deployment. Missing values are explicitly `missing/not_observed`; “pressure” or “familiarity” must not be inferred from the terminal result or future actions. The model represents only “the behavioural distribution conditional on observed context”; it does not explain why a human moved that way.

To prevent highly active players and very long games from dominating the loss, training weights are normalised first by player and game, then sampled within preregistered Elo/Phase/state-source strata. AI actions, imported test games, duplicate captures, and decisions for which rule state cannot be reconstructed do not enter the human-behaviour target. Only out-of-fold HumanPolicy predictions may construct teacher labels for the same training fold. Ordinary historical game records have no known logging propensity and support behavioural prediction only. IPS/CRM/DR is used solely on newly collected logs that explicitly record safe-set randomisation probabilities and satisfy overlap.

### 7.4 Deduplication and Two Independent Generalisation Protocols

Perform exact, symmetry-canonical, and near-duplicate clustering at the complete-game level before splitting. Every symmetry copy, branch of the same manually constructed scenario, and member of the same near-duplicate cluster must remain on one side.

`near_duplicate.version=trace-lcs-v1` supplies the sole reproducible definition. Within a matched variant, represent each game as a sequence of complete `DecisionState + atomic action` records. Apply D4 and any legal colour/actor exchange as a **single game-wide transform**, and choose the lexicographically minimal globally transformed sequence as the canonical trace. Identical hashes are exact duplicates. Near-duplicate detection does not rely on a MinHash approximation. For sequence lengths $n,m$, longest common subsequence $L$, and longest common prefix $P$, add an edge within the same variant when `min(n,m)/max(n,m) ≥ 0.80` and $2L/(n+m)\ge0.90$. A prefix-only edge with $P/\min(n,m)\ge0.90$ is allowed only when source lineage, an abnormal-termination/missing-tail flag, or evidence from the raw archive confirms that the shorter record may be a truncation or duplicate export. Shingles and MinHash serve only as candidate blocking; stratified all-pairs audits estimate missed pairs. Before the split, freeze the default threshold of 0.90 after a 0.85/0.90/0.95 sensitivity analysis on development-only data. Retain one traceable record per exact duplicate. Assign every connected component of near-duplicates as a unit to one split and normalise its training weight by component. The manifest stores algorithm, threshold, hashes, global transform, component ID, edge reason and evidence, and threshold sensitivity.

Before membership is frozen, connected components must pass `near_duplicate_component_gate`. Report component-size and weight quantiles; the largest component's fraction of games, decisions, unique players, and each target stratum; graph diameter and longest single-link chain; composition by edge reason; articulation/bridge records; and threshold sensitivity. Planning defaults flag `giant_component_review` if any component exceeds 1% of total game mass or 5% of unique-player mass. The split cannot be frozen until the audit is closed. Permitted remedies are: correct an erroneous edge rule when supported by evidence and open a new version; deduplicate confirmed duplicates; remove a preregistered minimum bridge/vertex-cut quarantine set from **all** model splits and recompute; or place the complete component on one side and accept the resulting supply conclusion. It is prohibited to break transitive closure by force and place records that still share a near edge on opposite sides.

The two generalisation protocols are independent and cannot substitute for one another:

- **Temporal extrapolation**: train → validation → future test across a time boundary. Test reflects future traffic and may include known players, but it never feeds back into tuning. Temporal extrapolation must also satisfy preregistered calendar-coverage and independent-player/event gates. For the highest conclusion level, the planning default requires a prospective window of at least 90 days or one complete product/rules-version cycle, whichever is longer. The current raw data covers only about eight months; a conclusion that misses the calendar gate is labelled `player_disjoint_short_horizon` or `temporal_scope_limited`.
- **Player extrapolation**: first select a completely unseen set of players. Every complete game in which either side is a holdout player is excluded from train and validation. Split the player-game graph and report the amount of data removed to preserve isolation.

Run `player_split_feasibility_screen` before final membership is frozen. Once deduplication clusters and the player-game graph have been built, but before any model is trained, enumerate or constraint-solve a preregistered set of candidate holdout ratios, time boundaries, and four-layer assignments. For each candidate, report retained games, decisions, independent players, player-cluster ESS, maximum player weight, and target supply for train/cross-fit/selection; and independent players/events, calendar coverage, and expected interval width for confirmation/final. Lower bounds defining the feasible region are frozen before the first screen and cannot be inferred backwards from model results. If no solution exists, output `not_feasible_from_current_humandb`; do not freeze a formally complete split that has no effective power on one side. Choose among feasible rules using supply/power and a preregistered tie-break only. Section 7.6 recomputes the screen once on the materialised split.

For each protocol, freeze `train → selection_dev → confirmation_dev → final_test` by complete game and player cluster outside test. Grouped K-fold produces out-of-fold teacher inputs only within train. `curriculum_dev` is an explicit subset of `selection_dev`. Architecture, features, temperature, loss, curriculum rule, checkpoint, and DAgger-round selection may inspect only `selection_dev`; the plan locks the search space, maximum run and seed count, and primary selection metric. For each requirement/data version, only one winning recipe may access `confirmation_dev`, once. Failure ends that branch; after reading the result, it is prohibited to return to selection, retune, and reuse the same confirmation set. `final_test` runs once after every method and threshold is frozen. Four-layer memberships and hashes, together with the access ledger, enter contamination scanning. Failure of the builder, split isolation, or leakage gate makes later learning Stages a `stage_block`. Insufficient target support closes only human-target specialisation under §7.6; it does not block safe reference work.

$$
\operatorname{MDE}^{optimistic}_g(n_{max})=
\left(z_{1-\alpha_g/2}+z_{1-\beta}\right)
\sqrt{\widehat{\operatorname{Var}}_g(\widehat{\Delta Q})/n_{max}}+b_{HP,min,g}.
$$

## 9. Role of Alpha-Beta in Training

### 9.1 Adjustable-Strength Sparring Partners

One deepest search trains only “hold a draw against perfect defence”; one overly shallow search produces unrealistic elementary errors. Define search profiles rather than passing a single difficulty value:

| Profile | Initial search range | Behavioural use |
| --- | --- | --- |
| AB-low | 3–4 ply, 1–2 ply of tactical extension, higher temperature | Emulate beginners who see immediate threats but often miss consequences |
| AB-mid | 5–7 ply, 2 ply of tactical extension, medium temperature | Emulate ordinary experienced players |
| AB-strong | 8–10 ply, 2–4 ply of tactical extension, low temperature | Emulate stronger human players |
| AB-deep | 11–14 ply or an explicit node/time budget, deterministic | Emulate strong search, generate principal variations and shallow/deep counterexamples; theoretical adjudication remains with Malom |

These numbers are starting points for the first experiment only. A profile ID locks the evaluation function, candidate ordering, transposition-table semantics, extension and repetition rules, node/time budget, temperature, and random stream; depth is merely one field. A target-Elo label is permitted only after calibration of the choice distribution and error types on held-out HumanDB. Otherwise, these are simply distinct search profiles.

### 9.2 Feasibility Boundaries for Depth Scheduling and Tactical Extension

- The game tree during early placement is very wide. AB-low/mid may use shallower budgets as **human proxies still requiring calibration**. Their leaf evaluation uses a minimal baseline and is validated by fitting HumanDB, Malom regret, and feature ablation. AB-deep and FrozenTeacher do not deliberately reduce depth to appear human.
- HumanPolicy or historical frequency may order moves, prefetch ponder states, and allocate `T_full` budget, but may not delete low-frequency replies from rule or positional safety proofs, W liveness, `horizon-safe(X)`, or the reference-best-reply branch. Frequency-gated pruning preserves the omitted probability mass and value bounds. If the unexplored tail could reverse the candidate ordering, the only valid result is tie or abstention.
- Do not apply a mechanical uniform `depth += 1` at phase boundaries. Boundary candidates use a transition-complete extension: within a separate node/time limit, search at least through the relevant side's first settled decision state after the Phase transition. If the boundary is not crossed, record `transition_not_reached`.
- A capture after a Mill belongs to the same atomic action; search cannot stop legally at “Mill formed, capture pending.” Tactical extensions carry the search beyond an immediate Mill/capture by the next side at a leaf, a unique block, double threat, 4→3 transition, or forced sequence.
- Ordinary chess-style quiescence uses stand pat, which assumes a lower bound from the option not to move. In Mill, where there is no pass and zugzwang is possible, that assumption may be invalid. The default is bounded tactical search with an extension budget, repetition detection, and complete legal actions. Enable stand-pat pruning only after property tests prove it valid for a specified state class.
- Branching is wide at 4→3 and in flying. Malom adjudicates positional value; AB supplies short-term ordering or a human proxy only. Extensions obey node, time, and extension budgets and record realised depth and truncation reason.
- “Triggers frequently” is not evidence that an extension works. Before it enters training labels, compare no extension, each extension, deeper search, and the reference on a fixed boundary/forced-sequence suite, reporting top-k, regret, misordering rate, and cost. Remove rules with no stable net gain under equal budget. `ai/game_ai.py` already contains an opening depth ramp, SE-8 extension, and `_qsearch`, but existing tests establish only interface behaviour and determinism, not reduced horizon error.

### 9.3 Mix Depths Even Within One Elo Band

For each game, sample from calibrated profiles: 55% from the central target-Elo profile, 20% one tier lower, 20% one tier higher, and 5% from a time-pressure profile. Time pressure does not mean choosing poor moves at random. It shortens the budget, raises the root-candidate softmax temperature, and retains basic recognition of an immediate one-move threat.

### 9.4 Alpha-Beta Actions Must Be Probabilistic

Convert root candidate scores into an Elo-calibrated probability distribution. When score gaps are small, permit several natural choices; when gaps are large, increase the probability of the strong move sharply. A human-familiarity prior may raise familiar moves. The probability of an immediate losing blunder is determined from actual data for the target Elo. Calibration metrics include HumanDB top-1/top-3 accuracy, negative log-likelihood, and theoretical error rate and severity by Phase.

### 9.5 Generate Training Counterexamples

Run shallow, medium, and deep search on important states simultaneously. The following disagreements are especially valuable: shallow search prefers an immediate Mill that deep search or Malom shows to be losing; shallow search captures the fourth piece while deep search sees that flying is released; shallow search dismisses a waiting move that deep search finds compels self-destruction; shallow and deep best moves differ but are both theoretically safe, with one more likely to induce an error; and positions where deep search is stable but shallow search disagrees widely, which are often good traps. Record `depth_disagreement`, rank changes, score variation, principal-variation stability, and top-k entropy for each profile as search evidence for FrozenTeacher and candidate inputs for an optional amortiser.

## 10. Multi-step First-Error Teacher, GapNet, and SelfRisk

### 10.1 First Separate One-Step from Multi-Step Quantities

GapNet is unnecessary for the opponent's next-action first-error probability. Use cross-fitted HumanPolicy to assign a probability to each legal reply, use Malom to determine whether the reply is below the opponent's current optimal W/D/L, and sum directly to obtain the one-step unsafe mass in §6.4.

“Will a first error occur within several future decisions?” depends on both continuation policies and must be expressed as a horizon-indexed survival curve:

$$
P(T\le h\mid S,a,e,\pi_h^{(v)},\pi_{self}^{(u)},\text{variant}),\qquad h\in\mathcal H_{eval},
$$

where $e$ is the target-human condition and $u/v$ are frozen policy versions. Gap/first-error time is measured in **future opponent decisions**; SelfRisk time is measured in the **AI's own future decisions**. The two must never be conflated. Training and teacher computation use preregistered state-conditional limits $H_{opp,max}(S)$ and $H_{self,max}(S)$, which may vary with Phase pair, distance to the end of placement, 4→3, remaining rule-counter margin, forced sequences, and branch budget. The scheduling function reads only fields declared and versioned before freezing. Frozen acceptance reports survival/hazard, time-dependent Brier/log loss, and restricted mean time-to-event on a common preregistered actor-decision grid $\mathcal H_{eval}$. Computational truncation is right-censored and retains residual bounds. Pilot event counts and coverage determine the grid; it is entirely distinct from $X$ in §6.

State-conditional scheduling must not systematically assign shorter observation windows to more complex candidates. Within each preregistered Phase/W/D/L/branch stratum, `censoring_complexity_gate` uses train/planning data to regress computational censoring, effective horizon, and residual width on log branch factor, proof p95, forced-sequence length, and history depth, reporting standardised slopes and the absolute censoring-rate difference between the lowest and highest complexity deciles. Planning defaults require absolute standardised slope ≤ 0.10 and absolute decile difference ≤ 5 percentage points. Adjudication does not use point estimates. The contract freezes `alpha_censor`, default 0.05; a stratified cluster bootstrap whose minimum resampling unit is `trajectory_id/branch-tree`; simultaneous max-stat correction; and numerical tolerance. Compute a one-sided simultaneous UCB for each statistic. Pass only if both UCBs remain within their gates. Fail if either lower confidence bound exceeds its gate. An interval crossing a gate is `inconclusive` and, under fail-closed handling, marks the stratum `horizon_limited`. First equalise that stratum's horizon/budget or increase residual/proof budget, then confirm once on data not used for the adjustment. If the gate still fails, the entire stratum is excluded from target-specialised labels, cross-state mode regret, curriculum $e_g$, and comparable proxy aggregation.

### 10.2 Discrete-Time Hazard and Censoring

`opponent_event_schema.version=certw-aligned-v2` does not collapse every “positional first error” into one event. Let $T_{pos}$ be the target actor's decision index at which it first chooses an action below its then-current $V_{pos}$. It is a diagnostic hazard reportable across modes:

$$
h^{pos}_k=P(T_{pos}=k\mid T_{pos}\ge k,S,a,e,\pi_h^{(v)},\pi_{self}^{(u)}),\qquad
P(T_{pos}\le h)=1-\prod_{k=1}^{h}(1-h^{pos}_k).
$$

For each positional first error, inspect the complete post-atomic-action state from the AI's perspective and record `error_class ∈ {certified_exploitable, proof_unknown_candidate, positional_only}`. The first class applies if and only if the state enters precisely the same certified W as in §13.2. The second has reached positional W for the AI but its full-history W-liveness proof remains unresolved. All other cases are `positional_only`. Severity separately records the actor's W→D, W→L, or D→L. D-mode primary utility defines a separate $T_{certW}$: the target actor's future decision index at which an action first enters certified W. An earlier `positional_only` event does not end this risk set. Probability mass for `proof_unknown_candidate` enters the same event bounds as the teacher rather than being recorded as zero. Each rollout record stores both `at_risk_pos[k]` and `at_risk_certw[k]`; the D-aligned output is:

$$
h^{certW}_k=P(T_{certW}=k\mid T_{certW}\ge k,S,a,e,\pi_h^{(v)},\pi_{self}^{(u)}),\qquad
P(T_{certW}\le h)=1-\prod_{k=1}^{h}(1-h^{certW}_k),
$$

with time-indexed probability mass and a partial-identification interval for `proof_unknown_candidate`. At trajectory termination, subsequent times are right-censored or marked as a competing terminal under the rules of each endpoint. The two at-risk masks are not interchangeable.

Report sources separately. **Observed**: HumanDB supplies event/censoring information only along the actual path; it cannot assign counterfactual labels to unchosen candidates. **Teacher rollout**: multi-branch rollout under a frozen out-of-fold HumanPolicy ensemble and frozen self continuation, retaining an additional reference/strongest-safe-reply branch and recording policy, seed, truncation, and oracle versions in the label. **Exact/search**: where a small domain is enumerable or budget permits, validate the rollout teacher. The three records for one candidate must not be merged into one “truth.” The teacher must pass calibration and ranking validation on actual paths and exact/search subsets before generating StrategyPolicy labels.

### 10.3 GapNet Is Only an Optional Amortiser

If the direct multi-branch teacher meets the online budget, GapNet is unnecessary. Only if profiling shows unacceptable latency may a candidate-level GapNet be trained to distil the frozen teacher. It outputs: actor-decision-indexed $T_{pos}$ diagnostics and teacher-aligned D-mode $T_{certW}$ hazard/survival, with cumulative quantities on the shared $\mathcal H_{eval}$ grid; error class, severity, and corresponding intervals; teacher one-step safe mass, rollout lower bound, and strongest-branch result; and uncertainty, OOD score, label source, and policy versions. Each endpoint uses its own at-risk mask and discrete survival NLL. Evaluation includes time-dependent Brier/log loss; calibration by Elo/Phase/source/error class; coverage-risk curves; and generalisation to held-out policy versions. D-mode target acquisition and any “error-induction rate” may read only the lower bound or interval for $T_{certW}$; $T_{pos}$ is diagnostic only. On OOD, version mismatch, or an excessively wide interval, abstain and return to the direct teacher/reference.

The default `risk_source_contract.version=rollout-distill-v1` prohibits concatenating the three sources into one survival NLL. The deployment opponent-error head trains only on frozen `teacher_rollout` labels. The SelfRisk head trains only on frozen shadow-rollout gate-event labels. HumanDB `observed` performs behavioural-consistency and transport validation only along actual paths. `exact/search` confirms teacher bias, ranking, and calibration only. Any training use of observed or exact data requires a new source-specific head. Before the first round, freeze each head's source, target event, at-risk/competing/censoring rules, policy version, sampler, and loss weight. Primary calibration and the release gate use rollout confirmation data that was not used for training and is aligned with the deployment policy/version.

### 10.4 SelfRisk Is Gate-Intervention Risk

On the oracle workbench, the final policy is always behind the gate, and the theoretical downgrade rate is controlled by gate invariants. SelfRisk does not predict “whether the deployed system will lose.” It measures how strongly the pre-mask raw policy depends on the gate. If $\pi_{raw}$ is normalised over all legal actions, the one-step intervention mass is directly computable:

$$
r_{gate}^{(1)}(S)=1-\sum_{a\in A_{allow}(S)}\pi_{raw}(a\mid S).
$$

To make this quantity identifiable, training adds a safety auxiliary over **all legal candidates**—oracle allow/deny BCE or listwise classification. StrategyPolicy distillation, ranking, and optional RL remain restricted to `A_allow`. Multi-step `gate_intervention_risk` is computed by shadow rollout under frozen $H_{self,max}(S)$: advance with the frozen raw policy and opponent policy, record an event and stop when the raw policy would choose an action outside the gate, and never actually execute that action. The risk is bound to strategy, safety-auxiliary, opponent, oracle, and horizon-schedule versions. One-step intervention mass diagnoses the dependence of the Stage 3 safety prior and compact policy on the exact gate only. Compact online authorisation still comes from the pack or proof; the mass must not be interpreted as a “neural interception guarantee.”

GapNet and SelfRisk need not each have a wholly separate network. They may share candidate enumeration, rollout, discrete survival loss, encoder implementation, horizon scheduling, calibration, and OOD infrastructure. If throughput genuinely improves, an experiment may use a `RiskSurvivalModel` with `task_id`, actor perspective, and policy-version embeddings. The supervision heads, label tables, at-risk masks, calibrators, and version manifests must nevertheless remain separate. The opponent-error head—compatibility name GapNet—predicts the diagnostic $T_{pos}$ endpoint and D-primary $T_{certW}$/`proof_unknown_candidate` interval on the frozen target-human decision clock. The gate-risk head—compatibility name SelfRiskNet—predicts the first required gate intervention by the frozen raw StrategyPolicy on its own decision clock. The two differ in actor, event, risk set, and update frequency; combining them in one output head destroys the denominator of the probability. Begin with small independent-head baselines. Share a backbone only if neither task regresses in its own calibration, AUPRC/coverage-risk, OOD, and held-out-policy-version metrics, and latency or memory improves materially.

### 10.5 Decision Authority

Within `A_allow`, FrozenTeacher prefers: a W candidate certified by the same augmented-history proof whose strongest branch is no worse; a D candidate with a high robust lower bound on $T_{certW}$; and a candidate whose own gate-intervention risk does not exceed the preregistered robustness baseline. $T_{pos}$ may explain the process only. GapNet/SelfRiskNet may approximate these quantities only inside their calibration domains. They cannot cross RuleAwareSafetyGate, replace the reference comparator or historical proof, or present a finite-horizon probability as a long-run win rate.

## 11. Specialised Training for Phases and Their Boundaries

### 11.1 Model Structure

One variant-aware rules function computes `self_phase` and `opp_phase`, encoding the nine `{place, move, fly} × {place, move, fly}` cells and `reachable`. Unreachable combinations retain stable IDs, but the sampler rejects and counts them; routing to the “nearest” specialist is prohibited. `place/move` in the final placement round and possible `place/fly` must have regression tests.

The default is one conditional model reading Phase-pair and transition fields. A shared encoder with Phase gating, or independent Phase heads, is only a budgeted candidate experiment on a fixed audit subset and enters full training only if both grouped validation and the boundary stress suite show no regression. Dedicated representation must express abrupt changes in rules value: empty points are placement resources during placement; adjacency determines mobility during movement; in flying, almost every empty point is reachable and old positional values change; opponent flying and self flying are entirely different; and `placement_remaining_self/opp`, just-ended placement, capture-to-fly, and distance to a rules-draw threshold cannot be inferred from coarse Phase alone.

The deployment interface returns one `StrategyBundle`: state/feature/action-schema hashes, an explicit mapping from reachable Phase pairs to models or heads, the rejection action for a missing route, and whole-bundle latency benchmarks. Phase routing may read only a Phase computed by the rules engine. The oracle workbench may separately map W/D/L heads; compact mode defaults to the unified Stage 3 deployable head and prohibits routing by predicted or oracle W/D/L. Routing is constant-time and has no “nearest specialist” fallback. Whole-bundle p50/p95/p99 is accepted under §§15 and 17. Old 122- and 134-dimensional checkpoints are rejected directly because their schemas differ. A Phase pair is a condition and sampling index, not a replacement for 24-node occupancy, in-hand/on-board counts, mobility, atomic capture, transitions, and rules history. The model must pass dependency diagnostics that retain the complete state while shuffling or removing the Phase ID.

### 11.2 Mandatory Oversampling Windows

The final four ply of placement; the first four ply of movement; four ply before and after either side's 4→3 transition; every reachable asymmetric Phase pair, especially `place/move`, `place/fly`, `move/fly`, and `fly/move`; 6v4, 7v4, 4v4, 4v3, and 3v3; same-state counterfactuals for capturing versus not capturing the fourth piece; repetition states; and no-progress counters near the draw threshold. Windows are defined relative to either side's first Phase-transition event. Each sample stores `plies_to_transition` or `plies_since_transition` and the corresponding actor. Oversampling raises training coverage; it does not imply that §9 search must deepen mechanically.

The planning-pilot starting point for the teacher/Strategy loader is 60% natural distribution, 20% Phase-pair balance, and 20% boundary and 4→3 specialisation. Select by validation, then lock. Natural test performs no Phase resampling. Log source, Phase pair, transition, and sample weight for every batch to prevent duplicate charging with opening buckets.

### 11.3 “Do Not Capture into Flying” Is Not an Absolute Rule

It is an important candidate feature, not a fixed prohibition. Compare on every occasion: capture the fourth piece immediately; retain four pieces and continue containment; capture a different piece; or defer forming/open the Mill. RuleAwareSafetyGate remains the boundary, followed by the complete reference value and the practical teacher. Some 7v4 arrangements can establish multiple threats and force a win before releasing flying; a simplistic rule must not discard them.

## 12. Training Methodology: Curriculum, Algorithms, and Data Flow

This section is the training execution specification. See §0 for the rationale, §8 for sample proportions, §13 for losses and attribution, and §15 for acceptance thresholds.

### 12.0 General Principles—Hard Methodological Constraints

1. **Distillation cannot replace the oracle**: during label generation, training, and acceptance, RuleAwareSafetyGate always queries a reference-compatible Malom implementation directly; no prediction head has authorisation authority. In compact mode, the candidate authorisation set is produced only under §6.7, from W/D exact-pack positional closure or a completed survival proof. An L-state pack value only orders candidates; the safety prior supplies search order and OOD signals only; offline measurement is not called certified.
2. **Oracle-mode training uses one semantics for the permitted set; compact mode models the deployment gap explicitly**: Stage 2 StrategyPolicy loss, sampling, and optional RL are all normalised over exact `A_allow`. Stage 3 separately maps frozen targets onto every legal action and distils a unified deployable policy. Do not pretend that the approximate runtime pool equals `A_allow`; `compact_exposure_gate` and §15.4 report their difference explicitly.
3. **Separate observation from counterfactuals**: HumanDB supervises observed actions only. An unchosen candidate may be labelled only by a versioned teacher, search, or rollout, and its source is never discarded.
4. **Freeze the teacher before training the student**: freeze HumanPolicy, oracle, opponent continuation, search budget, and teacher objective before generating a label version.
5. **HumanPolicy is not StrategyPolicy**: the former fits humans; the latter distils a safe-set teacher. The human mode is not a hard AI label.
6. **Separate tasks by default**: every task has its own loader, batch, update budget, and independent-model baseline. Share an encoder only after ablation demonstrates no negative transfer.
7. **Tie or abstain under uncertainty**: do not fabricate A ≻ B when teacher confidence intervals overlap. When an amortised model is OOD, return to the direct teacher/reference.
8. **Fixed examination material never enters training**: test, gold, and regression sets are read-only. Validation supports selection; once selection is complete, freeze it.
9. **One contractual source**: variant, $X$, $X_{rt}$, horizon scheduling, $\mathcal H_{eval}$, proportions, thresholds, bundle budgets, model/oracle/data hashes, and permitted fallbacks all reside in versioned configuration. Training and inference read the same contract.

#### Label Authority and Gold-Standard Contract

No single “universal gold standard” answers every task in this project. For every field, `label_authority_registry.yaml` declares semantics; unit, actor, and perspective; authority; support domain; generator and independent verifier; rule/oracle/data hashes; evidence level; permitted uses; and failure action:

| Label family | Authority and qualification | What it must not impersonate |
| --- | --- | --- |
| Legal actions, terminal states, repetition/no-progress | Executable rules specification for the locked variant; exact only after differential validation against an independent move generator, boundary-state-machine tests, and enumerable small domains | Historical rules cannot be inferred from game frequency or a Malom positional value. |
| Malom raw/full value, positional W/D/L/comparator | Fixed asset hash + complete C++ reference behaviour + independent comparison against Sanmill Rust raw decoding; exact only in the declared settled `std` support domain | Cannot be extended to an unsupported variant, mid-removal state, or full-history value. |
| `A_allow`, W liveness, `horizon-safe(X)` | Composition of the preceding two; exact/certified only for states and horizons covered by exhaustive or proof search, unknown outside the proof domain | Zero errors in a finite sample cannot become a global proof. |
| HumanDB chosen action/Elo/time | Observed fact after raw-record hashing, full-game replay, and deduplication | Not the best action, the outcome of an unchosen candidate, or truth about human cognition |
| HumanPolicy, FrozenTeacher, Gap/Self heads, conditional regret | Versioned estimates or silver labels, carrying intervals, source, policy/horizon versions, and abstention | Never a theoretical gold standard and never authorisation for the gate |
| Product effect on humans | Preregistered endpoint from a prospective randomised A/B test of frozen versions | Cannot be replaced by HumanPolicy/AB proxy games |

Gold tests have two layers. `conformance_dev` supports development diagnosis. `acceptance_gold` is held read-only by the verifier/CI; its payload never enters a training snapshot, and the repository stores only its signed manifest. Expected Malom values must be generated by the reference path, never by the adapter under test marking its own work. Rules are cross-covered by hand-authored boundary vectors, differential comparison with an independent implementation, and enumerable small domains. A hash change to any gold payload, test, or expected output creates a new major version and requires recertification. The training builder runs a contamination scan and stops immediately if a golden/test canonical key, same-game branch, or symmetric near-duplicate enters train. **Until Stage 0 has passed reference differential testing, the rule proof domain, asset integrity, and independently verified acceptance artifacts, this document contains only a plan for establishing gold standards; it is not entitled to claim that those standards are already reliable.**

#### White-Box Self-Checks and Failure Classification

White-box tests and fail-fast behaviour are not a “perfect guarantee”: tests may restate the implementation, the specification may be wrong, and two implementations may share a defect. The verification stack includes at least white-box unit and state-machine tests; property/metamorphic tests for colour exchange, D4, and side-to-move recurrence; exhaustive small domains; black-box C++/Rust/Python differential testing; verifier-owned acceptance gold; mutation and fault injection; and frozen end-to-end replay plus runtime shadow/canary. Test code, expected output, and the verifier itself are versioned and hashed.

Errors are classified by semantics. `training_contract.yaml` declares one action for each error code:

| Level | Examples | Mandatory action |
| --- | --- | --- |
| `fatal_stop` | Required asset unreadable; hash/schema/variant mismatch; reference/gold disagreement; illegal or missing candidate; supported-domain query error; perspective or atomic-action assertion failure | Stop the process or batch immediately. Generate no label, make no move, and do not fall back. |
| `stage_block` | Contractual failure of teacher throughput, coverage, calibration, natural-distribution, or performance gates; presenting decision/ply-level $3/n$ as a formal safety bound | Mark the current Stage incomplete and prohibit dependent Stages. Retain diagnostic artifacts. |
| `target_abstain` | HumanPolicy OOD; excessively wide `T_full` interval; target-specialised candidates not distinguishable within budget | If the corresponding proof exists, use explicit `T_ref` groups/mass; otherwise preserve `proof_unknown`. Count the event under a rate cap. |
| `unsupported_reject` | Database-family mismatch or a predeclared unsupported state/mode | Reject before entry into the formal path and report the precise reason. Do not merge it with an unexpected query error. |

Only a `target_abstain → T_ref` transition explicitly listed in the contract is a safe fallback. Fault injection must cover every fail-closed path and assert that the process produced no checkpoint, training shard, or `passed` artifact. A `fatal_stop` terminates a bounded transaction: the current decision commits no partial action; the worker exits non-zero with a structured error code; and the supervisor marks the game instance unavailable and retains a reproduction package. Product rules explicitly decide how the game itself is stopped, but a weaker engine must never take over covertly.

### 12.1 Learning Algorithm by Stage—Overview

| Stage | Primary method | Artifact | Explicitly excluded |
| --- | --- | --- | --- |
| 0 | No learning; differential/property/state-machine verification | Trustworthy oracle, rules, and data contract | Generating training labels before the gate passes |
| 1 | Supervised learning and calibration | HumanPolicy ensemble, safety auxiliary, independent baselines | Treating unchosen candidates as negatives; replacing Malom with a prediction |
| 2a | Frozen multi-branch search/rollout | FrozenTeacher ordered-group/group-mass data with confidence | Mutually scoring models that drift together |
| 2b | Listwise distillation | StrategyPolicy | Behaviour-cloning the human mode; forcing an order on uncertain pairs |
| 2c | DAgger-style state aggregation | Teacher labels for student-reached states | Executing unsafe actions to collect data |
| 2d | Supervised hard-state correction | Boundary/transition/disagreement coverage | Treating a temporal precursor as a causal root cause |
| 3 | Fixed-target deployment distillation + runtime closure + shadow replay | Unified compact policy, bundle, and offline safety report | Changing the teacher objective; granting theoretical authority to a prediction head; online learning |
| E-PPO—isolated experiment | Incremental masked-PPO experiment | Reversible experimental checkpoint | Entering the default dependency graph or consuming a reserved milestone |
| 4 | No learning | Frozen acceptance and release manifest | Training during the examination or online write-back |

A specialised release candidate uses the frozen Stage 2d policy. A reference release candidate uses the frozen `q_ref` source directly. Both must undergo unified Stage 3 deployment distillation and runtime closure before becoming compact candidates. A Stage 2 oracle-mode checkpoint is not itself a release artifact. E-PPO is outside the Stage 0–4 dependency graph.

### Stage 0: Correctness First

Objective: turn every deterministic premise into an executable gate.

- Startup preflight must resolve and open the read-only inputs selected by `sanmill_checkout` and `malom_db_path`, including any link target, plus the raw data/manifests under `../human_database`. If any is unreadable, `fatal_stop`; do not download a substitute, create an empty database, switch to a mock, or skip the check.
- Validate `std.secval`, the 498 sector files, and a size/hash manifest. Asset paths are read-only and supplied by configuration.
- Implement complete `OracleValue` and a reference-compatible `move_value` comparator. Compare separately against Sanmill Rust raw decoding and complete C++ value behaviour. Cover sector correction, entry kind, symmetry redirect, both side changes, placement hand counts, virtual terminals, and complete capture continuation. Include a dedicated Cartesian matrix for both colours across move/place→Mill→capture→4→3, end of placement after capture, side change, and symmetry redirect.
- Lock the rules variant; correct general threefold repetition, the no-progress state machine, and counter resets on capture and placement. `DecisionState`, caches, FEN v2, and the canonical key share one state definition.
- Build adversarial search and property tests for W liveness and L-state `horizon-safe(X)`, verifying the unit of $X$ and its threshold boundary.
- Rebuild HumanDB v3 only after P0-D0 opens the human-specialisation branch; deduplicate before splitting and run the §7.5 data assertions. P0-D0 itself is read-only, and the full builder is not an exit condition for Stage 0 on the reference path.
- Generate and freeze `label_authority_registry`, development conformance data, and verifier-owned acceptance gold. Run the contamination scan.
- Apply mutation/fault injection to sector correction, perspective, candidate omission, atomic capture, unreadable assets and hash mismatch, and the gate write path. Confirm that each error triggers the contractual stop level and emits no label or checkpoint.
- Remove every silent fallback from the formal path; complete full-candidate batch queries, cache-consistency checks, and the throughput benchmarks in §17.
- Generate a read-only Stage 0 manifest containing code commit, rule/oracle/data schemas, database and test-set hashes, commands, exit codes, and counts.

Every exit condition is deterministic: required-root preflight passes; field-level disagreement on reference development and acceptance gold is zero; the number of illegal, intermediate-removal, or wrong-perspective samples entering a snapshot is zero; query success is 100% for supported settled candidates; and every history-state-machine, safety-property, and fault-injection test passes. “No observed error” in a finite random test cannot replace these gates.

### Stage 1: Human Behaviour and Foundational Safety Models

Stage 1 does not train the final AI move selector. For the human-specialisation branch, HumanPolicy and the safety auxiliary are both mandatory, independent tasks. The reference path does not wait for HumanPolicy at P2-B1, and the proof-only minimum closure does not use the safety auxiliary as an authorisation prerequisite. If a reference source enters full P3, complete the safety-prior/OOD head according to that bundle's approved recipe. Each task records its update count, batches, early-stopping metric, and state-source weights separately in configuration.

#### Stage 1a — HumanPolicy

- Input one complete observed decision, its legal choice set, continuous Elo, platform/time conditions, and both sides' Phases. The target is the action actually chosen.
- Optimise masked categorical NLL over the choice set with player/game-normalised weights. Primary metrics are NLL/Brier and calibration; top-1/top-k are readability metrics only.
- Produce out-of-fold logits with grouped K-fold. Freeze final ensemble membership, temperature/calibrator, and training-fold hashes. The teacher may read only OOF or strictly out-of-sample predictions.
- Output epistemic spread, effective sample size, and OOD/abstention. A static heuristic cannot masquerade as HumanPolicy where data coverage is absent.

The cold-start order is fixed: first train representations and the safety auxiliary on reachable states for legal-action, Phase, and D4 consistency; train HumanPolicy by behavioural cloning on observed choices only; train StrategyPolicy on complete `q_ref` first; and add target-specialised ranking only after both the support gate and the `T_full` viability gate pass. Synthetic states, symmetry tasks, and Malom labels must never enter HumanPolicy's human-choice field.

#### Stage 1b — Safety Auxiliary

- For **every legal atomic candidate** in each training state, obtain `allow_tier ∈ {theory_safe, horizon_safe_X, deny}` from the oracle/rules gate. Mark candidates unresolved within budget as `proof_unknown`; exclude them from the allow/deny classification loss and never disguise them as deny or horizon-safe.
- Train the auxiliary with stratified classification or BCE over all legal candidates. It makes pre-mask logits and the §10.4 gate-intervention risk interpretable and becomes the `safety-prior head` used in Stage 3 for candidate search order, secondary policy suppression, and OOD warning. It may over-block and reduce strength, but cannot allow by itself: the final compact candidate still requires W/D exact-pack positional closure or a complete runtime survival proof, and an L state cannot rely on the pack alone. The oracle workbench always queries the gate directly.
- Report unsafe recall, precision, Brier score, and calibration separately for W→D/L, D→L, W-liveness failure, L below $X$, Phase, sector, and boundary. Overall accuracy cannot be the sole exit condition.
- Sample hard negatives from oracle value boundaries, capture-to-fly, perspective transforms, history thresholds, and symmetry variants. Select loss reweighting on train/validation only.

#### Stage 1c — Optional Representation Auxiliaries and Sharing Ablation

Optionally train auxiliary tasks for coarse W/D/L, complete Malom value bins, Phase/transition, and strategic tags; they support representation and diagnosis only. For every head, plot learning curves by data volume and on held-out sectors/canonical families; remove a head that does not generalise. Independent encoders remain the default. A sharing candidate uses deterministic round-robin task batches, separate heads and calibrators, and fixed update counts. Record gradient cosine on shared parameters and held-out metrics for every task. Persistent conflict or regression on any task rejects sharing.

The Stage 1 exit artifacts for the human-specialisation branch are the frozen HumanPolicy ensemble, safety auxiliary, separate calibration/OOD reports, and version manifests. Thresholds and margins are locked after the pilot and before formal validation comparison. These artifacts are not prerequisites for the reference path entering P2-B1; dependencies follow the machine-readable DAG in §18.

### Input Representation and Feature Budget

The default input is the minimally sufficient state: occupancy of the 24 nodes; fixed adjacency and Mill lines; side to move; on-board and in-hand counts for both sides; matched variant; rules-history summary; Phase and transitions; candidate from/to/capture; and legal mask. Use a fixed-graph encoder or an equivalent action-conditioned model. “A graph model should learn board patterns by itself” remains a hypothesis to be tested. Representation selection must include a clean handcrafted baseline on the same data and `A_allow`, matched as closely as possible in parameter count and latency: a phase-aware MLP reading piece count, mobility, blocked pieces, closed/open Mills, immediate Mill/capture, capture-to-fly, and candidate-local relationships. If the graph model does not reliably outperform this baseline on natural and stress teacher regret, retain the simpler model.

Manage features in three tiers. **Required state fields** determine the Markov/rules state and action semantics; reject a sample when they are absent. **Optional deterministic derivatives** include mobility, immediate Mill, open Mill, capture-to-fly, and safe width; ablate each group independently and retain it only after stable improvement in held-out regret or calibration within the latency budget. **Data-aggregate features** include HumanDB frequency, Elo-conditional priors, and empirical win rate; generate them out of fold from the train fold only, and prohibit them from RuleAwareSafetyGate. Maintain one `feature_registry` recording name, semantics, dtype/range, perspective, computation function, dependency versions, cacheability, training/inference availability, and schema hash. Every new feature requires a single-group ablation, leakage check, and batch benchmark; it is not merged if it exceeds budget.

### Stage 2: Frozen Teacher, Distillation, and State Aggregation

Before any target-specific labelling, deliver and freeze the `reference_safe_baseline` comprising `RuleAwareSafetyGate + mode-specific q_ref`. Then train a minimum reference-only student on `q_ref` and run `compact_reference_closure_gate`: the unified deployable policy reads no oracle-only mode or feature; the runtime proof completes on target hardware; and `release_support_contract` cannot be reduced to a small demonstration domain when its target-start-mass floor or opponent closure fails. Training-owned, closed-loop compact rollouts must satisfy the planning gates for unsupported/OOD states, support escape, the cluster UCB of `runtime_bound_unavailable`, `oracle_X_viability_failure_rate`, shortfall audit, proof-pool censoring, the upper bound on theoretical downgrade, W conversion, size, and latency. This gate proves only that the release architecture may close; it does not establish target specialisation or a human gain. If it fails, repair the compact architecture or stop the release path before building full `T_full` or DAgger. After it passes, compare every subsequent component incrementally against the same-version oracle reference and frozen P2-B1 reference-only prototype. Remove HumanPolicy, `T_full`, DAgger, or a scenario curriculum if it has no attributable net benefit, while preserving the minimum closed loop.

#### Stage 2a — Construct FrozenTeacher

The teacher receives the complete `DecisionState` and every atomic candidate. RuleAwareSafetyGate first produces `A_allow`; the teacher then evaluates every permitted candidate offline:

1. Store the complete reference value, W/D/L, W proof artifact/rank, or L-state $D_{survive}$.
2. Enumerate every next reply by the opponent, read out-of-fold HumanPolicy probabilities, and compute one-step safe mass directly.
3. Continue branches covering at least $1-\epsilon_{mass}$ probability under frozen $H_{opp,max}(S)$/$H_{self,max}(S)$ schedules. Bound unexplored probability mass by best and worst cases. Tighten a residual bound from facts only when a rules terminal, an already-observed mode event, a W-liveness/impossibility proof, or an exact oracle tier logically fixes the event to 0 or 1. A Malom rank or W/D/L value cannot be mapped into a human event probability.
4. Always add a reference-optimal or strongest-safe-reply branch so that “the opponent is likely to err” cannot conceal that the line is refutable.
5. Freeze self continuation, HumanPolicy ensemble, AB profile, search/node budget, random seeds, and truncation rules.
6. Compute separately the target outcome, diagnostic $T_{pos}$ survival, D-primary $T_{certW}$ interval, conversion, gate-intervention risk, OOD, and ensemble/search intervals.

The labelling pipeline is tiered, and every label records `teacher_tier`:

| Tier | Mandatory computation | Strategy labels it may produce |
| --- | --- | --- |
| `T_ref` | Full-candidate rules/oracle, complete ultra-strong comparator, and W-liveness/L-horizon gate | Mode-specific `q_ref` groups/mass only on matched support where the corresponding proof is complete; otherwise `proof_unknown` |
| `T_one` | `T_ref` plus every one-step opponent reply and HumanPolicy safe mass/OOD | Strategy labels remain reference-only; one-step quantities serve diagnosis and acquisition escalation only |
| `T_full` | `T_one` plus state-conditional multi-horizon rollout, strongest-safe branch, and residual/ensemble bounds | The only tier permitted to produce non-reference ranking specialised to the target humans |
| `T_exact` | Enumerable subdomain or higher-budget expectimax/adversarial search | Teacher validation and confirmation only |

The machine contract fixes `tier_permissions.T_one.non_ref_q: false`. The verifier asserts that every record whose highest tier is `T_one` agrees field by field with `T_ref` in ordered groups, group mass, and tie reason; a violation is `fatal_stop`. When `T_full` cannot distinguish candidates because of budget, OOD, or interval width, it may explicitly return to `T_ref` groups/mass and record `target_abstain_reason` only if the corresponding W/L proof is complete. If the proof is unresolved, preserve `proof_unknown` and exclude the record from policy labels. A genuinely large tie in the reference remains a genuine large tie.

`q_ref.W` requires a passed `w_liveness_feasibility_pilot`, a hit by the current full-history proof, and non-empty `A_allow^W`. If the pilot fails, only diagnostic output labelled `positional_reference_shadow` is permitted; it cannot enter certified move selection, W policy loss, the `wdl_floor.W` denominator, or `reference_safe_baseline`. The valid responses are to expand offline proof/cache, narrow the release support domain, or issue `stage_block`.

Before constructing the full `T_full` pipeline, pass a read-only statistical and small-simulation `analytic_power_screen`. The product first freezes `delta_specialize.D`. By target stratum, candidate count, number of simultaneous comparisons, rollout correlation, event base rate, HumanPolicy uncertainty radius, and residual/proof-unknown mass, the screen computes the minimum detectable difference under a per-candidate budget $n$:

$$
\operatorname{MDE}_g(n)\approx
\left(z_{1-\alpha_g/2}+z_{1-\beta}\right)
\sqrt{\widehat{\operatorname{Var}}_g(\widehat{\Delta Q})/n}
+b_{HP,g}+b_{resid,g}+b_{proof,g},
$$

where $\alpha_g$ already includes simultaneous control for multiple candidates in the same state. None of the three bias terms vanishes by adding Monte Carlo samples at the same tier. The screen and formal `T_full` must read the same `random_coupling_spec=actor-counter-v1`: shared latent variables are generated by comparison ID, actor, the $k$-th future decision, and draw kind. The artifact reports $\widehat{Var}(\Delta Q)$ under both common random numbers and independent Bernoulli draws and gives their ratio. If coupling consistency fails, MDE always uses the independent upper bound. A scale check must display a grid over event rate, difference, candidate count, and proof coverage. For example, comparing event rates 0.10 and 0.13 at two-sided 5% significance and 90% power requires approximately **2,400** rollouts per candidate; simultaneous multi-candidate correction to effective two-sided $\alpha=0.01$ requires about **3,400**, before HumanPolicy and proof bias. If MDE still exceeds the product delta under the optimistic boundary of `resource_envelope`, or supported-state supply is insufficient, close target specialisation immediately and continue only `reference_safe_baseline`. This screen decides whether the design has power; it does not prove that a difference exists.

Stage 2a then runs an independent pilot and produces and locks `teacher_budget_contract`. By Phase/W/D/L/branch tier it declares per-state and per-candidate node/time/branch limits, probability-mass coverage, and maximum residual width; cold/warm throughput, peak memory, and queue capacity; `wdl_floor.{W,D,L}`; target proportions for each tier and `tier_permissions`; labelable `T_full` states per hour; upper limits on abstention and large ties; and hard limits for DAgger collection, cheap labelling, and full escalation in each round. Costing must include all $|\mathcal U_{self}|$ continuations and their common-random comparisons in `self_continuation_stability_gate`. The D-mode budget jointly accounts for rollout and the proof after arrival in W. In every branch stratum, reserve according to the upper confidence bound on positional-W arrival probability and proof p95:

$$
C_{state,hi}=C_{rollout,hi}+\sum_b \overline p_b(\text{positional-W})\,C_{proof,b,p95}+C_{reserve}\le C_{state,cap}.
$$

The contract also supplies expected and maximum exact-tie group size, a lower bound on distinguishable pairs, and an upper bound on the supply of proof-covered W samples; the last enters the feasibility solver for `wdl_floor.W`. The pilot is divided into two non-overlapping parts. `planning_pilot` estimates only cost, variance, event rate, and attainable coverage. After budget and thresholds are frozen from it, `confirmation_pilot` makes one go/no-go decision on new states. Neither enters final validation or test. For each stratum $g$, the artifact computes at least `C_full_hi = Σ_g N_g × r_g × c_g,p95` and uses measured parallel efficiency to give point estimates and conservative upper bounds for CPU/GPU hours, wall-clock time, I/O, and storage. The implementation plan must provide in advance `resource_envelope.{cpu_core_hours,gpu_hours,wall_clock_hours,storage_gb,workers}`, which an Agent cannot increase autonomously. Before confirmation, freeze `min_t_full_coverage`, `max_target_abstain_rate`, `min_distinguishable_pair_fraction`, `min_effective_full_labels`, and every stratum floor. If the projected upper bound exceeds the resource envelope or confirmation fails, issue `stage_block`. Permitted responses are to optimise batch queries or caches, narrow the target support domain, or stop target specialisation—not to add compute automatically, lower interval requirements, or fill labels with `T_one`.

Cost for `horizon-safe(X)` is stratified separately by L/history/capture-to-fly. Use terminal bounds, validated transposition/proof caches, reference ordering, and admissible lower and upper bounds for cheap acquisition, then deepen candidates that may reach $X$ one tier at a time and preserve a verifiable proof artifact. “Not proved to $X$” is `proof_unknown`; “proved to fall short” is shortfall. They must not be conflated. Prewarm the proof cache offline over the target support domain and validate cold and warm p99. A state with no proof within the real-time deadline must not be described as `horizon-safe(X)`. Beyond enumerable small domains, `T_exact` draws confirmation samples from natural random states, highest-disagreement states, Phase/history boundaries, and 4→3 strata to estimate teacher-cascade rank-flip rate.

Labelling jobs use a content-addressed key—canonical `DecisionState` + candidate list + teacher/oracle/rule/HumanPolicy/budget hashes—and deterministic random streams. Workers write append-only shards only. The merger verifies candidate completeness, field-for-field equality of duplicate-key outputs, and tier state before producing a read-only snapshot. An interrupted retry must yield the same label or an explicit nondeterminism error.

If, under the contract, `T_full` coverage is too low, or abstention and large ties leave too few effective ranking samples, Stage 2a is incomplete. First narrow to preregistered strata with adequate support. If the gate still fails, deliver `reference_only` explicitly; do not let Stages 2b–2d spin idly on “uniformly safe moves.” The only permitted degradation path is to narrow specialisation support to strata covered by HumanPolicy and confirmable by `T_full/T_exact`; add `T_exact` labels in enumerable or high-budget confirmation subdomains; and use `q_ref` explicitly elsewhere. HumanDB paths observed and verified as `certified_exploitable` may receive higher D-mode acquisition priority. `positional_only/proof_unknown_candidate` first errors and ordinary precursors enter diagnosis or the proof queue only. Because every observed path lacks outcomes or propensities for unchosen actions, none may become a candidate-preference label directly.

The teacher is stratified by the hard constraints and modes in §13. W guarantees reference conversion. D preserves D and uses the complete Malom ultra-strong policy as its baseline. L first satisfies `horizon-safe(X)`. A target-specialised departure from the ultra-strong baseline requires all three checks: a conservative lower-bound advantage under the label-generation HumanPolicy ensemble; directionally consistent results from an audit HumanPolicy version that did not participate in candidate selection and used train/validation only; and no deterioration under the reference-best-reply branch. The audit is only a **same-source cross-fold stability check**, not a second human-data source. Failure of any check returns the reference groups/mass. Output consists of ordered candidate groups, group mass, teacher tier, tie reason, confidence, abstention reason, and a complete version manifest.

HumanPolicy error must propagate into candidate differences. `behavior_uncertainty_contract.version=cluster-envelope-v1` fixes one algorithm for constructing the set. Within each frozen support stratum $g$, generate calibrated action-probability vectors $\tilde p_r(\cdot\mid S)$ from different model families, OOF folds, player/game-cluster bootstrap refits, temporal slices, and preregistered density-ratio extremes; then define:

$$
\mathcal P_h(S)=\operatorname{conv}\!\bigcup_{r\in\mathcal R_g}
\left\{p\in\Delta(A_{legal}(S)):\lVert p-\tilde p_r(\cdot\mid S)\rVert_1\le \rho_g\right\}.
$$

Register member-generation rules, $\rho_g$, action alignment, convex-hull/distance semantics, the numerical infimum algorithm, and random streams before planning. An OOD state triggers support abstention; it does not feign coverage by inflating $\rho_g$. For every candidate difference, the teacher takes infimum and supremum over the **same** frozen set. Calibrate $\rho_g$ only against observable behavioural functions. The contract freezes `alpha_HP`, default 0.05, and a function family—preregistered action/equivalence-group mass, safe/unsafe mass, one-step error severity, and Phase-conditional choice rate. On player/time-held-out calibration cells, use nested cluster bootstrap and simultaneous max-stat bands so that the lower confidence bound on empirical joint coverage of held-out frequencies by set-induced intervals is at least $1-\alpha_{HP}$. A non-overlapping `confirmation_dev` performs one confirmation only. Such coverage does not prove that the true state-conditional distribution lies in $\mathcal P_h$, much less identify human continuations for unchosen candidates. A same-source audit cannot narrow $\mathcal P_h$ or be counted again as a second independent significance test. If P0-E obtains target-condition-matched behavioural data with disjoint source and players, it may be preregistered before collection as `external_behavior_confirmation`. An opposing direction returns to reference or abstention; agreement still cannot replace the final human A/B test. Specialisation advantage uses the conservative bound `inf_{p∈P_h} ΔQ_p` and reports rank-flip rate. If any reasonable perturbation reverses the ranking, tie or abstain.

A calibrated AB profile supplies heterogeneous search-model stress but does not automatically receive veto authority under a human label. Its rank reversal enters the conservative gate only if it passes choice calibration, error severity, and Phase-conditional fit on held-out HumanDB not used for teacher selection, and the contract predeclares it as an outer member of $\mathcal P_h$. If both a validated AB profile and HumanPolicy show a candidate to be brittle, tie or abstain.

HumanPolicy robustness does not imply robustness to self continuation. For every target-label version, `self_continuation_stability_gate` freezes $\mathcal U_{self}$ with at least two reasonable continuations. The initial round contains the formal label continuation and at least one different policy from the current product, `q_ref`, or independent reference search. Once a student exists, it must also include the adjacent previously passed checkpoint. Every member obeys the same gate, horizon, and actor random stream. For each non-reference candidate edge, calculate $\Delta Q_u$ separately under the same common-random-number comparison. Authorisation uses a conservative interval across continuations:

$$
L^{self}_{ab}=\min_{u\in\mathcal U_{self}}\operatorname{LCB}\!\left(Q_u(a)-Q_u(b)\right),\qquad
U^{self}_{ab}=\max_{u\in\mathcal U_{self}}\operatorname{UCB}\!\left(Q_u(a)-Q_u(b)\right).
$$

Retain the edge only if $L^{self}_{ab}>\delta_{order.m}$ and the specialised top exceeds `delta_specialize.m` against the active control for every $u$. If any continuation reverses direction, an interval contains zero, or the top group changes with $u$, the entire state becomes `target_abstain`. Reports stratify `rank_flip_under_self_continuation`, top-group Jaccard, effect range, and additional cost. A change in the hash of $\mathcal U_{self}$ invalidates old specialised labels. If two deployment-relevant continuations cannot be found, D target specialisation is a `stage_block`.

Every target label must also pass `human_support_gate`. Using train-fold-only statistics or density models, report raw decisions, independent games and players, player-cluster ESS, maximum player weight, neighbourhood distance, calibration upper bound, and support-domain version by target Elo band, both sides' Phase/transition, opening family, choice-set/occupancy structure, and data source. If cumulative probability mass on unsupported human decisions during rollout exceeds `max_unsupported_human_mass`, or a critical branch is OOD, candidate differences retain only worst/best residual bounds. If those bounds can reverse the ranking, issue `target_abstain`. Widening an Elo or structural backoff changes the target condition and requires a new target ID.

The first `target_specialisation.phase_scope` excludes states where either side is already flying. Those states still receive complete rules processing, mode-specific `q_ref`, W-liveness/L-horizon handling, and regression acceptance. Enabling flying specialisation requires separate confirmation gates for support, branch cost, residual width, and effective-label rate.

Teacher data is constructed from the mutually exclusive §8 state buckets and the independent Phase/W/D/L loader in §11. The manifest records the training data, folds, model families, and uses of the label-generation and audit HumanPolicy versions. Validate one-step quantities, survival, ranking, and cross-version stability first on actual HumanDB paths, temporal/player validation, and enumerable/search subsets. A failure prohibits student training.

Before building full Strategy, DAgger, hard-state, or RiskSurvival tasks, pass `core_hypothesis_gate`. Use only a preregistered small confirmation set of high-support, $|A_{allow}|>1$, D-mode strata and generate selective `T_full`. First compare the online direct teacher with `q_ref`, then train a minimum policy-only student and conduct paired validation on an audit set not used for label selection, validated AB/search stress, the reference-best-reply branch, and natural held-out starts. The gate requires the LCB of the direct-teacher effect to exceed `delta_specialize.D`; all stability checks to pass; the student to retain a preregistered fraction of the teacher gain; distinguishable-pair, abstention, and cost metrics to meet their contractual gates; and natural/reference metrics not to regress. Any failure stops target specialisation and delivers only `reference_safe_baseline`. This gate remains proxy feasibility evidence and does not replace the §15 human A/B test.

#### Teacher Capability Boundary and Distillation Ceiling

This plan does not assume that FrozenTeacher is a “perfect demonstration of playing style.” Malom gives exact positional value and comparator only within its support domain. A W proof answers only whether a win can be forced under the complete history. HumanPolicy estimates behaviour only within observational support. `T_full` is an action-conditioned event probability under frozen policies for both players, a finite horizon, a branch budget, and an uncertainty set. Distillation can approximate only the teacher's supplied conditional partial order and group mass. Every candidate version decomposes the gap into four stages; one aggregate win rate may not conceal them:

| Comparison | Primary diagnosis | Conclusion |
| --- | --- | --- |
| `q_ref` → online direct FrozenTeacher | Teacher specialisation gain, interval, and unit cost | Whether the teacher itself produces a reproducible gain |
| Online direct FrozenTeacher → student on the same states | Group-mass CE, ordered-pair violations, top group, and mode regret | Distillation gap caused by representation or optimisation |
| Student on offline states → student-reached states | On-policy regret, support rejection, and DAgger ROI | Covariate-shift/coverage gap |
| Frozen proxy → player-disjoint human A/B | Both co-primary effects and intervals | Proxy gap and whether the system truly works better against humans |

Only when the first stage is stably positive and the second and third are below preregistered tolerances may the remaining plateau be diagnosed as “possibly constrained by the teacher/objective ceiling,” after which the bottleneck-escalation ladder may compare a stronger teacher, independent search, or E-PPO. If the online teacher itself does not outperform `q_ref`, neither a larger student nor PPO is a justified remedy.

#### Stage 2b — Listwise Policy Distillation

StrategyPolicy is normalised only over `A_allow`. The teacher emits ordered candidate groups $\mathcal G=(G_1,\ldots,G_J)$ with group masses $Q_j$. Define first the **group-mass listwise** per-state term without state weight, then apply the frozen sampling contract:

$$
\ell_{group}(S)=-\sum_{j=1}^{J}Q_j(S)\log\!\left(\sum_{a\in G_j(S)}\pi_\theta(a\mid S,A_{allow})\right),
\qquad
\widehat{\mathcal L}_{policy}=\frac1B\sum_{i\in\mathcal B}w(S_i)\,\ell_{group}(S_i).
$$

Candidates occupy different ordered groups only if a simultaneous interval distinguishes them and they pass the §13.2 robustness gate. Overlapping intervals mean “the evidence cannot distinguish them,” not equal utility. The primary loss constrains whole-group mass and inter-group partial order; it does not require equal logits within a group. When `tie_reason=reference_exact`, the frozen §13.2 `uniform-canonical-v1` conditional policy is part of the label. Training **must** add within-group conditional KL with its weight locked in the contract. `interval_unknown` means insufficient evidence and requires within-group loss weight zero. The two classes use distinct reason codes. On `T_full` abstention, inherit explicit `T_ref` groups, group masses, and exact-tie conditional policy. Unsafe candidates enter no policy loss; they enter only the Stage 1b auxiliary. Add D4 action-equivariance consistency and validate forward and inverse transforms of captures.

An optional ablation may add margin ranking, Bradley–Terry, or tie-aware Plackett–Luce terms only over teacher-distinguishable pairs or partial orders, weighted by frozen simultaneous LCB or interval width. Every indistinguishable pair has weight zero. Retention is decided by held-out mode-specific regret. The training sampler may oversample the `distinguishable_pair` subset by information content, subject to update-ledger caps and the natural-replay floor. A genuine exact large tie uses `uniform-canonical-v1` in both training and deployment, with counter-based sampling. If distinguishable pairs remain below the contractual minimum, target specialisation is not learnable; neither handcrafted principles nor log ordering may fill the gap.

Select checkpoints by held-out group-mass CE, ordered-pair violation, top-group coverage, regret, symmetry consistency, and natural/stress strata. Every round reports `teacher_tier_rate`, `target_abstain_rate`, the distribution of `tie_group_size`, group-mass entropy, distinguishable-pair fraction, reference-only fraction, and effective ranking-sample count. Compare simultaneously with random-allowed, Malom strong, complete Malom ultra-strong, AB, and human-frequency-only baselines. Dependence of `bounded-effect-softmax-v2` on group count must be measured rather than condemned a priori. Stratify every CE, top-group probability/calibration, gradient norm, and mode regret jointly by $J$, $|G_1|$, $|G_1|/|A_{allow}|$, Phase, and teacher tier. Trigger a same-budget `top-mass-anchored-v1` ablation only if, after controlling these strata, `selection_dev` still shows a monotonic deterioration with $J$ beyond preregistered `max_J_stratum_regret_gap`. For $J=1$, it sets $Q_1=1$. Otherwise, it fixes top-group mass to $Q_1=[1+\exp(-g_1/\tau_m)]^{-1}$ and conditionally normalises the remainder over the v2 tail logits. The default v2 and this variant each use at most the preregistered seed/run count, and only one winner accesses confirmation once.

**The default StrategyPolicy architecture contract is `policy_only`.** A Stage 2 student on the oracle workbench may use a shared state/action encoder with three lightweight W/D/L policy projections. The gate supplies the current full-rules mode, and only the corresponding head is updated. Each mode has independent early stopping, regret/calibration, and a minimum-sample gate. The shared trunk records per-mode gradient cosine and scale; regression in one mode cannot be offset by the average of the other two. Before round one, the conflict fuse locks its monitoring window and patience. Only persistent conflict accompanied by held-out regret regression triggers a mode adapter, freezing of a shared block, or an independent encoder. **These oracle-mode heads are not a compact inference interface.** By default, Stage 3 distils their frozen distributions into one action-conditioned compact head whose inputs exclude W/D/L, proof, or mode fields unavailable on a pack miss. If an implementer chooses a runtime router instead of a unified head, they must first prove that the router reads deployable fields only and treat router error, fallback, and latency as primary gates. Predicted W/D/L has no safety or mode-adjudication authority. Teacher $Q_\nu$ remains in offline candidate records; the student need not reproduce it online. An optional `aux_q_head` is an ablation only, without gate or override authority. If E-PPO is separately initiated, build a distinct critic conditioned on mode and opponent whose target is the frozen rollout return.

#### Stage 2b-trajectory — Continuous Coverage and Mode-Routing Curriculum

Multi-step competence does not require PPO. Under the locked rules, `DecisionState` is a Markov input, and multi-step $Q_\nu(S,a)$ from `T_full` already attributes future differences to the current candidate. The student still needs to see reached states where it must switch from inducement to conversion. Reuse teacher/DAgger branch trees whose cost has already been paid, preserve continuous and non-spliceable `trajectory_id/step/mode/event`, and construct three kinds of supervised chain: W-start liveness→terminal conversion; occurrence of $T_{certW}$ in D→W conversion; and rescue from L into D/W→the new mode. An actual HumanDB path enters the corresponding observed chain only if verified `certified_exploitable`; every unchosen candidate still requires a teacher/search label.

Batch continuous short chains, but at every AI decision use that state's own mode-specific group-mass or partial-order target. Broadcasting the final result along the chain is prohibited. `mode_transition` determines only which W/D/L head reads the next state; the first step after entry into W immediately uses the W-liveness teacher. The update ledger sets non-zero floors and caps for the first $k$ AI decisions after a transition and reports `D→W chain coverage`, `L→D/W coverage`, teacher regret on the first W decision, switch latency, and final conversion. This is not a sequence-level joint loss: CE/KL remains per-state along the chain. Positive examples may be stratified from the already expanded `T_full` tree, but retain the original HumanPolicy branch probability, sampling probability, and importance weight. If coverage, event-probability mass, or effective sample size fails the contractual gate, Stage 2 returns to local specialisation or the reference baseline.

`trajectory_weight_contract.version=target-measure-iw-v1` specifies how these fields enter loss. The frozen snapshot first partitions every continuous short chain into non-overlapping draw segments $c$, recording the actual HumanPolicy branch/path probability, source, and canonical occurrence. The update ledger defines normalised **target update mass** $\mu_c$ and within-segment state mass $\nu_{i\mid c}$, with $\sum_c\mu_c=1$ and $\sum_{i\in c}\nu_{i\mid c}=1$. Natural-subdistribution $\mu_c$ retains branch probability. Artificial transition floors and caps enter $\mu_c$ as a separate curriculum subdistribution. If multiple windows cover the same `(trajectory_id, step)`, apportion or merge its target mass first. Let $s_c>0$ be the exact probability that the loader draws segment $c$ in one optimizer draw, including every sampling stage. The primary loss weight for state $i\in c$ in that draw is fixed as:

$$
w(S_i)=\frac{\mu_c}{s_c}\,\nu_{i\mid c},\qquad
\widehat{\mathcal L}_{traj}=\frac1B\sum_{c\in\mathcal B}\frac{\mu_c}{s_c}
\sum_{i\in c}\nu_{i\mid c}\,\ell_{group}(S_i).
$$

Use `branch_probability` exactly once, when constructing natural $\mu_c$. The `reference_exact` conditional KL and any optional partial-order loss enabled on the same state must use precisely the same $w(S_i)$. A missing or zero `sampling_probability`, disagreement with sampler logs, or omission of any sampling stage is `fatal_stop`. v1 does not permit silent clipping or per-record renormalisation of importance weights. Every epoch and stratum reports Kish $ESS=(\sum r_c)^2/\sum r_c^2$, where $r_c=\mu_c/s_c$; CV/design effect; maximum weight share; and the reconstruction error between the weighted source/mode/event mass and $\mu$. Freeze `min_iw_ess`, `max_iw_cv`, `max_iw_share`, and mass tolerance after planning.

This is the project's default intermediate tier between one-step distillation and on-policy PPO: search-augmented, trajectory-consistent offline policy improvement. Generic offline RL such as CQL or IQL is not included by default. A comparison requires the same `A_allow`, an independent critic target, coverage diagnostics, and a new implementation plan.

#### Stage 2c — DAgger-Style Aggregation of Student-Reached States

Pure offline distillation suffers exposure bias, but FrozenTeacher cannot be treated as an infinitely queryable, cheap expert. In each round, the current StrategyPolicy interacts **always behind the gate** with the frozen HumanPolicy ensemble, AB profiles, Malom reference, and a population of old student checkpoints. Target-HumanPolicy reached states are the primary practical distribution. The default is `arrival_mix.target_human_floor = 0.60`, with all robustness opponents combined capped at 0.40.

First apply uniform `T_ref/T_one` cheap labelling, canonical deduplication, and visit counting to reached states, then escalate to `T_full` by acquisition stratum: high visit frequency; high student/reference or cached-teacher regret; high policy/ensemble disagreement; oracle boundary; Phase/W/D/L boundary; and fixed-scenario regression. Before a full query, “high regret” may use only a cheap disagreement or uncertainty proxy. Each stratum has a cap and a natural-replay floor. Before entering the `T_full` queue, run a cheap `human_support_gate`, phase-scope check, and branch-cost upper bound. Before each round's formal acquisition, use an independent cheap forecast sample of newly reached current-student states to estimate intervals for `targetable_on_policy_rate`, support rejection, and proof/branch-cost rejection. If their conservative bounds miss the preregistered round gate, do not start `T_full` that round. Redirect states with existing proofs into `reference_robustness_queue` for `q_ref`/safety/OOD/regression coverage, and unresolved states into the proof queue. A coverage floor may have a different planned value frozen per round, but it cannot be adaptively lowered after observing a decrease in the preceding round.

- Never execute an unsafe action proposed by the raw policy. Record a gate-intervention event, then continue with the post-gate action or terminate the shadow branch.
- Fix per-round budgets for reached-state collection, cheap labelling, `T_full` escalation, teacher nodes/time, natural-data replay, and hard states. All are hard-limited by `teacher_budget_contract`. Once budget is exhausted, leave the state in a resumable queue; do not fabricate a label.
- Deduplicate states by canonical key and retain visit frequency without copying them into falsely independent samples. The teacher version remains fixed within a round; an upgraded teacher opens a new data version.
- Intermediate rounds default to controlled warm starts on the complete aggregate dataset with optimizer and scheduler reset. Retrain the final candidate once from scratch on the locked aggregate set with preregistered seeds and compare path dependence. Do not release if the difference exceeds tolerance.
- The default is `dagger.max_rounds = 2`, excluding initial offline distillation; zero rounds are permitted. Under each teacher version, full-label a canonical state only once. The contract separately computes `C_dagger_hi = Σ_r,g N_new(r,g) × full_rate(r,g) × c_g,p95`.
- Freeze `on_policy_t_full_coverage_floor` and `min_regret_reduction_per_1k_t_full` for each round. Stop immediately if full coverage misses its gate or the conservative bound on held-out regret improvement is below the marginal-cost threshold. A third round requires a new implementation plan and budget approval.

DAgger corrects only covariate shift in student-reached states; it does not correct the teacher objective. In addition to teacher regret, promotion in each round requires no regression on random natural states excluded from acquisition, a `T_exact`/reference confirmation subset, and a frozen end-to-end proxy. Roll back the entire round if hard-state refitting repeats without external confirmation improvement.

#### Stage 2d — Supervised Hard-State Correction

Hard states arise from high student/teacher regret, a safety-auxiliary false negative, oracle value boundary, history/liveness, capture-to-fly, Phase transition, Malom/AB disagreement, or a fixed trap. A candidate first enters quarantine. It is promoted to a training hard state only when an exact/reference boundary holds, or the failure recurs in a preregistered number of independent seeds/checkpoints and the same-state teacher difference exceeds its uncertainty interval. After repair, retest for a locked number of rounds and downweight or retire states with sustained absence of failure; fixed regression scenarios remain read-only. A decrease in safe width, increase in search entropy, or increase in gate risk merely localises a precursor candidate and does not automatically label a “root cause.” Hard states, opening bucket C, and curated scenarios share the §8 update ledger. A state discovered in validation or test enters only the next version's candidate queue; it cannot train the current version.

#### Optional RiskSurvivalModel—GapNet/SelfRiskNet Heads

Only after the direct teacher is complete and validated may the §10 amortiser be trained for latency: two independent heads over shared infrastructure, or independent models, strictly following `rollout-distill-v1`. Each head reads a frozen label version and is calibrated by task and policy version; critical decisions are sampled back to the direct teacher to measure error. If direct computation meets budget, deploy no Net.

Mandatory Stage 2 exit artifacts are `reference_safe_baseline` and a complete validation report. FrozenTeacher manifest and a specialised StrategyPolicy are delivered only after the specialisation gate passes. The observed oracle-downgrade count in gated rollout must be zero, but that count proves only sample-level execution; the safety conclusion continues to rest on Stage 0 invariants.

### Stage 2 Bottleneck-Escalation Ladder

Narrowing the default path does not mean redesigning from scratch after a plateau. Before every escalation, identify the bottleneck from frozen reports and open only the corresponding small, falsifiable experiment. A failed experiment returns to the most recent passed artifact:

| Frozen symptom | First diagnosis | Next approved experiment | Prohibited shortcut |
| --- | --- | --- | --- |
| `T_full` throughput or intervals fail | Teacher-economics bottleneck | Batched oracle, proof/branch cache, admissible bounds, narrower support domain, or `T_exact` in an enumerable subdomain | Reduce coverage, discard residual mass, or rank with `T_one` |
| Insufficient HumanPolicy support | Human-counterfactual-evidence bottleneck | Narrow the target ID, or prospectively collect safe-set micro-randomised data with true propensity | Fabricate target-Elo labels from AB, a population-wide average, or a heuristic |
| Student teacher regret remains high | Distillation/distribution-shift bottleneck | Add distinguishable samples, adjust a verifiable architecture, run at most two ROI-justified DAgger rounds, or acquire continuous chains | Terminal broadcast, unlimited DAgger, or relaxed ties |
| Student matches teacher but independent outcome plateaus | Possible teacher/objective ceiling | Upgrade `T_full/T_exact` or an independent search control first; initiate E-PPO only when evidence is sufficient | Claim to exceed the teacher from training win rate alone |
| Insufficient D→W/first-W-decision coverage | Routing/reached-state bottleneck | Target continuous branch-tree chains, a mode-balanced loader, and a confirmation set | Broadcast the final outcome to every precursor |
| `horizon-safe(X)` proof cost exceeds budget | Proof-coverage/support-domain bottleneck | Offline proof cache, iterative deepening, admissible bounds, and narrower support domain | Present a value/head probability as an $X$-ply guarantee |

Escalation choice, trigger metrics, maximum budget, success/failure gates, and downstream dependencies enter a new detailed plan and execution lock. If no compliant path produces a detectable net gain, the correct deliverable is `reference_safe_baseline` or `inconclusive`.

### Isolated Experiment E-PPO: Masked PPO Under a Fixed Objective

E-PPO is outside the default path and is not a milestone that “will eventually be required,” but it is not banned permanently. Only after Stage 2 is frozen and passed, student teacher regret reaches a preregistered approximate floor, `T_full` and HumanPolicy support are adequate, and independent outcomes still plateau across several seeds and budgets may “possibly constrained by the teacher ceiling” become the project hypothesis. A newly approved experimental objective and separate resource envelope under §18 then preregister primary metrics, non-inferiority margins, sample size, seed count, and stopping rules.

Constraints: rollout, old/new-policy log probabilities, and entropy are normalised over the same stored `A_allow`; the mask cannot be removed or recomputed during training. RuleAwareSafetyGate remains the sole source of safety; PPO clipping constrains only the surrogate update. HumanPolicy, opponent population, teacher/RiskSurvival heads, reward, $X$, horizon schedule, and start-state distribution are all frozen within an experiment. The critic is a separate mandatory component targeting the experiment's frozen rollout return, with mode, opponent description, and Elo in its input. Use KL/trust-region control relative to the Stage 2 policy, gradient clipping, and validation early stopping. Preserve an independent checkpoint per seed and never report only the best seed.

Reward semantics: if the objective is only terminal outcome under a fixed opponent distribution, use a terminal outcome conditioned on the starting rules value. Credit acceleration may use only a frozen potential, $F(S,a,S')=\gamma\Phi(S')-\Phi(S)$, and must compare potential shaping with no shaping in a paired ablation. The permitted set contains only `Phi_zero(S)=0` and one W-only candidate, `Phi_w_proof_v1`: for a certified-W state with an existing liveness proof, set $\Phi(S)=-\min\{r_{proof}(S),R_{cap}\}/R_{cap}$, and set $\Phi=0$ in terminal or absorbing states. It is used only in conversion experiments starting from certified W without a mid-trajectory mode switch. Any non-potential process term such as `safe_mass_delta + Gap - SelfRisk + MillBonus` changes the optimal policy and must be named and tested separately as an “alternative objective.” W-mode reward cannot buy off the winning-strategy proof. D mode optimises winning against the frozen target opponent only among D-preserving actions. L mode applies `horizon-safe(X)` first.

Retention requires the preregistered number of independent seeds and a comparison with Stage 2 on paired start states under identical opponent random streams. A PPO checkpoint enters Stage 4 only if the primary-effect interval exceeds the superiority margin; safety, conversion, and natural-distribution metrics satisfy non-inferiority; and the result is not driven by one seed or opponent. “No stable advantage across seeds; retain Stage 2” is a normal successful conclusion.

### Stage 3: Compact Deployment Distillation and Offline Safety Measurement

Stage 3 does not change the teacher objective, but it may train a **fixed-target release student** to eliminate oracle-only inputs and mode routing. It cannot simply package the Stage 2 checkpoint with a safety prior; it must complete the following closure:

1. **Freeze an all-legal-action target and coverage gate**: for every state with an existing Stage 2 label, map the frozen policy distribution for that state's true oracle mode onto all legal actions. Preserve the original group/conditional mass inside `A_allow` and assign target mass zero to proved-denied actions. Do not generate a deployment target for a state with unresolved `proof_unknown`, incomplete candidates, or version mismatch. Run `deployment_target_coverage_gate` first and report eligibility, missingness, and a confidence lower bound by Phase pair, branch count, history depth, proof cost, sector/pack boundary, and target visit mass. Aggregate coverage must not conceal informative missingness concentrated in complex states. If any preregistered critical stratum lies below its floor or missingness correlation exceeds its cap, add proofs or samples only on training-owned data and create a new target version. If the gate still fails, issue `stage_block`; do not represent the easy remainder as the release domain. Once generated, the target snapshot does not change with the compact student.
2. **Unified deployable policy**: by default, distil the target above into one action-conditioned compact head reading only genuine runtime fields such as board, rules history, Phase, and atomic action—not W/D/L, proof rank, teacher tier, or `A_allow`. If the Stage 2 model already satisfies this without further distillation, prove it with its inference graph and feature manifest. If a router is used, accept it separately under the closure invariants in §0.
3. **Runtime safety prior**: derive an independent prior/OOD head from the safety auxiliary. Threshold routing may use only deployable fields such as Phase, pack hit, and completed-search status. Oracle W/D/L and boundaries are offline strata only and cannot choose runtime $\tau$. The prior changes proof-search order and policy suppression only; it cannot authorise a candidate.
4. **Build `endgame_exact_pack`**: read measured sector sizes and choose the pack by benefit per byte of target visit mass and decision closure, comparing at minimum with a “fewest pieces first” baseline. Produce a per-sector hash manifest. Enable the pack only if the current state and every candidate successor hit it, and compare every in-pack query field against P0-A gold. W/D reconstructs only a mode-relative positional set and explicitly handles avoidable immediate historical draws. L supplies only the best positional tier and still requires a survival proof. No pack W may impersonate W liveness.
5. **Calibrate runtime proofs**: on target hardware, freeze global $X_{rt}$ or its Phase-pair/transition mapping, the global floor, candidate expansion order, admissible fast-certificate tiers, node/time budgets, and `proof_pool_early_stop_rule`. Early-stop thresholds may be selected only from strength-latency-energy curves on `selection_dev`; they cannot change after confirmation. Only a complete proof as defined in §6.7 authorises. Report `runtime_survival_proved`, raw `runtime_survival_shortfall`, `runtime_bound_unavailable`, and `proof_pool_incomplete` separately and stratify by authorisation tier, stop reason, and proved/refuted/unresolved pool; never merge them into a timeout fallback.
6. **`compact_exposure_gate` and a non-vacuous support domain**: freeze `release_support_contract` first, then run closed-loop games with the complete runtime stack on training-owned starts only. Record states it induces, candidate pools, OOD, support escapes, unavailable events, theoretical downgrades, W conversion, and runtime latency. In parallel, check closure after every legal opponent reply in enumerable subdomains. If additional labels or retraining are needed, create a new Stage 2/3 data version and rerun. Iterate no more than the contractual limit; failure to close at the limit is `stage_block`. `selection_dev` selects one bundle only after a version is frozen. A state found in `confirmation_dev/final_test` cannot train the current version.
7. **Shadow replay and gates**: replay natural-distribution and stress starts for the frozen bundle on the oracle workbench under §15.4. In the same batch, run the oracle-gated original Stage 2 student, `compact_reference_candidate`, and oracle-gated reference, separating losses from target specialisation, deployment distillation, and the runtime pool. Gates include at least deployment KL/regret, runtime-state support, confidence upper bound on theoretical downgrade rate, `post_policy_unsafe_selection_rate`, over-blocking, zero implementation errors inside the pack, `oracle_X_viability_failure_rate` over every visited state, independent oracle classification of shortfalls, cluster-UCB cap on unavailable, proved-pool censoring, W conversion, L survival, size, and latency.

Stage 3 artifacts are a unified compact policy, bundle, `offline-verified` safety report, runtime-closure artifact, and release-support declaration. If any gate fails, repair a training-owned version without viewing confirmation, or narrow the domain once to a domain still closed under opponent replies. A continued failure is `stage_block`. The distilled artifact has no theoretical adjudication authority, and compact product copy must not use `certified`.

### Stage 4: Frozen Acceptance

Freeze the model, HumanDB, natural and stress tests, start states, random-seed plan, opponent versions, oracle/rule/feature configuration, and statistical-analysis scripts. Disable all learning, threshold adjustment, and online write-back during acceptance. First generate blind predictions or game logs, then run the locked analysis once. Exit only when the preregistered primary gates in §15 pass and a reproducible version manifest is produced. A failure of any primary gate prevents release; do not modify a margin after seeing test and retest on the same test set.

## 13. Teacher Objectives, Training Losses, and Attribution

### 13.1 Three Decision Modes

Every mode first performs legality, complete atomic-action, oracle, and rule-history checks. Lower-level metrics can never compensate for a hard-constraint violation.

| Current full-rules state | Allowed set and primary objective | Teacher objective within the safe set |
| --- | --- | --- |
| W | Preserve positional W, and require the action to belong to a winning-strategy proof over the same augmented history state | In v1, distil proof-rank `q_ref` and continuously cover the W chain; finite-horizon $Q^W$ is only a future, separately approved target-specific tie-breaking objective |
| D | Allow only D-preserving actions; the actor's own move cannot turn D into W | Use the complete Malom ultra-strong policy as the control and maximise the probability of first entering certified W after an error by a target human opponent |
| L | Prefer `horizon-safe(X)`; when fewer than $X$ plies can be guaranteed, maximise $D_{survive}$ | Within the same survival tier, maximise the probability of first rescuing the position to D/W; then break ties by the complete Malom L-value ordering |

Actively permitting W/D→L under `bounded-risk` is a separate product mode available only on the oracle workbench; its UI, model ID, training data, and reports are isolated.

### 13.2 FrozenTeacher Constrained Optimisation and Ties

For each allowed candidate $a$, the teacher stores a structured vector rather than a scalar total score: reference value, W proof artifact/rank where applicable, worst-safe-reply outcome, interval for terminal outcome against the target opponent, diagnostic survival for $T_{pos}$, partial-identification interval for $T_{certW}$, conversion, gate risk, OOD status, and compute-truncation bounds.

The default is `teacher_objective.version = event-prob-v1`. Record trajectories from the AI's perspective. Define $T_{win}$ as the time of a rules-terminal win; $T_{certW}$ as the first entry, after an opponent move from D, into a W state covered by W-liveness; and $T_{rescue}$ as the first entry from L into D or certified W before a terminal loss. Each mode's $H_m(S)$ is derived from the frozen two-actor decision schedule in §10. The default single primary utility is

$$
Q_\nu^m(S,a)=
\begin{cases}
P_\nu(T_{win}\le H_W(S)\mid S,a), & m=W,\\
P_\nu(T_{certW}\le H_D(S)\mid S,a), & m=D,\\
P_\nu(T_{rescue}\le \min\{H_L(S),T_{loss}\}\mid S,a), & m=L.
\end{cases}
$$

This is a finite-horizon teacher event probability, not a long-run real-match win rate. Unexpanded probability mass, censoring, the HumanPolicy ensemble/calibration, and Monte Carlo uncertainty jointly produce a simultaneous interval $[L_a,U_a]$ for each candidate. W liveness, D preservation, the L-mode $X/D_{survive}$ tier, and reference-best-reply checks are hard filters applied before $Q$, not weighted terms within it. First-error rate, safe mass, Mills, mobility, strategic-principle tags, and speed must not be added again. By default, the longest reliable preregistered horizon is primary, and candidates must not be significantly worse than the control at shorter points in $\mathcal H_{eval}$. A “quick trap” that is superior only at a short horizon but whose interval ordering reverses at longer horizons is tied or abstained.

When a D rollout reaches positional W, uncertainty caused by the proof budget is not an ordinary “event did not occur.” Every branch must distinguish `certW_observed`, `positional_W_proof_unknown`, and `not_positional_W`. The lower bound counts only proved mass as an event; the upper bound treats `proof_unknown` probability mass as a possible event and adds the unexpanded residual. If the unknown mass can reverse the candidate ordering, tie the candidates or return `target_abstain`. Reports give `proof_unknown_mass`, proof coverage, and budget by candidate and state complexity, and test whether censoring depends on the candidate or structure. Symmetric offline budgets, precomputation, and a proof cache may narrow the interval, but `proof_unknown` must never be recorded as zero or imputed as certW by ValueNet.

`event-prob-v1` fixes the division of responsibility: the D head optimises only “entry into W already covered by an augmented-history winning proof”; $P(T_{certW})$ does not judge which line will convert after W is reached. The first AI decision after the event must route to the W head. In v1, the W head converts with proof-rank `q_ref`. If the $T_{certW}$ intervals of same-tier D candidates are indistinguishable, retain the reference/tie; conversion, speed, or Mill bonuses must not be inserted back into the D score. Directly optimising `P(certW and eventual conversion)` in the future would require a new joint-objective teacher version. Disabling W specialisation does not leave W without a learnable objective: v1 distils `q_ref` at every decision in the W chain, permitting only actions supported by a full-history proof and ordering them by successor proof rank, while continuously extracting W-to-terminal chains from the same teacher rollouts as a routing curriculum. Before enabling W target specialisation, separately establish what incremental meaning $Q^W$ has over proof rank. By default, $Q^W$ may break ties only within an identical successor-proof-rank tier; selecting across tiers requires a separate objective and preregistered non-inferiority gates for worst-case conversion distance and draw slack.

The **sole label path for error-inducing ranking** is a qualified `T_full` evaluation that computes D-mode event-probability intervals for every allowed candidate of the same `DecisionState`, then creates a non-reference partial order through the three checks below. A HumanDB trajectory in which the opponent later actually erred may only raise acquisition priority and validate event parsing; `T_one` safe mass may only support diagnostics or escalation; scenario tags may only control coverage. Without distinguishable same-state counterfactual evidence, retain `q_ref/tie`. End-to-end reports give `P(certW)`, `P(win | certW)`, joint-chain completion, and final match outcome together; success on the first measure cannot substitute for acceptance on the last merely because training was modularised.

The selection procedure is fixed:

1. Apply the mode-specific hard filters in §13.1.
2. Require the strongest-safe-reply branch to be no worse than that mode's robust baseline, and require gate-intervention risk not to exceed its preregistered tolerance bound.
3. In D mode, use the Malom ultra-strong choice as the active control. Any target-specialised group order that deviates from the reference groups/mass must originate from `T_full`, inherit all three conjunctive checks from Stage 2a—label-generation LCB superiority, agreement in direction under audit, and no degradation under the reference best reply—and also pass `self_continuation_stability_gate`. If any check is absent or the conclusions conflict, forcibly use the reference groups/mass.
4. W mode must not exploit target-opponent fallibility by selecting an action outside the winning-strategy proof. L mode compares $D_{survive}$ before considering a swindle.
5. If intervals overlap or residual rollout mass makes the conclusion unstable, place the candidates in the same tie group. Never force a tie-break using an arbitrary candidate order.

Before constructing a specialised label, save the complete `q_ref`. For every ordered candidate pair $(a,b)$, a qualified `T_full` record stores the simultaneous interval of $Q_\nu^m(a)-Q_\nu^m(b)$. Add an $a\succ b$ edge only when its LCB exceeds the contract-frozen `delta_order.m`; leave every other pair incomparable. The resulting directed graph must be acyclic, or the result is `fatal_stop`. **Every candidate** in the maximal set must, relative to the active reference control, have an LCB meeting `delta_specialize.m` and individually pass all three checks. Let this set be $G_1$. It must also satisfy the conservative group-level separation

$$
d_1=\min_{a\in G_1,\,b\notin G_1}\operatorname{LCB}\!\left(Q_\nu^m(a)-Q_\nu^m(b)\right)\ge\delta_{order.m}.
$$

If even one member of the maximal set lacks authorisation, if $d_1$ fails its gate, or if residual/proof-unknown mass means an unauthorised candidate could still belong to the maximal set, the entire state returns `target_abstain` and falls back field-for-field to `q_ref`. If the checks pass, $G_1$ is the sole specialised top group. Restrict the remaining candidates to the frozen `q_ref` group order, append the non-empty residual groups as $G_2...G_J$, and renormalise the exact reference conditional target over the remaining actions. A disabled mode has a `null` delta and must not construct a specialised graph.

The default mapping from ordered groups to target masses for the primary loss is `group_mass_map.version=bounded-effect-softmax-v2`. For a reference ordinal boundary, define $d_j=\delta_{order.m}$; for an authorised specialised boundary, use the measured $d_1$. Let

$$
g_j=\operatorname{clip}\!\left(\frac{d_j}{\delta_{order.m}},1,C_m\right),\qquad
\ell_1=0,\qquad
\ell_j=-\frac{1}{\tau_m}\sum_{k<j}g_k,\qquad
Q_j=\frac{\exp(\ell_j-\max_r\ell_r)}{\sum_{k=1}^{J}\exp(\ell_k-\max_r\ell_r)},
$$

where $C_m\ge1$, $\tau_m>0$, and `delta_order.m>0` are frozen by mode before label generation. All $g_j=1$ under `T_ref`, reducing the mapping to ordinal rank-softmax. Under `T_full`, only a proved specialised top boundary amplifies inter-group separation by the clipped conservative effect size. Residual, unexpanded, and proof-unknown mass affects only the simultaneous interval, reducing $d_1$, merging groups, or causing abstention. State weight $w(S)$ comes only from the frozen update ledger/sampling contract; trajectory-derived states strictly use `target-measure-iw-v1`. A `T_full` abstention must be bit-for-bit identical to `T_ref` in group IDs, $d_j/g_j/\ell_j/Q_j$, and the conditional tie target. Computation uses fixed float64 arithmetic, stable softmax, canonical candidate order, and a fixed sum tolerance. `bounded-effect-softmax-v2` is the default candidate; `rank-softmax-v1` is only a preregistered equal-budget ablation. Select one using held-out mode regret and ordered-pair calibration on `selection_dev`, pass the winner once through `confirmation_dev`, and then write it into the final contract. Golden samples store, field by field, pairwise intervals, dominance edges, the maximal set, $d_j/g_j$, the three check booleans, group IDs, $\ell_j$, $Q_j$, the conditional target, abstention reason, numerical precision, and schema/generator hash. The active delta is defined from the minimum product-meaningful improvement and approved before any planning-pilot candidate effects are viewed.

`q_ref_tie_policy.version=uniform-canonical-v1`: an exact tie set is produced solely by an equivalence class under the mode-specific **complete comparator**, after sector correction, perspective/`undo_negate`, proof rank, and every lexicographic field have been applied. The within-group conditional target is $c(a\mid G)=1/|G|$. Training computes conditional KL. Deployment transforms the state and actions into the same canonical representation, sorts by canonical atomic-action encoding, and samples from this uniform categorical distribution using a counter-based PRNG. Its seed hash includes the canonical `DecisionState`, mode, policy/tie-policy version, game seed, and actor-decision index; the action is then inverse-transformed. Training and deployment share the comparator, canonicaliser, seed function, and version hash. Switching to a deterministic one-hot canonical tie requires a new tie-policy version, retraining, and reacceptance.

Thus, D mode and ultra-strong are not combined linearly: the reference is the control and fallback, while target event probability replaces the corresponding ordering as a whole only after passing the gates above. Any non-reference specialisation in W or L uses the same three checks. If too few `T_full` samples exceed `delta_specialize`, the system is a safe reference player, not a “successful constrictor specialisation.”

### 13.3 Physical Separation of Losses by Task

- HumanPolicy: observed choice-set categorical NLL plus separate calibration;
- safety auxiliary: allow-tier classification loss over every legal candidate;
- StrategyPolicy: the Stage 2b teacher group-mass listwise loss and a distinguishable partial-order loss;
- opponent-error head (compatibility name: GapNet): separately compute, using their respective at-risk masks, diagnostic $T_{pos}$ survival NLL and D-primary $T_{certW}$ survival/partial-identification loss only from frozen teacher-rollout labels; observed/exact sources are not mixed in, and $T_{pos}$ cannot authorise D acquisition or ranking;
- gate-risk head (compatibility name: SelfRiskNet): survival/probability distillation only from frozen shadow-rollout gate events;
- optional representation heads: their own supervised losses, sharing a backbone only when an ablation authorises it.

These losses have different batch units, data sources, and update counts; they are not written as one mixed-sample loss. The two risk heads may belong to one optional `RiskSurvivalModel`, but their losses, calibration, and versions remain physically separate.

Stage 2b→2c→2d progressively expands the data and acceptance scope; it does not discard earlier objectives stage by stage. Every aggregate-loader round retains the §8 natural/reference replay floor, W/D/L floors, random natural states from the preceding accepted version, and fixed boundary anchors. A new hard state adds update weight only once and only within its cap. Cumulative gates follow `gate_frequency_manifest`: `per_iteration` runs schema/asset hashes, legality/mask/gate smoke tests, a fixed small gold/metamorphic set, prior mode regret, natural-distribution checks, W conversion, and safety-auxiliary non-inferiority; `per_promotion`, before a checkpoint enters `confirmation_dev`, runs stratified differential/property/fuzz tests and proof/cache samples; `per_release`, or whenever any rule/oracle/gold/adapter/schema hash changes, runs full Stage 0 acceptance, enumerable domains, fault injection, and every cumulative gate. If foundational hashes are unchanged, a signed, complete Stage 0 artifact may be reused, but its signature must be verified. The default anti-forgetting controls are replay, cumulative acceptance, and from-scratch retraining on the final locked aggregate set. If functional drift remains observable, a new plan may ablate output-KL/LwF against the frozen old policy on anchor states. Parameter-space L2 distance is not the default.

### 13.4 Quantities That Are Not Unconditionally Valuable

Forming a Mill, capturing, fastest/slowest play, a fixed configuration, mobility, safe width, search-depth disagreement, and human frequency may all be features, scenario labels, or analysis quantities, but none is automatically a good move. The complete Malom comparator and history-aware liveness are oracle rules, not tunable heuristics. A bare `key2` is not called DTW until its reference semantics have been verified. Draws do not receive a universal penalty: in W, a draw is a conversion failure; in D, a draw preserves the theoretical result; in L, a draw is a successful rescue. Every outcome is interpreted relative to the full-rules value of the sampled initial `DecisionState`.

### 13.5 Protocol for Attributing Early Turning Points (“Root Cause” Is Only a Restricted Colloquialism)

A “root cause” cannot be inferred backwards merely because the game was eventually lost, nor equated with the earliest unattractive diagnostic. First separate four forms of evidence:

| Label | Decision basis | Permitted wording and use |
| --- | --- | --- |
| `theoretical_first_error` | The actual actor's complete atomic action lowers coarse WDL below its pre-decision $V_{pos}$, or violates proved liveness while in certified W | Deterministic “first theoretical degradation”; enters safety/hard-state correction |
| `conditional_counterfactual_regret` | The actual action remains in `A_allow`, but under the frozen teacher there is an allowed same-state alternative whose conservative lower bound on mode-specific utility is significantly higher | “Worse under the specified teacher/opponent/horizon”; may generate a listwise label for that state |
| `precursor` | Change in safe mass/width, search disagreement, gate risk, a forced chain, or a Phase boundary | Candidate discovery and explanation only; generates no preference label |
| `posthoc_narrative` | A narrative turning point selected solely from the final outcome or manual review | Presentation only; prohibited from training and formal causal reports |

For the actual action $a_t$ at time $t$, let the version context be $\nu=(\text{mode},\pi_h^{(v)},\pi_{self}^{(u)},\text{oracle/rule},\text{horizon schedule},\text{branch budget})$. Define frozen-teacher conditional regret as

$$
R_t^{\nu}=\max_{a'\in A_{allow}(S_t)}Q_{\nu}(S_t,a')-Q_{\nu}(S_t,a_t),
$$

where $Q_{\nu}$ means only the mode-specific primary utility in §13.2. This quantity is a **simulator-conditional counterfactual given $\nu$**; it must not be described as an intrinsic causal effect of the position. The execution protocol is:

1. Fix the scan rule first: scan every eligible decision, or use a deterministic sampler independent of the terminal outcome. Lock thresholds, minimum effect sizes, and the discovery/confirmation split before viewing the confirmation set.
2. Make the exact determination first. If a theoretical first error has occurred, locate the first actual WDL/liveness degradation. Earlier safe actions may still be reported as having conditional regret, but cannot rewrite this deterministic realised turning point.
3. Perform strategic attribution only within the safe set. Pass the actual action and every alternative through `A_allow` under the same mode, and compare the teacher's complete ordered candidate groups and intervals. If intervals overlap, the conclusion is `undetermined`.
4. Freeze branch versions and perform sensitivity analysis. The primary analysis uses the cross-fitted HumanPolicy ensemble and frozen self continuation; additionally run the reference strongest-safe reply, adjacent frozen self checkpoints, and the preregistered horizon/branch-budget grid. Use `robust_conditional_regret` only when effect direction agrees throughout.
5. Align stochastic coupling semantically. Paired rollouts use a shared latent keyed by comparison ID, actor, the future $k$-th decision, and draw kind. Coupling reduces variance; it does not create causal identification.
6. Control the bias from “scan everything, then select the maximum.” When a game contains multiple scanned plies/alternatives, use simultaneous intervals, a max-statistic/cluster bootstrap, or a preregistered hierarchical test, and report the raw comparison count. A result passes only when $LCB(R_t^{\nu})$ exceeds the locked minimum effect.
7. Handle horizons and censoring. If the difference appears only beyond the compute limit, or residual bounds can reverse the conclusion, record `horizon_limited`/`insufficient_compute`.
8. Permit an attribution set rather than forcing a unique point. If the intervals of several adjacent decisions are indistinguishable, return an attribution set.
9. Report observational and generative evidence separately. Ordinary HumanDB records contain no action propensity, so this strategic attribution cannot claim an action-level causal effect on actual humans.

Training authority is confined to the state being compared. Theoretically unsafe candidates enter only the safety auxiliary/hard-state set. A conditional regret result within the safe set modifies only the teacher listwise label at $S_t$; neither terminal outcome nor regret at step $t$ may be broadcast backwards. A root cause found in the formal test must not flow back into training for the current version.

### 13.6 Reward for the Isolated E-PPO Experiment

Stage 2 has no process reward; it directly distils FrozenTeacher. Only a separately chartered E-PPO experiment needs a reward, using the actual terminal objective under a fixed opponent distribution and conditioned on W/D/L mode. Credit acceleration may use at most the W-only `Phi_w_proof_v1` defined in §12, with a paired `Phi_zero`/no-shaping control; experiments that cross D→W use `Phi_zero` by default. Any non-potential weighted sum changes the task objective and requires a separate teacher/reward version. Terminal outcome must not be indiscriminately broadcast as a supervised label for every earlier action.

## 14. Core Strategic-Scenario Library: Small, Rigorous, and Deliberately Incomplete

Create a `strategy_scenarios` dataset serving only the seven core strategic principles in §3. Each record contains: `scenario_id`, `principle_tag`, a complete legal position and rule-history state, every legal candidate, Malom annotations for both the position and every candidate, the natural-response distribution for the target Elo, the key Phase pair and transition, the principal counterfactual (hold/attack, capture/do not capture, block/do not block), `applicability_evidence` and at least one `matched_counterexample_id`, symmetry variants, and regression-test assertions.

Restrictions: do not collect a comprehensive catalogue of named openings and tactics. Retain only positions that are quantifiable, admit counterfactual construction, and can be verified by Malom. Initial scale need only cover the seven core decisions; add a category only when a fixed test repeatedly exposes a new weakness. Every positive example of “patience/constriction/delayed capture” must be paired with a superficially similar counterexample in which the reference or `T_full` clearly requires immediate conversion. `applicability_evidence` records only which oracle/teacher conditions make that particular comparison valid. An item without a distinguishable counterexample is for presentation only and cannot enter training.

Ten per cent is the total cap for curated counterfactual lessons. A same-state counterfactual pair from this library becomes a primary listwise or distinguishable-pair sample only when complete-oracle/`T_full` simultaneous intervals distinguish it; the `principle_tag` itself may train only an optional concept/contrastive auxiliary head. The 4→3 transition, capture/do-not-capture choices, and long forced chains use separate branch-cost strata in `teacher_budget_contract`. If a complete comparison cannot be made within the approved budget, retain unknown/abstain. Ingestion must proceed in this order: construct or mine; check legality; verify all candidates with Malom; conduct human strategic review; freeze the test. Neither scenario count nor naming is a success metric.

## 15. Acceptance Design

### 15.0 Provenance, Accountability, and Freezing of Primary Thresholds

Before `analytic_power_screen`, `human_data_sufficiency_gate`, or any candidate-effect computation, the product owner creates and signs `objective_thresholds.yaml`. Methodology/statistics reviewers check only that the estimands and power are internally coherent; training implementers have no authority to fill in values ad hoc. A threshold is a product decision about “what absolute improvement over the current frozen product control justifies added complexity, a human trial, and release risk.” It is not inferred backwards from model results, the most favourable seed, or statistical significance. Blinded historical/feasibility data may estimate the control rate, ICC, and event rate for sample-size calculations, but must not be used to move the minimum effect to whatever happens to be detectable.

Unless the product owner supplies other signed values, the v1 **planning defaults** are `delta_human.win=0.10`, `delta_human.conversion=0.10`—both absolute percentage-point differences—`delta_specialize.D=0.03`, and `delta_order.D=0.01`. W/L target specialisation is inactive, so the corresponding deltas are `null`. Before the first screen, any replacement must record the accountable owner, target population/control, minimum product value, unit/horizon, decision time, and a declaration that candidate results have not been viewed; it creates a new objective version. Perform two coherence checks before freezing: whether the two human gates have sufficient power under the recruitment ceiling, ICC upper bound, and `min_future_days`; and whether the D action-level gate is detectable within the resource envelope given HumanPolicy support and proof/residual uncertainty. If either is infeasible, the current strict objective is `not_feasible`; do not lower a delta merely to start the pipeline. `delta_specialize.D` is the teacher mechanism's minimum action-level difference and has no unvalidated algebraic conversion to `delta_human.win`.

### 15.1 Four Evidence Layers; Conclusions Must Not Leap Between Them

1. **Deterministic verification** (oracle workbench): rules, oracle, gate, symmetry/perspective, atomic capture, history, and schema; determines whether the system may operate.
2. **Offline predictive validation**: HumanPolicy, hazard, safety auxiliary, StrategyPolicy teacher-following, and the compact bundle's §15.4 offline safety measurements; determines whether models are trustworthy within the frozen data domain.
3. **Frozen proxy matches**: HumanPolicy/AB/search opponents, with the compact bundle as treatment; establishes proxy performance against those simulators only.
4. **Prospective human A/B test**: compact bundle as treatment; this alone supports “improved against humans.” Without Layer 4, the conclusion must state “the effect on humans has not been measured.”

Acceptance states use hierarchical names. Stage 0 plus a `T_ref` that is fully available throughout the declared support domain produces `reference_safe_baseline` on the oracle workbench only. P2-B1 produces only an unspecialised closure prototype; a reference source becomes `compact_reference_candidate` only after complete P3 closure. A specialised source becomes `compact_proxy_specialised_candidate` only after complete P3 closure and after passing the frozen proxy and `delta_specialize` gates. It becomes `human_validated_release` only when a prospective human A/B test passes both win-rate/conversion superiority and the compact runtime/safety gates. `offline-verified` is only a level of compact-safety evidence; it does not rename a candidate as safe or certified.

The natural-distribution test and balanced stress suite remain separate. The former follows the §7 temporal/player extrapolation protocol without resampling. The latter covers oracle-value boundaries, one-ply traps, gradual constriction, placement→movement, every reachable Phase pair, capture-to-fly, sacrifice/induced capture, repetition/no-progress, W conversion, the L-mode $X$ boundary, and disagreement between shallow and deep search. They are not combined into one weighted score.

### 15.2 Required Controls

- the current Alpha-Beta and current three specialists—the status-quo baselines, explicitly identifying whether they lack a gate;
- Malom strong: preserves only coarse WDL and uses a locked neutral tie policy;
- Malom ultra-strong reference: the complete `move_value` comparator and original tie policy;
- RuleAwareSafetyGate + AB;
- direct online move selection by FrozenTeacher;
- the Stage 2b distilled model and Stage 2c/2d aggregate models, oracle-gated;
- `compact_reference_candidate`, `compact_proxy_specialised_candidate`, and paired controls using their respective oracle-gated sources, separately isolating loss from target specialisation, deployment distillation, and the runtime pool;
- the direct teacher and optional GapNet/SelfRiskNet amortised variants;
- if E-PPO is separately run, the best Stage 2 model and every PPO seed;
- a one-component-at-a-time ablation only for increments actually enabled in this candidate; components disabled by default are not additionally trained merely to fill a “complete matrix.”

All comparable controls use the same start states, rules, colours, time/node-budget semantics, and random streams. When compute budgets differ, also plot the strength–latency Pareto frontier.

### 15.3 Deterministic Acceptance of Safety and Rules on the Oracle Workbench

- Against the reference gold set, compare raw/absolute key, entry kind, complete candidate ordering, WDL, side-to-move change, and symmetry redirect field by field; zero discrepancies are permitted.
- On generated/replayed states, assert 100% query coverage over all legal candidates, query only after a complete capture, and require the final action to belong to `A_allow`.
- In the W-conversion suite, if a rules-terminal winning atomic action exists, both the `q_ref` top group and the actual deployment choice must be among such actions; otherwise the action with minimum successor proof rank must be selected. Zero `missed_immediate_terminal_win` events are permitted. Cover wins that become terminal only after capture, 4→3, repetition/no-progress boundaries, and both colours.
- Property tests cover actor-perspective minimax recurrence, colour exchange, D4 action equivariance, the nine-cell reachable-Phase mask, repetition/no-progress state machines, W liveness, and the adversarial lower bound for `horizon-safe(X)`.
- Report certified and bounded-risk results entirely separately.
- Use exhaustive verification in small enumerable state domains and stratified property/fuzz testing elsewhere; report both covered and uncovered domains.

Observing zero downgrades during execution is reported only as `0/n`. If **independent start states** are approximated as Bernoulli trials, the one-sided $1-\alpha$ zero-event upper bound is $1-\alpha^{1/n}$—approximately $3/n$ at 95%. This is only a development-time synopsis; `n` must not be populated with decision/ply counts from the same game. Formal safety reports use a game/start-state cluster bootstrap, a compatible cluster-binomial upper bound, or an enumerable proof. Using decision-level $3/n$ as the formal upper bound is `stage_block`. No statistical upper bound replaces deterministic tests.

### 15.4 Offline Safety Measurement for the Compact Release Product

Run shadow replay of the frozen compact bundle on the oracle workbench. Include both a fixed decision-state suite and complete games generated in closed loop by the compact runtime stack itself. At every step, adjudicate retrospectively with the complete oracle plus `RuleState`; on the same batch of starts, run the oracle-gated source student, `compact_reference_candidate`, and oracle-gated reference in parallel as paired controls.

- **Primary strategic and theoretical metrics**: `theory_downgrade_rate`—W→D/L and D→L, from the mover's perspective—and its game/start-cluster confidence upper bound. Define `post_policy_unsafe_selection_rate` as the proportion of actual compact moves that do not belong to the oracle-workbench `A_allow` for the same complete `DecisionState`, mode, and history; stratify by pack comparator, runtime prover, policy/pool, rule-state/schema, and replayed authorisation reason. It contains both the product risk of “possessing an $X_{rt}$ proof yet degrading relative to oracle theory-safe” and the implementation error of “moving without any authorisation.” Report them separately, with zero tolerance for the latter. Also report safety-prior false-allow/over-block, deployment KL/regret, frozen-target starting-mass coverage, runtime-state support, and `support_escape_rate`.
- **Pack implementation invariants**: when the pack hits completely and the current state is W/D, the count of actual actions outside the pack's mode-relative positional set must be zero. In L, the count of actions labelled `runtime-loss-bound` without a survival proof must be zero. Any non-zero count is an implementation bug → `fatal_stop`. A partial hit over candidates must not be counted as a pack decision. A pack W is separately checked against full history for W liveness and avoidable rules draws.
- **Support-domain geometry audit**: for every visited state on both the actual pack and proof paths, an independent verifier computes whether at least one legal action satisfies the currently effective $X_{rt}$. The first absence per game/start cluster is the event defining `oracle_X_viability_failure_rate`. Separately attribute it to an already-unsalvageable start, entry into an unsalvageable tail because of an earlier compact choice, or forced entry by the opponent. The non-inferiority control for methodological attribution must be `compact_reference_candidate` under the same starts, pack/prover/$X_{rt}$/deadline/early-stop/safety contract. The oracle-gated reference is used only for absolute disclosure and a theoretical upper bound, not as the control for this relative gate. This is a safety-availability/strategic-quality metric, judged against a preregistered absolute cap and the compact-reference non-inferiority comparison above; a single genuine loss does not automatically turn it into an implementation error.
- **Independent shortfall classification**: every raw `runtime_survival_shortfall` must undergo an oracle audit independent of the runtime prover's code path, cache, and budget. If the oracle finds an action satisfying $X_{rt}$, record `shortfall_contradicted`, apply zero tolerance, and issue `fatal_stop`. If the oracle confirms that no such action exists, record `shortfall_verified_unavoidable`; it contributes only to `oracle_X_viability_failure_rate`, best-effort play, and the actual match result, and does not automatically fail the candidate. Upstream policy responsibility is adjudicated by the first theory downgrade, first X-viability failure, and the paired `compact_reference_candidate` gate on the same trajectory. Under the same first-event/cluster convention, `shortfall_verified_unavoidable` is a subset of `oracle_X_viability_failure_rate`: enforce a numerator-subset assertion, use it only as a nested diagnostic, and neither add it to nor penalise it again with the parent event. If the audit still cannot adjudicate, record `shortfall_audit_unknown` and return `stage_block/inconclusive`; promotion is prohibited. Retain every occurrence for diagnosis, but rates/intervals count only the first event per game or start cluster, so the same doomed tail cannot masquerade repeatedly as independent observations.
- **Budget uncertainty and pool censoring**: do not disguise `runtime_bound_unavailable` as a shortfall, and do not require its raw observed count to be zero. Report its rate and one-sided confidence upper bound by game/start cluster; it must remain below the presigned `max_runtime_bound_unavailable_rate`. If a proved action exists while other candidates remain unresolved, the selected action still has a stepwise guarantee, but `proof_pool_incomplete` indicates censoring of strategic quality. Report the stop reason; legal/proved/refuted/unresolved pool; original policy mass on unresolved actions; whether the top action was unresolved; deliberation time/energy; and paired regret. Use `compact_reference_candidate` under the same runtime stack as the non-inferiority control.
- **Move-execution and ITT invariants**: continuing play when neither pack nor proof has authorised an action, or labelling a timeout/partial search as `runtime-loss-bound`, is `fatal_stop`. By default, `runtime_bound_unavailable` stops the instance and is adjudicated under the presigned rule. If a raw `runtime_survival_shortfall` occurs at runtime and the presigned rule permits it, continue best-effort using the proved lower bound; only afterwards may an independent audit classify it as contradicted, verified-unavoidable, or audit-unknown. Proxy/human ITT records exactly one actual terminal result or presigned adjudication per assignment. Shortfall/unavailable is a separate safety-availability endpoint and cannot add a synthetic second loss.
- Report W conversion and L survival/draw rescue separately inside and outside the pack. Runtime health measures include at least every shortfall subtype, `runtime_survival_proved`, `runtime_bound_unavailable`, `proof_pool_incomplete`, the trigger rate/stop reason/mass coverage of `proof_pool_early_stop_rule`, `pack_decision_hit_rate`, authorising proof tier, compact OOD/unsupported status, and measured size/RSS/latency/energy.
- Freeze every threshold after planning and before confirmation. If a gate fails on training-owned data, only the non-vacuity rules in §15.8 may alter the support domain, or a new version may be generated. Difficult states must not be removed retrospectively from confirmation/final.
- Reports use the term `offline-verified`, state the support domain, sample size, and bundle hash, and report zero events as `0/n` with a cluster upper bound under §15.3; they do not call this a proof.

Input audits prohibit terminal result, future actions, supervision source, Malom-derived labels, or answer-equivalent fields from leaking into a model claimed not to read them at deployment. Report `solved_db`, `heuristic_weak`, observed, and rollout/search separately. In addition to NLL/Brier/calibration, provide confusion, AUPRC, and effective denominator by W/D/L, Phase, theoretical boundary, and class. At minimum, run target permutation, answer-equivalent field masking, a source-only baseline, before/after duplicate-removal comparison, and replay of wholly unseen games. If negative controls do not collapse, or draw/placement samples are near zero, keep the model in shadow with status `stage_block`.

If any fails, produce only the reference baseline. Calibration thresholds, coverage targets, and non-inferiority margins are frozen under the planning/confirmation separation protocol.

Any cell without an advance sample-size allocation is exploratory.

### 15.7 Prospective Human A/B Test

To claim “improved against humans,” randomise the frozen control and treatment under identical product conditions, recording the realised assignment probability, version, anonymised player ID, Elo, colour, time, and complete rules state. The control is the **current frozen product version** named in `objective_thresholds.yaml`; the treatment is the complete `compact_proxy_specialised_candidate`. `reference_safe_baseline` is an oracle training-layer control and cannot replace the human control. The strict player-generalisation primary analysis uses a privacy-preserving membership check with the same salt/version as P0-D to confirm that participants are disjoint from all players seen by HumanPolicy, teacher acquisition, Strategy training, and curriculum tuning. Overlapping players enter a separate returning-player estimand. If identity namespaces cannot be linked reliably, only unverified player-disjointness may be reported. Prefer within-player or player-stratified randomisation; cluster the analysis by player. Lock primary endpoints, minimum effects, loss non-inferiority, exclusion rules, sample size, and stopping rules before the first test game.

Product A/B testing and method attribution are separate. The primary treatment-versus-current-control A/B test establishes only whether the **entire new bundle** is better. Claiming that a gain comes from target specialisation rather than compacting, search, or pack changes additionally requires a `compact_reference_candidate` control under the same runtime stack. If recruitment for this niche game cannot support a three-arm human trial, that attribution remains at the frozen proxy/offline layer. The human conclusion is “the complete product improved,” not “error-inducement training has been causally validated in humans.” If the product owner requires human-level attribution, separately power a three-arm or staged trial; do not decompose a two-arm result after the fact.

The primary human trial must first pass `latency_balance_gate`. Both arms use the same UI, clock semantics, visible “thinking” state, per-move deadline, and a `visible_delay_schedule` determined by public variables other than assignment. Even if computation finishes early, reveal the move only at the same scheduled time. Before the first game, freeze permissible between-arm differences in visible p50/p95/p99 latency, deadline-miss/fallback handling, and ITT rules. Record internal computation-complete and actual reveal timestamps for every move. Timeouts or runtime degradation must not be removed from the assignment denominator; their match results and safety-availability events are accounted for separately as specified next.

The ITT match endpoint receives exactly one result per assignment. If raw `runtime_survival_shortfall` occurs and the presigned rule permits best-effort play, continue to a rules-terminal outcome and count only that outcome. Retrospective shortfall-audit classification affects only the safety report and promotion decision; it neither changes the runtime move condition retroactively nor rewrites the match result a second time. The default fail-closed `runtime_bound_unavailable` stops under the product rule signed before the first game—ordinarily adjudicated as a technical loss or resignation—and that adjudication is the sole match result. `runtime_survival_shortfall`, `runtime_bound_unavailable`, `proof_pool_incomplete`, and proof latency also enter separate safety-availability/performance endpoints, but must not fabricate another loss in addition to the actual terminal result or adjudication. If a separately defined availability-relaxed mode permits unproved moves, it must have a different bundle/product name, an independent `runtime_safety_contract`, independent randomisation, and an independent safety claim; it cannot be mixed into the default candidate. It is a variant within the compact release family, not a new delivery environment.

Before `W_conversion_rate` enters the joint primary effects, it must also pass `w_endpoint_alignment_gate`. The analysis plan first freezes the target positional-W population and full-rules/Phase strata. The oracle workbench constructs the W-start suite offline; every start has a valid proof artifact, both arms receive identical starts, and colour and visible timing are paired. `min_certw_reachable_coverage`—with a planning default that the coverage lower confidence bound is at least 0.95 in every confirmatory stratum—constrains how representative the suite is of the target W population. Reports show `positionalW_count → certW_start → terminal_win` and failure reasons separately by stratum. Every conversion failure by the compact product is a model/runtime failure and remains in the ITT denominator; an increase in suite coverage must not be described as an increase in model conversion rate. Because the D policy may select different W states, `P(win|certW)` after first certW entry in natural games is, by default, only a transport and joint-chain mechanism measure.

Conclusion strength is tiered without impersonation between tiers. A prospective A/B test with linkable identities, no overlap with any training/tuning player, and sufficient preregistered `min_future_days` and player/event power qualifies as `player_disjoint_human_validated_release`. Unseen players but insufficient calendar span is `player_disjoint_short_horizon`. Time-forward data without linkable identity is `time_forward_player_disjoint_unverified`. A randomised A/B test on previously seen players is `returning_player_validated`. “Deliverable product tier” and “completion of the strict research objective” are adjudicated separately. `compact_reference_candidate` may be released without an improvement claim only under a separate reference-product contract after passing runtime/safety, playing-strength non-inferiority, and performance gates. A specialised bundle completes this document's strict objective only when both joint primary effects pass for their preregistered estimands. Passing only one endpoint, a positive point estimate from a few dozen games, or a proxy result cannot be called “improved real-match win rate and winning-position conversion rate.”

Calculate sample size jointly for the two joint primary effects, `delta_human.win` and `delta_human.conversion`, incorporating within-player correlation, repeat participation, attrition, and expected event rate. Take the larger requirement and add the preregistered invalid/disconnection allowance. Order-of-magnitude examples: with a 30% control win rate, a target absolute improvement of 10 percentage points, two-sided 5%, and 90% power for a **single** endpoint, approximately 477 games per arm are required if clustering is ignored; detecting only 5 percentage points requires approximately 1,842 per arm. Both joint primary effects must pass. To conservatively guarantee a joint pass probability of at least 90%, one may first require at least 95% marginal power for each endpoint; the corresponding orders of magnitude are approximately 590 and 2,280 games per arm. A control W conversion rate of 80%→90% requires approximately 330 **valid W-start events** per arm. Freeze the formal numbers through joint cluster-aware simulation using control rate, endpoint correlation, W-start completion, paired discordance, player-cluster ICC, games per player, and invalidity rate estimated in a blinded feasibility pilot. Many games played repeatedly do not impersonate many independent players. The contract caps the information weight of any single player and sets minimum unique-player/cluster counts.

Recruitment for this niche project uses two tiers. First recruit 12–20 people for 4–8 games each in a blinded feasibility pilot, estimating only completion rate, control rate, W-event rate, within-player correlation, and fatigue/order effects; these 48–160 games cannot test whether the model improved. If the minimum meaningful effect is a 10-percentage-point absolute win-rate improvement, a planning-scale confirmation resource target is **1,200–1,600 valid natural-start games and approximately 80–150 independent players**, with paired colours/openings and balanced version order. Separately reserve approximately 600–900 valid games as an initial resource ceiling for the W-start substudy; lock the formal number from paired-design discordance and cluster simulation. A point estimate of ICC from 12–20 clusters is highly unstable. The feasibility artifact must report an interval using player-cluster bootstrap/profile likelihood; sample-size simulation operates over a joint uncertainty set for the control rate, W-event rate, attrition, and ICC, and the maximum recruitment budget uses at least the preregistered one-sided ICC upper bound. A blinded internal pilot in the formal plan may re-estimate nuisance parameters/ICC without examining the treatment effect and adjust up to the preregistered maximum sample. If arm-level effects are viewed, the already-frozen information fractions and alpha spending must be used. If a conservative ICC pushes the required sample beyond the recruitment ceiling, expand collection across batches under the rules or conclude `inconclusive`.

By default, niche recruitment uses blinded within-player or player-stratified randomisation, paired start states/colours, and balanced treatment order; period/order enters the analysis explicitly. The natural-win-rate trial and preregistered W-start conversion substudy may share a joint analysis plan, but their assignment, valid denominators, clusters, and stopping information are accounted for separately. They remain two joint primary effects, each of which must satisfy superiority to pass. If the recruitment ceiling provides insufficient power for the predefined minimum meaningful effect, the only options are expanded multibatch collection, an already-preregistered group-sequential design, or `inconclusive`. Do not lower the minimum effect after seeing results or redefine the independent unit from player to ply. When the expected effect is large, the sequential rule may stop early as specified.

A human trial has only three decisions: `pass`, `fail`, or `inconclusive`. It passes only if the confidence lower bounds for both joint primary effects exceed their respective superiority margins. It fails only when a preregistered harm or futility boundary is crossed. Insufficient sample size, too few W events, or an interval crossing either threshold is always inconclusive; retain the old control rather than taking a “majority vote” over point estimates. Statistical power and temporal coverage are orthogonal gates: many games in a short window cannot establish temporal stability. Until `min_future_days` is met, the strongest temporal-extrapolation conclusion remains `inconclusive/temporal_scope_limited`. If the product requires interim decisions, the analysis plan freezes the group-sequential maximum sample, information fractions, number of looks, alpha spending, futility rule, and joint decision rule before the first game. Otherwise analyse only once at the final sample.

Historical HumanDB, HumanPolicy continuations, and AB profiles cannot replace this trial. If it cannot be run, retain only the `compact_proxy_specialised_candidate` conclusion; do not rewrite “expected to improve against humans” as fact.

### 15.8 Statistical Protocol and Release Gates

Maintain an append-only `promotion_evidence_ledger` for every candidate: candidate/checkpoint/parent and data version; each gate's type, `deterministic | design_resource | statistical`; hypothesis and endpoint family; number of strata/candidates; $\alpha$, interval, and correction method; data used and access count; decision; and artifact hash. Deterministic consistency, resource ceilings, and schema gates have no statistical $\alpha$ and are not folded into an “overall false-positive rate.” What must be controlled is multiple candidates/strata/looks within one statistical family and the candidate-selection process of repeatedly changing candidates after viewing results. The former uses simultaneous/closed testing, a max statistic, or a preregistered hierarchical correction. The latter is constrained by maximum run/seed counts, `selection_dev`, one-time `confirmation_dev`, and the access ledger.

For each data version, only the single complete recipe that wins on `selection_dev` may advance to confirmation. Adding or replacing a model, threshold, stratum, or gate after viewing confirmation opens a new requirement/data-confirmation version. The `compact_proxy_specialised_candidate` report discloses every failed or unrun gate and the number of candidates eliminated. Human A/B testing uses an independent analysis plan and data; it does not inherit statistical significance from the proxy ledger.

- The product objective first specifies `delta_specialize.{W,D,L}` for active `target_specialisation.modes` and the minimum human-A/B improvements. An inactive mode has `null` and prohibits specialisation. The planning pilot estimates only variance, event rates, compute, and sample size; all thresholds are then frozen, and feasibility is adjudicated on a non-overlapping confirmation pilot.
- For paired starts, use paired differences and a cluster bootstrap by start/player/checkpoint, or a compatible mixed model. Report effect, 95% interval, and raw denominator, not only a p-value. With multiple seeds, treat seed/checkpoint as a source of variation rather than selecting the best.
- The only joint primary endpoints for human A/B are `human_match_win_rate` and `W_conversion_rate`. D-mode `P(certW)`, L survival/rescue, and per-mode proxies are mechanism/safety constraints. Include colour in the design/adjustment. Fine Phase-pair and narrower Elo cells support confirmatory conclusions only when separately powered in advance.
- Specialised release requires all of the following simultaneously: every deterministic gate; the preregistered `release_support_contract` meets its target starting-mass floor, and every observed compact action claiming `pack_exact_positional` or `runtime-loss-bound(X_rt)` has replayable authorisation; zero observed `support_escape`, `shortfall_contradicted`, unauthorised moves, or false safety claims, with game/start-cluster upper bounds reported; every `shortfall_audit_unknown` is adjudicated before blinded promotion, otherwise `stage_block/inconclusive`; the cluster confidence upper bound for `runtime_bound_unavailable` does not exceed `max_runtime_bound_unavailable_rate`; `oracle_X_viability_failure_rate` passes its preregistered absolute cap and is non-inferior to `compact_reference_candidate` under the same pack/prover/$X_{rt}$/deadline/early-stop/safety contract, with the oracle-gated reference used only for absolute disclosure; `shortfall_verified_unavoidable`, as a nested subset of the preceding metric, is checked only for subset identity, conditional proportion, and cause distribution, without being added, penalised twice, or compared against a second control; `proof_pool_incomplete` passes rate/mass caps and a paired pool-regret gate against the same compact reference—none of these events requires a raw count of zero; the §15.4 offline safety gates and size/latency budgets; preregistered superiority on both human primary effects; loss/natural-distribution non-inferiority; `latency_balance_gate` and `w_endpoint_alignment_gate`; separate oracle-workbench acceptance of W liveness, D certification, and L/X metrics; and a complete schema/hash/reproducibility manifest. Before confirmation, the training-owned phase may narrow the support domain once, provided that the result is closed under opponent replies and still meets the target-mass floor. No state, L-stress case, or availability event in confirmation/final may be deleted retrospectively or have its membership rewritten. A hard-invariant violation fails directly; legitimate verified-unavoidable/unavailable events are judged by the frozen rate/effect gates.
- An optional reference release of `compact_reference_candidate` uses an independent `reference_release_gate`, which may test only runtime/safety, playing-strength non-inferiority, and engineering budgets. It cannot bypass the specialised human primary effects or mark this document's objective complete.
- Every compact release must pass `claims_surface_gate` on the final bundle. The UI/API/README/status presentation of candidate name, authorisation source, `offline-verified/runtime-loss-bound`, unavailable/shortfall, and human evidence must conform field by field to `product_claims_contract`. The default compact route must neither expose nor enable the oracle-only “perfect DB” control. Omitting a failure state, replacing an evidence level with vague language such as “safe” or “perfect,” diverging between the claim and actual route, or allowing a legacy decision-maker to override the action is a release failure, not a diagnostic warning.
- Do not infer minimum effects backwards from statistical significance or test results. If a fine Phase/Elo cell is under-sampled, collect more data or mark it unknown; an overall average cannot impersonate a pass for that cell. If any specialised primary gate fails, do not replace the specialised product control.

## 16. Redesigning “90%,” Grey Lines, and Similar Metrics

Every percentage must state its actor, condition, unit, denominator, and version.

The **oracle workbench**—for label generation, training, and acceptance—requires complete query coverage only within the support domain where rule options match the `std` database family, every required removal has been completed, the settled sector exists, and differential comparison against the reference has passed. Display separately: `variant_match_rate`—reject mismatches rather than falling back; `required_probe_rate`—the proportion of all settled current/candidate states in the support domain for which a query was actually issued, target 100%; `removal_resolution_rate`—the proportion of mid-removal candidates whose legal continuation yields a stable candidate value, target 100%; `valid_full_value_rate`—the proportion yielding a complete `OracleValue` rather than WDL/steps alone, target 100%; `query_error_by_reason`—separate counts for not mounted, missing sector, hash/symmetry, entry kind, perspective, version, and rule-state, target zero; `theory_safe_choice_count/n`; `w_liveness_certified_count/n`; `horizon_safe_X_count/n` and `survival_shortfall`; `bounded_risk_downgrade_count`, separate from certified theoretical degradation; `raw_gate_intervention_mass/event`, a diagnostic rather than deployment loss; and `teacher_regret` / `ultra_strong_regret`. Any grey line within the support domain must be investigated. “Not called,” “unsupported variant,” “expected mid-removal continuation,” and “unexpected query error” each have a distinct status.

The **compact release product** has no online oracle. Its reports use the §15.4 offline-measurement semantics: the `theory_downgrade_rate` upper bound, `post_policy_unsafe_selection_rate`, safety-prior false-allow/over-block, `pack_decision_hit_rate`, `runtime_survival_proved`, the three-way shortfall classification, `runtime_bound_unavailable` cluster UCB, `proof_pool_incomplete`, `oracle_X_viability_failure_rate`, runtime-state support, and W conversion/L survival inside and outside the pack. Runtime logs record only authorisation source, actual proved depth, pool, event, and version; they do not manufacture a self-assessed “safety rate.” `90%` cannot simultaneously mean query coverage, model confidence, WDL preservation, or human probability.

## 17. Performance and Engineering Constraints

- Before training, estimate and measure the total Malom/search budget as “number of states × mean complete candidate count × teacher branch count × continuation depth.” Separately report cold/warm cache, random/sector-sorted access, p50/p95/p99, throughput, and peak memory.
- Prefer extending the Sanmill Rust/native bridge to return the complete `OracleValue`/comparator. Pure Python serves only as a reference-independent control. Every performance migration first passes the same golden differential; speed is not grounds for reverting to a coarse-WDL API.

- FrozenTeacher multibranch rollouts are the principal offline expense. Configure probability-mass coverage per state, residual bound, maximum nodes/time, and parallelism. Every truncation enters an uncertainty interval; never fill a timeout with a neutral value. On the same states, the teacher pilot performs paired rank-stability checks at 0.5×/1×/2× the approved budget. Any pair whose order can flip with budget enters an interval tie or abstention.

Synchronous direct-teacher fallback for an OOD/high-impact state is permitted only if the direct teacher's cold p99 also meets the online deadline. ## 19. Explicitly Rejected Approaches

- Do not guess that a bare `key2` means “fastest win” or a monotonically decreasing measure. Certified W conversion obeys only the augmented-history AND/OR reachability proof.
- Do not substitute any prediction head for the online Malom gate during label generation, training, or acceptance.
- Do not claim that a compact product's safety prior, distilled policy, or offline measurements establish theoretical non-loss or certification. Do not place the complete tablebase or a large subset of it in the release bundle.
- Do not use human strategic judgement directly as a theoretical label. Do not equate the deepest Alpha-Beta search with human simulation.
- Do not make global win rate the sole metric. Do not allow a specialist model to cross `A_allow`. Bounded-risk requires an explicit mode switch.
- Do not penalise every draw uniformly. Do not universally reward forming a Mill or making a capture.
- Do not let one opening/midgame/endgame label conceal flying differences. Do not make 100% of training games memorise a complete fixed line. Do not simulate low Elo by randomly selecting bottom-ranked candidates.
- Do not conceal model, database, or data errors with a silent fallback.
- Do not copy an actual HumanDB continuation onto an unplayed candidate, and do not treat an unselected action as a negative outcome example.
- In Stage 1, do not jointly train HumanPolicy, safety, Gap/SelfRisk, and StrategyPolicy from scratch in one step. Do not use the modal HumanDB action as a hard StrategyPolicy label.
- Do not use an unvalidated raw GapNet/SelfRisk score directly as teacher truth or reward.
- Do not run PPO by default, refresh reward/teacher/opponent within one PPO experiment, or describe a non-potential process reward as improving only credit assignment without changing the objective.
- Do not execute unsafe raw-policy actions to generate SelfRisk.
- Do not force candidates with overlapping confidence intervals into a tie-free total order. Do not mix labels across teacher/HumanPolicy/Strategy/oracle versions.
- Do not stack derived features or force a shared encoder without ablation, leakage checks, and a latency budget.
- Do not use a random position split, simulated opponent, or arbitrary percentage threshold to claim improvement against humans.
- Do not allow an automated coding agent to bypass the contract/CI with mocks, skips, changed thresholds, or a handwritten `passed: true`.

## 20. Intended Playing Style

After successful training, the machine should behave as follows: in the opening, occupy positions with enduring utility rather than damaging the position's overall freedom to capture one piece early. When the opponent presents no weakness, accept the theoretical draw and preserve structure. Recognise that “this Mill can be formed, but should not be formed now,” and “this piece can be captured, but doing so would release flying.” Use threats to consume the opponent's placements and mobility. Lead the opponent towards patterns familiar to the machine but error-prone for the opponent. If the opponent continues to answer correctly, preserve W/D and continuously satisfy the conversion proof in W. If the current state is already L, first satisfy `horizon-safe(X)`; if that is impossible, maximise guaranteed survival distance and disclose the shortfall truthfully. At the opponent's first error, switch immediately from constriction to reliable conversion.

The strong-guarantee semantics above belong to the oracle workbench. The compact release product is distilled from a frozen source, and its safety and conversion are disclosed faithfully in an `offline-verified` report. Online, only W/D exact-pack closure or a **completed** adversarial proof yields, respectively, a positional guarantee or `runtime-loss-bound(X_rt)`. If every candidate has been refuted below the threshold, display raw `runtime_survival_shortfall` and accept independent classification. If the proved pool is empty while unresolved candidates remain, display `runtime_bound_unavailable` and stop under the presigned rule. If proved actions exist, select only among them and disclose `proof_pool_incomplete`. The essential style remains unchanged: when a non-losing line exists, first establish stability, then make it progressively harder for the opponent to keep finding the correct move; when no non-losing line remains, extend forced survival and pursue a verifiable opportunity to rescue a draw.

## Methodological Basis and Verifiable Sources

- [Paper on Malom ultra-strong / extended solution](https://arxiv.org/abs/1408.0032): multivalued retrograde analysis, internal ordering of draws, objectives against a fallible opponent, and why a one-step coarse game-theoretic value alone cannot guarantee conversion in a cyclic state graph.
- [Malom project documentation](https://www.inf.u-szeged.hu/~danner/mills/): standard database assets and the background of the solver and program.
- [Invalid Action Masking](https://arxiv.org/abs/2006.14171): policy-gradient semantics of a state-dependent invalid-action mask.
- [DAgger](https://arxiv.org/abs/1011.0686): mitigating state-distribution shift in pure offline imitation by querying an expert on states reached by the learned policy.
- [Counterfactual Risk Minimization](https://proceedings.mlr.press/v37/swaminathan15.html) and [Doubly Robust Policy Evaluation](https://arxiv.org/abs/1103.4601): requirements for logged propensities, overlap, and offline counterfactual evaluation.
- [On Calibration of Modern Neural Networks](https://proceedings.mlr.press/v70/guo17a.html) and [Deep Ensembles](https://proceedings.neurips.cc/paper_files/paper/2017/hash/9ef2ed4b7fd2c810847ffa5fa85bce38-Abstract.html): probabilistic calibration and out-of-distribution uncertainty.
- [Policy Invariance under Reward Transformations](http://robotics.stanford.edu/~ang/papers/icml99-shaping.pdf): the structure required for potential-based shaping to preserve the optimal policy under the original objective.
- [PPO](https://arxiv.org/abs/1707.06347): the original clipped policy-gradient algorithm; this document does not interpret clipping as a safety proof.
- [Statistical Precipice](https://arxiv.org/abs/2108.13264): evaluation principles for multiple tasks, multiple seeds, interval estimation, and avoidance of point-estimate-only reporting.

Locally verifiable evidence includes sector decoding in `ai/malom_db.py`, historical-draw implementation in `game/game_engine.py`, and the actual split in `human_database/split_manifest.json`; Sanmill's `crates/perfect-db/src/database.rs`, `wdl_plane.rs`, and `mill.rs`; and the C++ `perfect_wrappers.h` / `perfect_eval_elem.cpp`. Stage 0 artifacts freeze local-code facts; Web sources provide methodological background only.
